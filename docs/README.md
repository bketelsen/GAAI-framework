# GAAI Documentation

## Where to Start

**Just want to try it?** → [Quick Start](guides/quick-start.md) — install, first Story delivered, under 30 minutes. No prior reading required.

**Want to understand why before using?** → Read [What is GAAI?](01-what-is-gaai.md), then [Core Concepts](02-core-concepts.md), then [Getting Started](03-getting-started.md).

**Setting up for a team?** → Path 2 first, then [Senior Engineer Guide](guides/senior-engineer-guide.md) and [Team Setup Guide](guides/team-setup-guide.md).

---

## Reading Paths

### Path 1 — Just try it (30 min, no prerequisites)
1. [Quick Start](guides/quick-start.md) — concrete steps: install → bootstrap → discover → deliver

### Path 2 — Understand first, then use (1 hour)
1. [What is GAAI?](01-what-is-gaai.md) — the problem and the solution
2. [Core Concepts](02-core-concepts.md) — dual-track, agents, skills, backlog, memory, artefacts
3. [Getting Started](03-getting-started.md) — installation and first session checklist

### Path 3 — Full governance setup (for teams)
1. All of Path 2
2. [Senior Engineer Guide](guides/senior-engineer-guide.md) — customize rules, CI integration
3. [Team Setup Guide](guides/team-setup-guide.md) — shared memory, git conventions

> **Quick Start vs Getting Started:** Quick Start is a hands-on walkthrough with examples and use cases — read it to *do*. Getting Started is a reference checklist — read it to *verify* your setup is correct.
