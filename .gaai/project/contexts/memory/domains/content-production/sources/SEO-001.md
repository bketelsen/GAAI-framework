## I. CORE RANKING PRINCIPLES

### AKU-001 — Human-First Authority (E-E-A-T)

**Type:** Normative Principle
**Confidence Level:** High
**Stability Horizon:** Long-term
**Applies To:** All niches (Critical for YMYL)

**Statement:**
Search engines prioritize content demonstrating Experience, Expertise, Authoritativeness, and Trustworthiness.

**Operational Signals:**

* Verifiable author identity
* First-hand experience evidence
* Transparent sourcing
* Clear editorial responsibility

**Guardrail:**
Do not assume E-E-A-T is a numeric score. It is inferred via proxies.

**GAAI mapping:** Informs CNT-003-draft (author signals in content), CNT-005-seo-optimize (E-E-A-T checklist), and CNT-009-quality-gate (E-E-A-T scoring dimension per AGT-001 AKU-AI005).

---

### AKU-002 — Trust Dominance

**Type:** Foundational Signal
**Confidence:** High
**Evidence Type:** Official documentation + consistent field validation

Trust is the dominant axis. Experience, expertise, and authority act as supporting trust vectors.

**Optimization Behavior:**

* Cite primary sources
* Avoid unverified claims
* Update timestamps
* Avoid exaggerated claims

**GAAI mapping:** Core principle for CNT-001-research (source selection bias toward trust) and CNT-004-edit (remove unverified claims during editorial pass).

---

### AKU-003 — Experience Differentiator

**Type:** Correlational (Strong)
**Confidence:** Medium-High

Content with original proof of use (photos, testing data, case evidence) outperforms generic summaries.

**Do:**

* Include original media
* Include quantified testing
* Include methodology explanation

**Do Not:**

* Rely on stock imagery alone
* Publish unverified AI-generated reviews

**GAAI mapping:** Input requirement for CNT-003-draft — research brief must flag whether proprietary experience data is available. See also AGT-001 AKU-AI015R (proprietary knowledge injection).

## II. TECHNICAL PERFORMANCE BASELINE

### AKU-004 — Core Web Vitals as Competitive Floor

**Scope: `coding/` blueprint — context only for content**

**Type:** Normative Threshold
**Confidence:** High
**Applies To:** Competitive SERPs

**Required Baselines:**

* LCP ≤ 2.5s
* INP ≤ 200ms
* CLS ≤ 0.1

**Penalty Correlation:**

* LCP > 3s → higher volatility risk
* INP > 300ms → mobile visibility decline

**Guardrail:**
Performance is a competitive floor, not a differentiator beyond threshold compliance.

**GAAI mapping:** Not actionable in content blueprint. This is a `coding/` skill (frontend performance optimization). Content skills should be aware that poor CWV undermines content SEO, but cannot fix it.

---

### AKU-005 — Mobile-First Indexing

**Scope: `coding/` blueprint — context only for content**

**Type:** Established Infrastructure Rule
**Confidence:** High

Mobile version = canonical ranking reference.

**Optimization Implication:**

* Parity between desktop and mobile content required.
* Structured data must exist on mobile.

**GAAI mapping:** Not actionable in content blueprint. Frontend implementation concern. Content skills should produce mobile-friendly formatting (short paragraphs, scannable structure — see WRT-001 AKU-W020).

## III. AI OVERVIEWS & ZERO-CLICK DYNAMICS

### AKU-006 — AIO CTR Impact

**Type:** Correlational (Tool-based studies)
**Confidence:** Medium

Presence of AI Overviews reduces traditional CTR for top positions.

**But:**
Citation within AIO may increase qualified traffic.

**Guardrail:**
Do not assume universal CTR decline across all intents.

**Cross-ref:** GEO-001 AKU-GEO010 + GEO011 provide a more complete economic model (CTR erosion + conversion amplification).

**GAAI mapping:** Strategic context for CNT-001-research (keyword/topic selection should factor AIO presence) and CNT-009-quality-gate (success metrics should weight conversion over traffic volume).

---

### AKU-007 — Dual Visibility Model

**Type:** Strategic Framework
**Confidence:** Medium-High

Content must optimize for:

1. Traditional ranking

**Cross-ref:** GEO-001 AKU-GEO016 provides the full interaction model shift (Search→Click→Read vs Prompt→Synthesis→Verification).

**GAAI mapping:** Architectural justification for having both CNT-005-seo-optimize and CNT-006-geo-optimize as separate skills in the content pipeline.

### AKU-008 — Semantic Chunking

**Type:** Extraction Optimization Pattern
**Confidence:** Medium

Preferred structure:

* 40–60 word self-contained paragraphs
* Descriptive H2 sections
* Clear topic boundaries

**Reason:**
Facilitates LLM chunk extraction.

**Cross-ref:** GEO-001 AKU-GEO007 (heading hierarchy as chunking markers) + AKU-GEO015 (nugget preference over mega-guides).

**GAAI mapping:** Formatting rule for CNT-006-geo-optimize skill.

---

### AKU-009 — Answer-First Structure

**Type:** Snippet Optimization Pattern
**Confidence:** Medium-High

Lead sections with a direct answer before elaboration.

**Cross-ref:** GEO-001 AKU-GEO005 provides experimental evidence and specific "1-2 sentence" standard. WRT-001 AKU-W009 covers the writing technique (Inverted Pyramid).

**GAAI mapping:** Mandatory formatting rule in both CNT-005-seo-optimize (snippet targeting) and CNT-006-geo-optimize (extraction probability).

---

### AKU-010 — Entity Anchoring

**Type:** Extraction Confidence Booster
**Confidence:** Medium

Replace generic references with specific entities (brands, product models, institutions).

**Cross-ref:** GEO-001 AKU-GEO014 (entity alignment over lexical repetition).

**GAAI mapping:** Checklist item in CNT-006-geo-optimize and editing rule in CNT-004-edit (replace vague references during editorial pass).

---

### AKU-011 — Quantitative Fact Density

**Type:** Correlational
**Confidence:** Medium-Low

Content containing structured quantitative evidence is more likely to be cited.

**Critical Guardrail:**
Never fabricate quantitative density to satisfy heuristic thresholds.

**Cross-ref:** GEO-001 AKU-GEO006 (Specificity Premium + Concreteness Bias mechanism). WRT-001 AKU-W003 (Specificity Bias from a writing perspective).

**GAAI mapping:** Input requirement for CNT-001-research (extract quantitative data from sources) and quality criterion in CNT-009-quality-gate.

## V. TOPICAL AUTHORITY MODEL

### AKU-012 — Topical Coverage vs Keyword Density

**Type:** Normative Shift
**Confidence:** High

Ranking correlates more strongly with semantic breadth than keyword frequency.

**Cross-ref:** GEO-001 AKU-GEO014 confirms keyword stuffing is actively penalized by generative engines.

**GAAI mapping:** Informs CNT-002-outline (outline should cover topic semantically, not repeat keywords) and CNT-005-seo-optimize (validation: semantic coverage > keyword density).

---

### AKU-013 — Topic Cluster Architecture

**Type:** Correlational Strategic Model
**Confidence:** Medium-High

Cluster architecture (pillar + subtopics) correlates with:

* Increased durability
* Higher AI citation likelihood

**Guardrail:**
Clusters must reflect genuine informational architecture, not artificial interlinking.

**GAAI mapping:** Strategic context for CNT-001-research (identify cluster position for each new content piece) and long-term content planning. Informs backlog story creation — related content pieces should be planned as a cluster.

## INSTRUCTION LAYER

When generating SEO recommendations:

1. Classify recommendation type:
   * Normative infrastructure
   * Correlational pattern
   * Debate-sensitive

2. Avoid:
   * Numeric absolutism
   * Universal claims
   * Overfitting tool-study correlations

3. Always:
   * Contextualize by niche
   * Distinguish YMYL vs non-YMYL
   * Clarify evidence strength
