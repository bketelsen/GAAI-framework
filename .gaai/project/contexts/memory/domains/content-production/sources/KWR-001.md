## I. DATA ACQUISITION & EXTRACTION

### AKU-KWR001 — Warehouse-First Query Extraction

**Scope: `coding/` blueprint — methodology concept relevant to content**

**Type:** Normative
**Evidence Source:** Platform Documentation (Google Search Console Bulk Export)
**Confidence:** High
**Causality:** Established

Full-fidelity query extraction requires warehouse-based exports when API row limits or sampling constraints apply.

### Mechanism:

Search Console APIs enforce row limits and filtering constraints. Bulk export to a data warehouse (e.g., BigQuery) provides near-complete daily query-level data (excluding anonymized queries), improving coverage and joinability.

### Operational Implications:

* Prefer bulk export for high-cardinality sites.
* Store raw + normalized query tables.
* Implement scheduled ingestion pipelines.
* Version datasets for reproducibility.

### Guardrails:

* Do not assume API pagination ensures completeness.
* Account for anonymized query suppression.
* Validate coverage vs UI totals.

**GAAI mapping:** Implementation is `coding/` blueprint (data pipeline). For content skills: CNT-001-research must understand that keyword data comes from GSC/BigQuery exports — the research skill consumes this data, it doesn't build the pipeline.

### AKU-KWR003 — Economic Value Proxy via Bid Ranges

**Type:** Normative
**Evidence Source:** Platform Documentation (Google Ads Keyword Planner)
**Confidence:** High
**Causality:** Probable

Top-of-page bid ranges and competition levels serve as proxies for commercial value.

### Mechanism:

Higher bid percentiles correlate with advertiser willingness to pay, indicating monetization potential.

### Operational Implications:

* Integrate CPC low/high range into scoring formula.
* Combine with volume and intent for composite scoring.
* Store bid distribution percentiles.

### Guardrails:

* Do not equate high CPC with guaranteed ROI.
* Adjust for industry inflation bias.
* Treat as economic proxy, not conversion predictor.

**GAAI mapping:** Input signal for CNT-001-research keyword prioritization. When scoring topic candidates, CPC provides the "business value" dimension alongside search volume (demand) and keyword difficulty (effort). Not all content needs high CPC — informational content for authority building may target low-CPC, high-volume terms.

**Confidence:** Moderate–High

## II. SEMANTIC EXPANSION & CLUSTERING

### AKU-KWR005 — Embedding-Based Semantic Clustering

**Scope: `coding/` blueprint — methodology concept relevant to content**

**Type:** Causal
**Evidence Source:** Peer-Reviewed (Sentence-BERT performance benchmarks)
**Confidence:** High
**Causality:** Established

Sentence embeddings significantly reduce similarity computation time at scale while maintaining semantic fidelity.

### Mechanism:

Precomputed dense embeddings allow cosine similarity clustering with dramatically lower computational cost than cross-encoding methods.

### Operational Implications:

* Use embedding pipelines for large keyword sets.
* Re-cluster periodically as new data arrives.
* Combine with SERP validation for precision.

### Guardrails:

* Embedding similarity ≠ ranking equivalence.
* Validate clusters against SERP overlap.
* Monitor drift in embedding models.

**GAAI mapping:** Implementation is `coding/` blueprint (ML pipeline). For content skills: CNT-001-research should understand that semantic clusters group related keywords — the research skill uses cluster outputs to identify topic opportunities and prevent cannibalization, but doesn't implement the clustering.

### AKU-KWR006 — SERP Overlap Clustering for Page Targeting

**Type:** Normative
**Evidence Source:** Industry Workflow Documentation (Ahrefs, similar tools)
**Confidence:** Moderate
**Causality:** Probable

Keywords sharing top-ranking URLs represent shared intent and should map to a single page target.

### Mechanism:

Cluster keywords by percentage overlap of URLs in top search results.

### Operational Implications:

* Prevent keyword cannibalization.
* Assign cluster_id → target_page.
* Define overlap threshold (e.g., ≥3 shared URLs).

### Guardrails:

* Threshold tuning required per niche.
* Do not assume SERP stability.
* Validate cluster homogeneity.

**Cross-ref:** Supports SEO-001 AKU-013 (Topic Cluster Architecture) — SERP overlap clustering is the data-driven method to build the clusters that AKU-013 describes strategically. Also supports SEO-002 AKU-OP005 (Internal Link Graph) — clusters define which pages should link to each other.

**GAAI mapping:** CNT-001-research must check whether a proposed topic is already covered by an existing page (cannibalization check). The research brief should include cluster_id and target_page mapping. New content should either target an uncovered cluster or explicitly refresh an existing page (CNT-010-repurpose).

**Type:** Normative
**Evidence Source:** Tool Methodology Documentation
**Confidence:** Moderate
**Causality:** Probable

Backlink-based difficulty scores approximate ranking effort.

### Mechanism:

Difficulty score correlates with estimated referring domains needed to rank.

### Operational Implications:

* Integrate KD into effort denominator.
* Use percentile thresholds per domain authority.

### Guardrails:

* Proprietary metrics are not universal.
* Validate against historical ranking success.

**Cross-ref:** Related to SRC-001 AKU-CRED011 (Domain Authority as Retrieval Gate) — domain authority determines which KD thresholds are realistic. Low-DA sites must target lower-KD keywords.

**GAAI mapping:** CNT-001-research must include KD in prioritization. For new/low-DA sites: target KD < 30 initially. The research brief should include KD with explicit threshold justification based on current domain authority.

### AKU-KWR011 — Composite Priority Score

**Type:** Heuristic
**Evidence Source:** Industry Practice + Platform Metrics
**Confidence:** Moderate
**Causality:** Unknown

Combining value, demand, effort, and click modifiers improves prioritization vs single-metric sorting.

### Formula:

Priority = (Value × Demand × Trend Modifier × CTR Modifier) / Effort

### Operational Implications:

* Maintain transparent weighting schema.
* Scenario-test weighting sensitivity.
* Version scoring formula.

### Guardrails:

* Avoid overfitting to historical outcomes.
* Do not treat score as deterministic outcome predictor.

**GAAI mapping:** The prioritization formula for CNT-001-research. Each component maps to a specific AKU: Value = CPC proxy (KWR003), Demand = search volume (KWR002), Trend Modifier = momentum (KWR004), CTR Modifier = AI feature adjustment (KWR012), Effort = keyword difficulty (KWR010). The research brief should include composite score + breakdown.

**Type:** Correlational
**Evidence Source:** Industry CTR Dataset (AI Overview Observations)
**Confidence:** Moderate
**Causality:** Correlated

Increased AI Overview presence correlates with reduced top-position CTR.

### Mechanism:

AI-generated summaries may absorb click demand.

### Operational Implications:

* Apply CTR discount factor for keywords with AI Overviews.
* Prioritize queries resilient to AI summarization.
* Track generative visibility metrics.

### Guardrails:

* Correlation ≠ causation.
* CTR impact varies by query class.

**Cross-ref:** Extends SEO-001 AKU-006 (AIO CTR Impact) with the specific operational action: apply a CTR discount factor in the scoring formula. Also extends GEO-001 AKU-GEO010/011 (economic model) with the keyword-level application.

**GAAI mapping:** CNT-001-research scoring adjustment: keywords with high AI Overview presence get a CTR discount in the composite score (KWR011). Counter-strategy: for these keywords, optimize for GEO citation (CNT-006) rather than traditional click-through.

## VI. CNT-001-RESEARCH EXECUTION SEQUENCE (Synthesized)

Based on the AKUs above, the keyword research workflow for CNT-001 is:

1. **Extract seeds** from observed demand (KWR002) + existing GSC data (KWR001)
2. **Expand** with LLM query generation (KWR007) + question cluster mapping (KWR013)
3. **Validate** AI-expanded candidates against demand metrics (KWR008)
4. **Classify** intent per keyword (KWR009)
5. **Cluster** by SERP overlap to prevent cannibalization (KWR006)
6. **Score** with composite formula: (CPC × Volume × Trend × CTR Modifier) / KD (KWR011)
7. **Prioritize** and produce research brief with full metadata per keyword

This sequence directly defines the CNT-001-research skill workflow.
