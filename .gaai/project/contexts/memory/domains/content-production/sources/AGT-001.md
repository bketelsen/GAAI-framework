## I. SYSTEM ARCHITECTURE PRINCIPLES

### AKU-AI002 — Research/Generation Separation

**Type:** Structural Best Practice
**Confidence:** High

High-performing systems strictly separate:

* Research skills (fact retrieval, source validation)
* Writing skills (synthesis and tone calibration)

**Reasoning:**
Separation reduces hallucination risk and improves factual reliability.

**Failure Mode:**
Single-skill approaches attempting both retrieval and narrative synthesis without validation.

**GAAI mapping:** Reinforces separation of CNT-001-research and CNT-003-draft as distinct skills with an artefact handoff (research brief → draft).

---

### AKU-AI003 — Context Stitching for Long-Form Stability

**Type:** Long-Form Generation Control Pattern
**Confidence:** High

Generate long-form content section-by-section (intro, section 1, section 2...) to maintain coherence beyond 1,000 words.

**Purpose:**

* Reduce drift
* Maintain topical consistency
* Lower hallucination probability

**Operational rule:**
Never generate long-form (>1500 words) in one pass when precision is required.

**GAAI mapping:** Core execution pattern for CNT-003-draft skill. The planning sub-agent produces a section-by-section outline; the implementation sub-agent drafts each section with rolling context.

## III. SKILL SPECIALIZATION MODEL

> **Note:** Original report proposed a "multi-agent hierarchy" (Research Agent, RAG Agent,
> Writing Agent, SEO Agent). This is incompatible with GAAI where agents are domain-agnostic.
> Reframed below as skill specializations orchestrated by existing GAAI sub-agents.

### AKU-AI013R — Skill Specialization Over Monolithic Execution

**Type:** System Design Principle
**Confidence:** High

Replace monolithic single-pass content generation with specialized skills:

* Research skill (fact retrieval + source validation)
* Knowledge injection skill (proprietary data / RAG)
* Drafting skill (synthesis + tone calibration)
* SEO review skill (semantic linking + structured data)

**GAAI mapping:** Each maps to a blueprint skill. Orchestration is handled by the existing delivery sub-agents (Planning → Implementation → QA), not by creating new agents.

---

### AKU-AI015R — Proprietary Knowledge Injection

**Type:** Strategic Differentiator
**Confidence:** High

Injecting proprietary data (internal metrics, case studies, original research) provides competitive insulation against commoditized AI content.

**Operational pattern:**

* Proprietary data stored as GAAI memory artefacts (`.gaai/project/contexts/memory/domains/content-production/`)
* Research skill reads memory before external search
* Draft skill receives enriched research brief with proprietary + external sources marked

**GAAI mapping:** Not a separate skill — an input pattern for CNT-001-research. The skill reads `memory/domains/content-production/` for proprietary data alongside external search results. The research brief artefact must tag each source as `[proprietary]` or `[external]`.

---

### AKU-AI016R — Brand-Calibrated Drafting

**Type:** Quality Control Pattern
**Confidence:** High

Drafting must apply tone and voice rules from an explicit Brand Brief artefact.

**Operational pattern:**

* Brand Brief stored in `memory/domains/content-production/voice-guide.md`
* CNT-003-draft skill loads voice-guide as mandatory input
* QA sub-agent validates voice consistency against the guide

**Guardrail:**
Never allow synthesis without validated input from research layer.

## IX. RISKS & LIMITATIONS

### UZ-AI001 — Workflow Complexity Risk

Advanced automation stacks require ongoing maintenance. Over-engineering the skill chain creates fragility.

**Mitigation:** Start with minimal viable skill chain (research → outline → draft → quality-gate). Add optimization skills (SEO, GEO, social-adapt) incrementally based on measured need.

---

### UZ-AI002 — De-Ranking Exposure

Purely AI-generated content without human oversight remains vulnerable to algorithm updates.

**Mitigation:** Mandatory human-in-the-loop at quality gate. Never auto-publish content that hasn't passed human review.

---

### UZ-AI003 — Brand Brief Dependency

Automated content quality depends heavily on the quality of the initial brand calibration.

**Mitigation:** `memory/domains/content-production/voice-guide.md` must be written by the founder and iterated after the first 5 published pieces. Not a one-time artifact.
