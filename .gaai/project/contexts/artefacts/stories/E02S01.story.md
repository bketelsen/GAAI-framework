# Story E02S01: Dashboard Scaffold

## User Story

**As an** expert visiting app.callibrate.io,
**I want** a fast, responsive dashboard application with server-side authentication,
**so that** I can securely access my profile, leads, bookings, and analytics without experiencing flashes of unauthenticated content.

---

## Context

The expert dashboard is the first expert-facing frontend in the Callibrate platform. It lives at `app.callibrate.io` and is built as a React Router v7 application in Framework Mode (SSR) deployed to Cloudflare Workers via the Cloudflare Vite plugin (DEC-113).

**Why React Router v7 Framework Mode (not SPA):**
- Server-side loaders replace TanStack Query boilerplate (less code, fewer bugs, auto-revalidation on actions)
- Auth guard in layout loader — redirect to `/login` before HTML is sent (no flash of unauth content)
- CF's recommended full-stack path: "the first full-stack framework to provide full support for Cloudflare Vite plugin"
- Same React components underneath — no learning curve penalty

The scaffold establishes the foundation that all subsequent E02 stories build on: auth flow, layout shells, API client, component library, CI/CD pipeline.

**Workspace:** `workers/frontend/dashboard/` — new npm workspace in the monorepo.

- [ ] AC3: Workers entry at `workers/app.ts` delegates to React Router's request handler. `app/entry.server.tsx` handles SSR rendering.

### Authentication

- [ ] AC4: Supabase Auth integration via `app/lib/session.server.ts`. Cookie-based session management using Supabase PKCE flow. Functions: `getSession(request, env)` → returns `{ user, token }` or `null`, `requireSession(request, env)` → returns session or throws redirect to `/login`.

- [ ] AC5: Auth layout route (`app/routes/_layout.tsx`): loader calls `requireSession()` — if no valid session, server-side redirect (302) to `/login`. Unauthenticated users never receive dashboard HTML. Loader passes `{ user }` to all child routes via `Outlet` context.

- [ ] AC6: Minimal login page (`app/routes/login.tsx`): email/password form calling `supabase.auth.signInWithPassword()`. On success, sets session cookie and redirects to `/dashboard`. Basic error display. Full login UX (forgot password, onboarding redirect) delivered in E02S02 AC2.

## Notes

- **React Router v7 route convention:** file-based routing in `app/routes/`. Nested routes use dot notation (e.g., `dashboard.leads.tsx` is a child of `_layout.dashboard.tsx`). Loaders run server-side, actions handle form submissions.
- **No TanStack Query or zustand:** React Router v7 loaders replace client-side data fetching. URL search params replace client-side state for filters/pagination (RR7 idiom: URL is the state).
- **Cookie-based sessions:** Supabase Auth tokens stored in HTTP-only cookies. The `getSession()` helper extracts and validates on each request. Token refresh handled automatically by Supabase JS `getUser()`.
- **CORE_API_URL:** Points to the callibrate-core Worker API (`https://api.callibrate.io` in production, staging URL for staging). Service Bindings are an alternative but introduce coupling — HTTP fetch keeps the dashboard independently deployable.
- **shadcn/ui:** Components are copied into the project (not imported from npm). This is by design — allows customization without forking.
- **Delivery decision:** The exact shadcn/ui component set may expand during E02S02–E02S08 delivery. AC13 establishes the minimum set; subsequent stories add components as needed.
