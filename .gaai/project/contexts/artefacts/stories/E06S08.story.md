# Story E06S08: AI Freetext Extraction Service (Project Description ‚Üí Structured JSONB)

## User Story

**As a** prospect describing my AI integration project in natural language,
**I want** the platform to understand and structure my requirements automatically,
**so that** I get accurate matches without having to learn technical vocabulary.

---

## Context

Static quiz forms are friction-heavy and miss nuanced project context. Prospects understand their problem but not the structured vocabulary to fill in fields. This service accepts freetext and uses Claude (claude-haiku-4-5) to extract `ProspectRequirements` JSONB with per-field confidence scores. Low-confidence fields (< 0.7) trigger targeted confirmation questions ‚Äî max 2‚Äì3. High-confidence extractions go straight to matching. This service is stateless: it does not write to DB; the caller (satellite funnel or E06S07) persists the result.

---

## Acceptance Criteria

- [ ] AC1: `POST /api/extract` accepts `{ text: string, satellite_id?: string }` ‚Äî `text` is the prospect's freetext project description, max 2000 characters
- [ ] AC2: Extraction uses Claude `claude-haiku-4-5` with a structured output prompt that returns `ProspectRequirements` JSONB + a `confidence` score (0.0‚Äì1.0) per field, in a single API call
- [ ] AC3: Response shape: `{ requirements: ProspectRequirements, confidence: FieldConfidence, needs_confirmation: string[], ready_to_match: boolean }` ‚Äî `needs_confirmation` = array of field keys with confidence < 0.7; `ready_to_match = true` when `needs_confirmation` is empty
- [ ] AC4: `ProspectRequirements` extracted fields: `challenge` (string ‚Äî problem summary), `skills_needed` (string[]), `industry` (string), `budget_range` (`{ min: number, max: number }` in EUR), `timeline` (string), `company_size` (string), `languages` (string[])
- [ ] AC5: When `needs_confirmation` has items: response includes `confirmation_questions: Question[]` ‚Äî max 3 questions, each targeting one low-confidence field, generated in the same LLM call. `Question` shape: `{ field: string, question: string, options?: string[] }`
- [ ] AC6: When `ready_to_match = true`: no confirmation questions returned ‚Äî caller can immediately invoke `POST /api/matches/compute` with the extracted requirements
- [ ] AC7: Extraction is stateless ‚Äî does not read from or write to DB. Caller is responsible for persisting extracted requirements to `prospects.requirements`
- [ ] AC8: Claude API calls route through Cloudflare AI Gateway (`CLOUDFLARE_AI_GATEWAY_URL` env var) for usage logging and cost visibility. `ANTHROPIC_API_KEY` bound as a Worker secret
- [ ] AC9: When `satellite_id` is provided: loads `satellite_configs.vertical` from DB and injects it as context in the extraction prompt ‚Äî improves accuracy for niche-specific vocabulary (e.g., "healthcare" vertical ‚Üí better recognition of compliance-related requirements)
- [ ] AC10: Integration test: given a realistic 150-word freetext description of an AI automation project, extraction produces all 7 fields with at least 5 having confidence > 0.5

---

## Out of Scope

- Multi-turn conversational extraction (the confirmation step is single-round, not iterative)
- Feedback loop integration for extraction quality (E06S09 handles score computation, not extraction quality tracking at MVP)
- Image or attachment processing (text only)
- Streaming responses (standard request-response)

---

## Dependencies

- E06S01 (Worker environment + secrets management)
- E06S02 (`satellite_configs` table for vertical context ‚Äî optional path)
- Anthropic API key provisioned + Cloudflare AI Gateway configured

## Notes

Model choice: `claude-haiku-4-5` for cost efficiency ‚Äî extraction is high-frequency (every prospect submission). Consider `claude-sonnet-4-6` post-MVP if extraction accuracy proves insufficient. Cloudflare AI Gateway URL format: `https://gateway.ai.cloudflare.com/v1/{account_id}/{gateway_id}/anthropic`. The prompt must explicitly instruct the model to return a JSON object with `requirements` and `confidence` keys ‚Äî use the Anthropic structured output / tool use feature to enforce schema. Budget extraction heuristic: look for EUR amounts, "k‚Ç¨" notation, and range expressions ("entre X et Y").
