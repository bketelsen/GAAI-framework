# Story E06S30: Google OAuth — Sensitive Scope Review (calendar.events)

## User Story

**As a** platform operator,
**I want** the Google OAuth app approved for the `calendar.events` sensitive scope,
**so that** experts can connect their Google Calendar and prospects can book calls with a Google Meet link.

---

## Context

E06S10 (GCal OAuth) et E06S11 (Booking engine) sont livrés. Mais le scope `https://www.googleapis.com/auth/calendar.events` est classé "sensitive" par Google, ce qui signifie :

- Sans vérification : l'app fonctionne uniquement pour les **test users** listés dans la Google Cloud Console (max 100 comptes). Elle affiche un écran d'avertissement "Cette application n'est pas vérifiée".
- Avec vérification : l'app est accessible à tous les utilisateurs sans avertissement.

Le processus de vérification peut prendre **4 à 6 semaines** (parfois plus). C'est le **principal bloquant go-live** cité dans la mémoire projet (Known Constraints). À initier immédiatement — indépendamment du reste.

**Pour le MVP staging :** les tests internes fonctionnent tant que le founder est listé comme test user. Mais les experts invités à tester ne peuvent pas être ajoutés à l'infini (max 100).

Cette story est **founder-executed** : formulaires Google + création de pages légales.

---

## Acceptance Criteria

### Prérequis légaux

- [ ] AC1: Page Privacy Policy publiée sur `https://callibrate.io/privacy` — contenu minimum : quelles données sont collectées (email, Calendar events), pourquoi (booking service), avec qui partagées (personne), droit de suppression. Google exige une URL publique accessible au moment de la vérification.
- [ ] AC2: Page Terms of Service publiée sur `https://callibrate.io/terms` — recommandé (Google peut demander)

### Vérification domaine

- [ ] AC3: Domaine `callibrate.io` vérifié dans Google Search Console (`https://search.google.com/search-console`) via méthode DNS TXT (Cloudflare DNS → ajouter le TXT record Google fourni)
- [ ] AC4: Dans Google Cloud Console (APIs & Services → OAuth consent screen) : `callibrate.io` ajouté comme authorized domain

### Soumission

- [ ] AC5: OAuth consent screen complété dans Google Cloud Console :
  - App name: "Callibrate"
  - User support email: `support@callibrate.io`
  - App logo uploadé (512×512px, PNG)
  - Authorized domains: `callibrate.io`
  - Privacy Policy URL: `https://callibrate.io/privacy`
  - Terms of Service URL: `https://callibrate.io/terms`
- [ ] AC6: Scopes requis déclarés : `https://www.googleapis.com/auth/calendar.events` (sensitive) + `https://www.googleapis.com/auth/userinfo.email` (non-sensitive)
- [ ] AC7: App publiée ("Publish app" dans OAuth consent screen) + bouton "Prepare for verification" cliqué
- [ ] AC8: Formulaire de vérification Google complété :
  - Justification du scope `calendar.events` : "Experts connect their calendar to display availability to prospects and create booking events with Google Meet links. The platform reads free/busy data and creates calendar events on behalf of experts who have explicitly granted access."
  - Demo video enregistrée (screen capture du flow GCal connect + booking) et uploadée
  - Contact email : `ops@callibrate.io`
- [ ] AC9: Confirmation Google "your app is under review" reçue par email à `ops@callibrate.io`

### Go-live (long-term)

- [ ] AC10: Confirmation Google "your app is approved / verification complete" → experts peuvent connecter leur Calendar sans écran d'avertissement
- [ ] AC11: Test post-approbation : expert (compte Google externe, non test user) complète le flow GCal OAuth sans warning

## Dependencies

- Aucune dépendance sur les autres stories E06 — peut être initié immédiatement

## Notes

**Test users (pour le MVP avant approbation) :** Google Cloud Console → OAuth consent screen → Test users → Ajouter les emails des experts beta. Limite : 100 comptes. Suffit pour les premiers tests mais pas pour le go-live public.

**Délai moyen :** 4 à 6 semaines après soumission complète. Google peut demander des informations complémentaires — répondre dans les 48h pour éviter de perdre sa place dans la file.

**Lien de suivi :** Google Cloud Console → APIs & Services → OAuth consent screen → "Verification Status".

**Privacy Policy minimale :** si `callibrate.io` n'a pas encore de frontend, déployer une page HTML statique temporaire via CF Pages ou une route Next.js. La page doit être accessible sans authentification.
