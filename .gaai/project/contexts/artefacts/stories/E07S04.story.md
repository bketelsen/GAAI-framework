# Story E07S04: PostHog Funnel Dashboards via API

## User Story

**As a** founder,
**I want** pre-configured PostHog dashboards showing prospect and expert conversion funnels,
**so that** I can immediately see where users drop off without building queries manually.

---

## Context

Once events flow (E07S02 + E07S03), the data needs to be visualized. PostHog has a Dashboard API that allows programmatic creation of dashboards and insights. This story creates the initial dashboards via a one-time setup script using the PostHog API, so the founder has immediate visibility from day one.

---

## Acceptance Criteria

- [ ] AC1: Script `scripts/posthog-setup-dashboards.ts` creates dashboards via PostHog API (`POST /api/projects/:id/dashboards/`)
- [ ] AC2: **Prospect Conversion Funnel** dashboard with insight:
  - Funnel steps: `prospect.form_submitted` → `prospect.matches_viewed` → `prospect.identified` → `booking.held` → `booking.confirmed`
  - Breakdown by `satellite_id`
  - Time range: last 30 days
- [ ] AC3: **Expert Activation Funnel** dashboard with insight:
  - Funnel steps: `expert.registered` → `expert.profile_updated` → `expert.gcal_connected` → `booking.confirmed`
  - Time range: last 30 days
- [ ] AC4: **Business Overview** dashboard with trends:
  - Daily `prospect.form_submitted` count
  - Daily `booking.confirmed` count
  - Daily `expert.registered` count
  - Overall conversion rate (form → booking)
- [ ] AC5: Script reads `POSTHOG_PERSONAL_API_KEY` and `POSTHOG_PROJECT_ID` from environment variables (Dashboard API requires Personal API Key, not Project API Key)
- [ ] AC6: Script is idempotent — running twice does not create duplicate dashboards (checks by name before creating)
- [ ] AC7: Script outputs dashboard URLs on success
- [ ] AC8: `SETUP.md` updated with dashboard setup instructions

---

## Out of Scope

- Real-time alerting (phase 2)
- Custom retention cohort dashboards (can be added manually in PostHog UI)
- Slack/email notifications on threshold breach (phase 2)

---

## Dependencies

- E07S02 + E07S03 (events must be flowing for dashboards to show data)
- **Human action required:** PostHog Personal API Key (not Project API Key) needed for Dashboard API authentication

---

## Notes

PostHog Dashboard API uses Personal API Key authentication (not Project API Key). The script should use `POSTHOG_PERSONAL_API_KEY` separately. The dashboards are a one-time setup — ongoing modifications happen in PostHog UI.

PostHog API base: `https://eu.posthog.com/api/projects/{project_id}/dashboards/`
