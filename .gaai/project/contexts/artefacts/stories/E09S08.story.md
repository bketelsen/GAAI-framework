# Story: Production deploy test gates for satellite and posthog-proxy

## User Story

**As a** founder deploying to production,
**I want** type-check and vitest to run before satellite and posthog-proxy production deploys,
**so that** production has the same safety guarantees as staging.

---

## Context

`deploy-production.yml` runs `tsc --noEmit` + `vitest run` for `deploy-core` only. The `deploy-satellite` and `deploy-posthog-proxy` jobs deploy to production with zero validation — no type-check, no tests. This is less safe than staging (which now has test gates via E09S01).

---

## Acceptance Criteria

- [ ] AC1: `deploy-satellite` job in `deploy-production.yml` runs `tsc --noEmit` in `workers/frontend/satellites` before the `wrangler deploy` step
- [ ] AC2: `deploy-satellite` job runs `vitest run` in `workers/frontend/satellites` before the `wrangler deploy` step
- [ ] AC3: `deploy-posthog-proxy` job runs `tsc --noEmit` in `workers/backend/posthog-proxy` before the `wrangler deploy` step
- [ ] AC4: `deploy-posthog-proxy` job runs `vitest run` in `workers/backend/posthog-proxy` before the `wrangler deploy` step
- [ ] AC5: If type-check or tests fail, the deploy step does NOT execute (job fails early)
- [ ] AC6: No changes to `deploy-staging.yml`, `preview-deploy.yml`, `deploy-dashboard.yml`, or `deploy-landing.yml`

> All acceptance criteria must be verifiable. Vague criteria are invalid.

---

## Out of Scope

- Adding new test files
- Matching-engine production pipeline (covered by E09S02 AC4)
