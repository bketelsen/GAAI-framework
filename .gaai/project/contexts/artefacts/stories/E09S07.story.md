# Story: Fix broken satellite booking-widget tests

## User Story

**As a** developer merging to staging,
**I want** all satellite tests to pass,
**so that** the deploy-staging pipeline is unblocked and E09S01 test gates work as intended.

---

## Context

E09S01 added test gates to `deploy-staging.yml`. Immediately after merge, 4 tests in `workers/frontend/satellites/src/pages/booking-widget.test.ts` began failing, blocking all staging deploys:

- `includes renderSuccess function`
- `renders meeting_url as a link`
- `renders prep_token as a prep link`
- `constructs prep URL from prep_token`

The 4 last `Deploy to Staging` runs are FAILURE. No code can reach staging until these are fixed.

---

## Acceptance Criteria

- [ ] AC1: All 4 failing tests in `booking-widget.test.ts` pass with `vitest run`
- [ ] AC2: No other tests are broken — full satellite test suite passes (0 failures)
- [ ] AC3: `Deploy to Staging` workflow completes successfully on the next push to staging
- [ ] AC4: Fix addresses the root cause (test expectations vs actual implementation), not by deleting or skipping tests

> All acceptance criteria must be verifiable. Vague criteria are invalid.

## Out of Scope

- Adding new booking-widget tests
- Refactoring booking-widget implementation
- Changes to other workers

## Dependencies

- None. This is a hotfix that unblocks the pipeline.
