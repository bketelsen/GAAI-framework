# Story E02S09: callibrate.io Expert Landing Page

## User Story

**As an** AI integration expert discovering Callibrate,
**I want** a clear, compelling landing page that explains the platform's value proposition,
**so that** I can decide to sign up and understand what I'm paying for before creating an account.

---

## Context

The expert-facing landing page at `callibrate.io` is the primary acquisition page for experts. It translates the validated promise language (DEC-102) into a marketing page with clear value props, pricing transparency, and a strong CTA to `app.callibrate.io/signup`.

**Tech stack (DEC-115):** CF Worker + Hono — same pattern as the satellite Worker (`workers/frontend/satellites/`). Server-rendered HTML, no client-side framework. This is a marketing page, not an app — it needs to be fast, SEO-optimized, and lightweight.

**Workspace:** `workers/frontend/landing/` — new workspace in the monorepo.

---

## Acceptance Criteria

### Worker Setup

- [ ] AC1: New workspace at `workers/frontend/landing/`. CF Worker using Hono (TypeScript). `wrangler.toml` with env-specific names (`callibrate-landing-dev`, `-staging`, `-prod`). Registered in root `package.json` workspaces.

- [ ] AC2: Routes: `GET /` (landing page), `GET /health` (200 OK), `GET /robots.txt` (allow all, sitemap reference), `GET /sitemap.xml` (static — `/` only).

### Landing Page Content

- [ ] AC3: Hero section with expert-facing promise (DEC-102 vocabulary): headline derived from "Stop filtering tire kickers. Get matched only with prospects who have confirmed budgets, real timelines, and a problem worth solving." Subheadline reinforcing the value. Primary CTA: "Commencer" → `https://app.callibrate.io/signup`.

- [ ] AC4: Value propositions section — 3-4 cards highlighting key benefits:
  - Pre-qualified leads (no more tire kickers — budget confirmed, timeline real)
  - Dynamic pricing transparency (49€–263€ grille publiée, pay only for confirmed leads)
  - Your calendar, your rules (GCal integration, availability controls, 7-day flag window)
  - Performance dashboard (composite score, lead analytics, conversion tracking)

- [ ] AC5: Pricing section — transparent pricing grid (DEC-67):
  - Non déclaré / < 5k€: 49€ (standard) / 56€ (premium)
  - 5k–20k€: 89€ / 102€
  - 20k–50k€: 149€ / 171€
  - 50k€+: 229€ / 263€
  - 100€ welcome credit mentioned. "Pas d'abonnement. Payez uniquement pour les leads confirmés."

- [ ] AC6: How it works section — 4-step visual flow: Créez votre profil → Recevez des leads qualifiés → Confirmez ou signalez → Suivez vos performances.

- [ ] AC7: Social proof section — placeholder structure (testimonial cards with placeholder content). Ready to populate with real testimonials post-launch.

- [ ] AC8: Footer with links: app.callibrate.io (dashboard), /privacy, /terms, contact email. Copyright.

### Technical Quality

- [ ] AC9: PostHog tracking (same pattern as satellites, E07S02): `page_view` on load, `landing.cta_clicked` on CTA clicks (with `cta_location`: hero/pricing/footer). Cookieless (`persistence: memory`). PostHog JS snippet in `<head>`. `POSTHOG_API_KEY` from env.

- [ ] AC10: SEO: `<title>` "Callibrate — Pre-qualified leads, booked to your calendar", `<meta name="description">`, OG tags (og:title, og:description, og:image, og:url), JSON-LD Organization structured data, canonical URL.

- [ ] AC11: Security headers (same middleware as satellite Worker, E08S02): X-Content-Type-Options, X-Frame-Options, HSTS, Referrer-Policy, Permissions-Policy, CSP.

- [ ] AC12: Mobile responsive: all sections stack vertically on viewports < 768px. Minimum touch target 44px. Pricing grid scrolls horizontally on mobile if needed.

- [ ] AC13: Performance: no client-side framework, inline critical CSS, total page weight < 50KB (HTML + CSS + PostHog snippet). Lighthouse performance score ≥ 90.

### CI/CD

- [ ] AC14: GitHub Actions workflow `.github/workflows/deploy-landing.yml`: triggers on push to `staging`/`main` when `workers/frontend/landing/**` changes. Steps: install, type-check, deploy via `wrangler deploy --env {staging|production}`.

- [ ] AC15: `tsc --noEmit` — zero errors.

## Notes

- **DEC-102 vocabulary is the source of truth** for all marketing copy on this page. Avoid inventing new language — use validated terms: "tire kickers", "confirmed budgets", "real timelines", "workflow architect", "found time".
- **Pricing transparency is a core value.** The pricing grid is not hidden behind a "Contact us" wall. Experts see exactly what they'll pay before signing up.
- **Same-origin cookie:** `callibrate.io` and `app.callibrate.io` share the `callibrate.io` domain. If the expert is already logged in on `app.callibrate.io`, the CTA could detect the session and show "Accéder au dashboard" instead of "Commencer". This is a post-MVP enhancement — at scaffold, always show "Commencer".
- **Privacy/Terms pages:** Already deployed on the satellite Worker (E06S30). The landing Worker links to `callibrate.io/privacy` and `callibrate.io/terms` — these routes need to be served by the satellite Worker or duplicated. Delivery to assess routing: either CF route rules send `/privacy` and `/terms` to the satellite Worker, or the landing Worker serves its own copies.
