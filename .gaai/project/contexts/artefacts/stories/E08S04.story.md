# Story: Admin auth hardening + booking input validation

## User Story

**As the** platform operator,
**I want** admin endpoints to use a dedicated API key with constant-time comparison, and booking endpoints to validate all inputs strictly,
**so that** admin access is properly isolated and invalid data cannot enter the system.

---

## Context

SEC-AUDIT-001 findings SEC-06 (HIGH: admin reindex uses SUPABASE_SERVICE_KEY with `!==`) and SEC-07 (HIGH: no email/date validation on confirm/reschedule).

---

## Acceptance Criteria

- [ ] AC1: A new secret `ADMIN_API_KEY` is added to the Env interface. The `handleVectorizeReindex` function checks `Authorization: Bearer ${env.ADMIN_API_KEY}` instead of `SUPABASE_SERVICE_KEY`.
- [ ] AC2: The comparison uses constant-time equality: encode both strings to `Uint8Array`, use `crypto.subtle.timingSafeEqual()` (or polyfill via HMAC-based comparison if not available in CF Workers runtime). If lengths differ → reject immediately (acceptable — length is not secret).
- [ ] AC3: `handleConfirm` validates `prospect_email` with a strict email regex or Zod `z.string().email()`. Invalid → 422.
- [ ] AC4: `handleConfirm` validates `prospect_name` as `z.string().min(1).max(255)`. Invalid → 422.
- [ ] AC5: `handleReschedule` validates `new_start_at` and `new_end_at` as valid ISO-8601 datetimes (Zod `z.string().datetime()` or `new Date()` + `isNaN` check). Also validates `new_start_at < new_end_at` and `new_start_at` is in the future. Invalid → 422 with specific error.
- [ ] AC6: SETUP.md is updated with `wrangler secret put ADMIN_API_KEY --env staging` and `--env production` instructions.
- [ ] AC7: All existing vitest tests pass. New tests: (a) admin reindex with wrong key → 401, (b) admin reindex with correct key → proxied, (c) confirm with invalid email → 422, (d) reschedule with end < start → 422.
- [ ] AC8: tsc clean.

---

## Out of Scope

- Per-admin audit logging (post-MVP)
- Admin IP whitelisting (CF Access, dashboard config)
- Rate limiting on admin endpoints

---

## Dependencies

- None

## Notes

- `crypto.subtle.timingSafeEqual` is not natively available in CF Workers as of 2026-02. Polyfill via: compute HMAC of both inputs with a random key, then compare HMACs. This is timing-safe.
- The ADMIN_API_KEY should be generated as 32+ bytes of random data, hex-encoded.
- Founder action required: generate and set the secret for staging and production.
