# Story E07S01: PostHog Reverse Proxy CF Worker

## User Story

**As a** platform operator,
**I want** a Cloudflare Worker reverse proxy routing analytics events through my own domain,
**so that** ad blockers don't suppress event capture and tracking data is complete.

---

## Context

PostHog's JS SDK sends events to `us.i.posthog.com` or `eu.i.posthog.com`, which ad blockers routinely block. A reverse proxy on a first-party subdomain (e.g. `ph.callibrate.io`) routes the same requests through our infrastructure, bypassing blockers and increasing event capture by 10-30%. PostHog documents this pattern officially for Cloudflare Workers.

---

## Notes

PostHog reverse proxy docs: the Worker must rewrite the `Host` header and forward the original request body unchanged. The subdomain must NOT contain "analytics", "tracking", or "posthog" in the name to avoid heuristic ad-blocker rules — `ph` is acceptable.
