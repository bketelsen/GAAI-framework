# Story E03S02: AI-Extracted Requirements Confirmation

## User Story

**As a** prospect who has described my project,
**I want** to review and confirm the AI-extracted requirements before matching begins,
**so that** I can correct any misunderstandings and get the most accurate expert matches.

---

## Context

Step 2 of the prospect funnel. After E03S01 extraction, the prospect sees a structured summary of what the AI understood. High-confidence fields are displayed as confirmed. Low-confidence fields show targeted questions (max 2–3, from `confirmation_questions` in extraction response). If all fields are high-confidence (`ready_to_match: true`), this step is streamlined — prospect reviews and confirms with one click.

On confirmation, the satellite frontend calls `POST /api/prospects/submit` (Core API, E06S07) which creates the prospect record, runs synchronous matching (DEC-40), and returns a JWT token. A Turnstile challenge protects this endpoint (E06S20). The page then transitions to match results (E03S03).

---

## Acceptance Criteria

- [ ] AC1: After extraction (E03S01), the page renders a "Vos besoins identifiés" section showing all extracted fields as a structured summary. Each field displays: label (e.g., "Compétences recherchées"), extracted value, and a visual confidence indicator (high ≥0.7 = green checkmark, medium 0.4–0.7 = orange, low <0.4 = red question mark). Fields: challenge, skills_needed, industry, budget_range, timeline, company_size, languages.

- [ ] AC2: When `ready_to_match: true` (all fields high-confidence): display the summary with a single "Confirmer et trouver mes experts" button. No editable fields required. Streamlined path — prospect reviews and confirms.

- [ ] AC3: When `needs_confirmation` is non-empty: for each low-confidence field, render the corresponding `confirmation_question` from the extraction response. Question types: (a) text input if `options` is null, (b) radio/select if `options[]` is provided, (c) budget range = dual number inputs with EUR symbol. Maximum 3 questions displayed.

- [ ] AC4: Prospect can edit ANY field (not just low-confidence ones) by clicking a "Modifier" affordance next to the field. Clicking transforms the display value into an editable input. This ensures prospects can correct high-confidence fields that are wrong.

- [ ] AC5: On "Confirmer" action:
  1. Merge confirmed/edited fields into the `requirements` JSONB
  2. Render the Cloudflare Turnstile widget (`appearance: 'interaction-only'` — invisible unless challenge needed)
  3. On Turnstile success: call `POST {CORE_API_URL}/api/prospects/submit` with `{ satellite_id, quiz_answers: {merged requirements}, utm_source, utm_campaign, utm_medium, utm_content, cf_turnstile_response }` via fetch()
  4. The submit endpoint creates the prospect, runs matching synchronously (DEC-40), and returns `{ prospect_id, token, token_expires_at }`

- [ ] AC6: During submit (API call): "Confirmer" button shows spinner + disabled, form fields become readonly, "Recherche d'experts en cours..." message. No page reload.

- [ ] AC7: On successful submit: store `{ prospect_id, token }` in client-side state. Transition to match results (E03S03) without server roundtrip.

- [ ] AC8: On submit error:
  - `422` (validation): display field-level errors from response
  - `429` (rate limited): "Trop de tentatives. Veuillez patienter."
  - Turnstile failure: re-render widget for retry
  - Other errors: "Une erreur est survenue. Veuillez réessayer." + retry button
  - No data lost — form remains editable

- [ ] AC9: PostHog events:
  - `satellite.confirmation_viewed` (satellite_id, ready_to_match, fields_needing_confirmation)
  - `satellite.confirmation_field_edited` (satellite_id, field_name) — when prospect edits any field
  - `satellite.confirmation_submitted` (satellite_id, fields_edited_count, turnstile_passed)
  - `satellite.prospect_created` (satellite_id, prospect_id) — on successful submit

- [ ] AC10: "Retour" link/button navigates back to E03S01 with the original freetext preserved (sessionStorage or history state). Does not create a new prospect or call any API.

- [ ] AC11: Turnstile site key is injected by the satellite Worker at render time (from environment variable `TURNSTILE_SITE_KEY`). The Turnstile JS SDK (`https://challenges.cloudflare.com/turnstile/v0/api.js`) is loaded asynchronously.

## Notes

- **API integration (Delivery to assess):** E06S07 AC3 accepts `quiz_answers` and normalizes to `requirements`. The freetext extraction (E06S08) already produces `requirements` JSONB. Delivery must verify that `POST /api/prospects/submit` can accept pre-extracted requirements directly via `quiz_answers`. If the normalization logic rejects the extracted format, a minor Core API enhancement is needed.
- UTM parameters should be preserved from the initial landing page URL (captured via URL params in E03S01, passed through client-side state to submit).
- The Turnstile widget uses `appearance: 'interaction-only'` to minimize friction for legitimate prospects — challenge only appears when Turnstile detects suspicious behavior.
