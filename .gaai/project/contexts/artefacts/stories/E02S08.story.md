# Story E02S08: Analytics Page

## User Story

**As a** registered expert,
**I want** to see my composite score, score breakdown, and performance trends,
**so that** I can understand my standing on the platform and identify areas for improvement.

---

## Context

The analytics page surfaces the expert's composite score (E06S09) — the quality signal that influences their ranking in matching results. Transparency about the scoring system is a core platform value: experts should understand exactly what drives their score and how to improve it.

Data comes from `GET /api/experts/:id/dashboard` (E06S38) for aggregate stats and the expert profile for composite_score breakdown.

**Composite score components (E06S09):**
- call_experience (weight 0.35) — average post-call rating from prospects
- trust (weight 0.20) — profile completeness + GCal connection + activity signals
- satisfaction (weight 0.20) — prospect satisfaction surveys (J+7)
- hire_rate (weight 0.10) — conversion from lead to actual project
- recency (weight 0.15) — activity freshness decay

**Quality tiers:** New (no data), Rising (0–50), Established (50–75), Top (75–100).

## Acceptance Criteria

### Composite Score Display

- [ ] AC1: Analytics route (`/dashboard/analytics`): loader fetches `GET /api/experts/:id/dashboard` (aggregate stats) + expert profile (composite_score, quality_tier).

- [ ] AC2: Composite score hero card: large score number (0–100), quality tier badge with color (New=gray, Rising=blue, Established=green, Top=gold), tier label ("Expert Top" / "Expert Confirmé" / "Expert Prometteur" / "Nouvel Expert").

- [ ] AC3: Score breakdown visualization: radar chart (recharts RadarChart) with 5 axes: Expérience d'appel (call_experience), Confiance (trust), Satisfaction (satisfaction), Taux de conversion (hire_rate), Activité récente (recency). Each axis 0–100 scale. Current values plotted. Weight percentages shown in legend (35%, 20%, 20%, 10%, 15%).

- [ ] AC4: Per-component detail cards below radar chart: one card per scoring dimension showing:
  - Component name + weight
  - Current value (0–100)
  - What drives it (brief explanation)
  - How to improve it (one actionable tip)
  Example: "Expérience d'appel (35%) — 72/100 — Basé sur les évaluations post-appel de vos prospects. Conseil : préparez chaque appel en consultant la page de préparation."

### Monthly Metrics

- [ ] AC5: Monthly leads chart (recharts BarChart): stacked bars per month showing leads_received (total), leads_confirmed (green), leads_flagged (red). Last 6 months. X-axis: month labels. Y-axis: count. If less than 6 months of data, show available months only.

- [ ] AC6: Conversion metrics cards (horizontal layout):
  - Leads → Réservations: "{bookings/leads}%" with trend arrow (vs previous month)
  - Réservations → Projets: "{conversions/bookings}%" with trend arrow
  - Taux de confirmation: "{confirmed/total_leads}%" (qualification_rate)
  - Leads ce mois: "{month_stats.leads_received}" count

### Vue d'ensemble (Dashboard Index)

- [ ] AC7: Dashboard index route (`/dashboard`) — replace E02S01 placeholder. Loader fetches `GET /api/experts/:id/dashboard` (E06S38 AC13–AC15b). Displays summary cards: unread leads count (link to `/dashboard/leads?status=new`), upcoming bookings count (link to `/dashboard/bookings`), credit balance (link to `/dashboard/billing`), composite score with tier badge (link to `/dashboard/analytics`). Quick-glance KPIs — expert sees their status in < 2 seconds.

- [ ] AC8: Vue d'ensemble monthly summary: leads this month (received/confirmed/flagged) as mini bar or counts. This month vs previous month trend arrow per metric. Data from `month_stats` + `monthly_history[0]` vs `monthly_history[1]`.

### Empty State

- [ ] AC9: When expert is new (no composite_score, no leads): "Vos statistiques apparaîtront ici après votre premier lead" + explanation of what the composite score measures + CTA to complete profile → `/dashboard/settings`.

- [ ] AC10: When expert has some data but not all components: partial display — show available components, gray out unavailable ones with "Données insuffisantes" label. Radar chart renders with available data only.

## Notes

- **recharts** is the recommended charting library: lightweight, React-native, SSR-compatible. The radar chart and bar chart are standard recharts components. Install as a dependency in the dashboard workspace.
- **Score transparency framing:** The analytics page should educate, not intimidate. Each scoring component includes a "how to improve" tip. The overall message: "Your score reflects the quality of your platform interactions — better preparation and honest evaluations lead to better leads."
- **qualification_rate visibility (DEC-68):** The confirmation rate (confirmed/total_leads) is visible on this page with a positive framing: "Un taux de confirmation élevé indique que vos critères sont bien calibrés." Never frame it as a penalty metric.
- **Radar chart SSR:** recharts uses SVG — it renders server-side without issues. No special client-only handling needed.
- **PostHog events (Delivery must include):** `expert.analytics_viewed` (page load), `expert.dashboard_viewed` (vue d'ensemble page load). Server-side capture in loaders.
