# Story: Matching-engine CI/CD pipeline in deploy-staging.yml

## User Story

**As a** developer modifying the matching-engine,
**I want** it to be automatically deployed to staging with pre-deploy validation,
**so that** matching-engine changes are not forgotten or deployed manually.

---

## Context

`callibrate-matching` is a standalone Cloudflare Worker with 10 source files (score, pricing, expert pool, vectorize, D1, index). It is consumed by `callibrate-core` via Service Binding. Currently:
- No CI/CD workflow deploys it — deployment is manual only
- No test files exist (addressed by E09S03)
- No type-check in staging CI
- `deploy-production.yml` also does not deploy it

This creates a class of failures where core is deployed with updated Service Binding expectations, but the matching-engine is stale.

---

## Acceptance Criteria

- [ ] AC1: `deploy-staging.yml` includes a new `deploy-matching` job that deploys `callibrate-matching` to staging via `wrangler deploy --env staging` in `workers/backend/matching-engine`
- [ ] AC2: The `deploy-matching` job runs `tsc --noEmit` before deployment
- [ ] AC3: The `deploy-matching` job runs `vitest run` before deployment (requires E09S03 to have tests — if no tests yet, the step should be present but the job should not fail on empty test suite)
- [ ] AC4: `deploy-production.yml` includes an equivalent `deploy-matching` job for production (`wrangler deploy --env production`)
- [ ] AC5: The `deploy-core` job in both workflows declares `needs: [deploy-matching]` so that the matching-engine is deployed before core (core depends on it via Service Binding)
- [ ] AC6: The matching-engine wrangler.toml has valid `[env.staging]` and `[env.production]` configurations (verify they exist — if not, story is blocked and must escalate)

> All acceptance criteria must be verifiable. Vague criteria are invalid.

---

## Out of Scope

- Writing matching-engine tests (covered by E09S03)
- Smoke testing matching-engine endpoints (covered by E09S04)
- Changes to matching-engine application code

---

## Dependencies

- E09S03 (for vitest step to be meaningful — but AC3 handles the case where tests don't exist yet)

---

## Notes

- The `deploy-core` → `needs: [deploy-matching]` dependency in AC5 ensures the Service Binding target exists before core deploys. Without this, core might deploy against a stale matching-engine.
- Check `workers/backend/matching-engine/wrangler.toml` for existing env configs before implementing.
- The matching-engine worker names in wrangler.toml must match the Service Binding names in core's wrangler.toml (`callibrate-matching-staging` / `callibrate-matching-prod`).
