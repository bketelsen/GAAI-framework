# Story: Security headers middleware + XSS fix

## User Story

**As a** user of Callibrate (expert or prospect),
**I want** all HTTP responses to include standard security headers and all HTML output to be XSS-safe,
**so that** my browser enforces content security policies and I'm protected against injection attacks.

---

## Context

SEC-AUDIT-001 findings SEC-04 (HIGH: JSON-LD XSS) and SEC-05 (HIGH: missing security headers) and SEC-17 (LOW: Content-Type validation). No Worker currently returns security headers. The satellite landing page has a stored XSS vector via `structured_data` JSON-LD injection.

---

## Acceptance Criteria

- [ ] AC1: A `securityHeaders` middleware function is created (shared utility) that adds to every Response:
  - `X-Content-Type-Options: nosniff`
  - `X-Frame-Options: DENY`
  - `Strict-Transport-Security: max-age=31536000; includeSubDomains`
  - `Referrer-Policy: strict-origin-when-cross-origin`
  - `Permissions-Policy: camera=(), microphone=(), geolocation=()`
- [ ] AC2: The middleware is applied in the Core API worker (`index.ts`) as a wrapper on all responses (after routing, before return).
- [ ] AC3: The middleware is applied in the Satellite worker on all HTML responses. The Satellite additionally returns `Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline' https://ph.callibrate.io; connect-src 'self' https://ph.callibrate.io; img-src 'self' data: https:; style-src 'self' 'unsafe-inline'`.
- [ ] AC4: The middleware is applied in the PostHog Proxy worker on all responses.
- [ ] AC5: In `landing.ts`, the JSON-LD output escapes `</` to prevent script breakout:
  ```typescript
  JSON.stringify(config.structured_data).replace(/</g, '\\u003c')
  ```
- [ ] AC6: All POST endpoints in the Core API reject requests where `Content-Type` is not `application/json` (return 415 Unsupported Media Type). Exception: webhook endpoints that may receive other content types.
- [ ] AC7: All existing vitest tests pass. New tests: (a) security headers present on API response, (b) JSON-LD XSS payload `</script><img onerror=...>` is escaped, (c) POST without Content-Type application/json returns 415.
- [ ] AC8: tsc clean across all 4 workers.

---

## Out of Scope

- CSP report-uri / report-to configuration
- Subresource Integrity (SRI) for external scripts
- Feature-Policy fine-tuning post-MVP

---

## Dependencies

- None — standalone fix across workers

---

## Notes

- `unsafe-inline` is required for PostHog snippet in satellite (inline script). Can be tightened post-MVP with nonce-based CSP.
- The Core API returns only JSON — CSP is less critical there, but `nosniff` and `X-Frame-Options` are important.
- Do NOT add CSP to the PostHog proxy — it proxies third-party content.
