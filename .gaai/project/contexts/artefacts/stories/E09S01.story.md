# Story: Pre-deploy test gates in deploy-staging.yml

## User Story

**As a** developer merging to staging,
**I want** type-check and vitest to run before deployment,
**so that** code regressions are caught before they reach the live staging environment.

---

## Context

`deploy-staging.yml` currently deploys 3 workers (core, satellite, posthog-proxy) with zero pre-deploy validation. By contrast, `preview-deploy.yml` and `deploy-production.yml` already run `tsc --noEmit` + `vitest run` for core. Dashboard and landing have their own workflows with test gates, but core/satellite/posthog-proxy on staging have none.

This is the single highest-impact change: adding 4 lines per job catches the majority of regressions that currently slip to staging.

---

## Acceptance Criteria

- [ ] AC1: `deploy-core` job in `deploy-staging.yml` runs `tsc --noEmit` and `vitest run` in `workers/backend/api` before the `wrangler deploy` step
- [ ] AC2: `deploy-satellite` job runs `tsc --noEmit` and `vitest run` in `workers/frontend/satellites` before the `wrangler deploy` step
- [ ] AC3: `deploy-posthog-proxy` job runs `tsc --noEmit` and `vitest run` in `workers/backend/posthog-proxy` before the `wrangler deploy` step
- [ ] AC4: If type-check or tests fail, the deploy step does NOT execute (job fails early)
- [ ] AC5: No changes to `deploy-production.yml`, `preview-deploy.yml`, `deploy-dashboard.yml`, or `deploy-landing.yml`

> All acceptance criteria must be verifiable. Vague criteria are invalid.

---

## Out of Scope

- Adding new test files (covered by E09S03)
- Post-deploy smoke testing (covered by E09S04)
- Matching-engine pipeline (covered by E09S02)

---

## Dependencies

- None. Workers already have vitest configured and test files.

---

## Notes

- Satellite and posthog-proxy have minimal test suites (1-5 test files). That's fine — even minimal tests catch import errors, type regressions, and compilation failures.
- If posthog-proxy or satellite have no `vitest` in devDependencies, the Delivery Agent should add it and create a minimal vitest config as part of this story.
- Estimated CI time increase: ~30-60s per job (type-check + vitest). Acceptable trade-off.
