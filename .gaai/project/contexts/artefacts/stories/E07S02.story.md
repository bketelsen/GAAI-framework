# Story E07S02: Satellite Client-Side Instrumentation (Prospect UX Tracking)

## User Story

**As a** founder optimizing prospect conversion,
**I want** satellite landing pages to capture page views, CTA clicks, and session recordings,
**so that** I can see where prospects drop off and debug UX issues visually.

---

## Context

Satellite sites are SSR HTML pages rendered by the `callibrate-satellite` CF Worker (`workers/satellite/src/pages/landing.ts`). There is currently zero client-side analytics. The HTML is pure template strings — no framework, no hydration. PostHog JS SDK must be injected as a `<script>` tag in the rendered HTML, configured to use the reverse proxy (E07S01) and cookieless mode for GDPR compliance.

---

## Acceptance Criteria

- [ ] AC1: PostHog JS SDK loaded via `<script>` tag in satellite HTML `<head>` (defer, non-blocking)
- [ ] AC2: SDK configured to use reverse proxy host (`ph.callibrate.io`) instead of PostHog direct
- [ ] AC3: Cookieless mode enabled (`persistence: "memory"`, no cookies set)
- [ ] AC4: Autocapture enabled (clicks, form submissions, page views captured automatically)
- [ ] AC5: Session replay enabled (records prospect browsing behavior on satellite)
- [ ] AC6: Custom event `satellite.cta_clicked` fired on CTA button click with properties: `{ satellite_id, cta_text }`
- [ ] AC7: Page view event includes properties: `{ satellite_id, referrer, utm_source, utm_campaign, utm_medium }` (parsed from URL query params)
- [ ] AC8: PostHog Project API Key injected via satellite config (environment variable `POSTHOG_API_KEY`), not hardcoded
- [ ] AC9: Tracking can be disabled per satellite via `satellite_configs.tracking_enabled` boolean (default: true)
- [ ] AC10: All existing satellite tests pass
- [ ] AC11: No impact on Largest Contentful Paint (script is deferred, async init)

## Notes

UTM parameters should be parsed from `window.location.search` client-side and passed as PostHog event properties. The satellite Worker already receives `utm_source` and `utm_campaign` in prospect form submissions — client-side tracking adds the pre-form attribution (which page view led to the form).

PostHog session replay captures DOM snapshots — ensure `.ph-no-capture` CSS class is applied to any future form fields containing PII (email input, phone). Currently the satellite landing page has no PII fields (the form is on core API).

### Automatic Friction Detection (zero config)

With autocapture enabled, PostHog automatically captures:
- **Rage clicks** — user clicks same element 3+ times in 1 second (frustration signal)
- **Dead clicks** — click produces no page change (broken/misleading UI element)

These are immediately available in session replay filters and heatmap overlays. No custom events needed — just enable autocapture (AC4).

### Heatmaps

PostHog toolbar provides three heatmap types on satellite pages:
- **Clickmap** — exact click counts per element
- **Scrollmap** — how far prospects scroll (content visibility cutoff)
- **Rageclicks overlay** — friction hotspots

These activate automatically once the JS SDK is loaded — no additional code.
