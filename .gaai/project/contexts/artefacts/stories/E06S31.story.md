# Story E06S31: Credit Billing Schema + Lead Pricing Function

## User Story

**As a** platform operator,
**I want** the database schema to support credit-based billing with a transaction audit trail and dynamic lead pricing,
**so that** the billing model (DEC-68) and pricing grid (DEC-67) have a solid foundation for all downstream billing stories.

---

## Context

DEC-68 replaces the per-lead LS checkout model with internal credit accounting. DEC-67 defines dynamic pricing based on prospect budget tier x qualification level. This story adds the necessary DB columns, creates the `credit_transactions` audit table, extends `leads` status tracking, and implements the pricing pure function.

Current state: `experts` table has no billing columns. `leads` table has `amount` (unpopulated), `ls_checkout_id`, `status` CHECK limited to ('pending', 'billed', 'disputed', 'refunded'). No `credit_transactions` table exists.
