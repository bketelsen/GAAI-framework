# Story E03S03: Match Results + Email Gate

## User Story

**As a** prospect who has submitted my requirements,
**I want** to see which experts match my needs with transparent scoring,
**so that** I can evaluate the quality of matches before sharing my contact information.

---

## Context

Step 3 of the prospect funnel. After prospect submission (E03S02), the satellite frontend calls `GET /api/prospects/:id/matches?token=xxx` (E06S07) to retrieve anonymized expert matches ranked by score with per-criterion breakdown.

The prospect sees anonymized expert cards — enough information to judge match quality (scores, skills, rate range, quality tier) but no identity (no name, photo, bio). To unlock full profiles, the prospect provides their email via `POST /api/prospects/:id/identify` (E06S07) — this is the email gate. After identification, full profiles are revealed with booking CTAs.

Email gate rationale (DEC confirmed 2026-02-27): protects against disintermediation (prospects contacting experts off-platform), aligns with credit billing model (identify = lead creation = billing event), and ensures expert fairness (profiles only shown to committed prospects).

## Acceptance Criteria

- [ ] AC1: After successful submit (E03S02), the page calls `GET {CORE_API_URL}/api/prospects/{prospect_id}/matches?token={token}` and renders the match results. Loading state: skeleton cards with shimmer effect while API responds.

- [ ] AC2: If API returns `202 { status: "computing" }` (edge case — E06S07 AC6): display "Calcul des correspondances en cours..." with a retry after `estimated_seconds`. Auto-retry up to 3 times with exponential backoff. On persistent 202 or empty results: "Aucun expert disponible pour le moment. Nous vous contacterons quand un expert correspondra à vos besoins."

- [ ] AC3: Anonymized expert cards display (per E06S07 AC5):
  - Rank badge (#1, #2, #3 visually prominent — larger cards, accent border, gold/silver/bronze tint per DEC-25)
  - Overall match score (0–100, displayed as percentage bar or score badge)
  - Per-criterion score breakdown as labeled horizontal bars (skills, industry, budget, timeline, languages)
  - Quality tier badge: "Top Expert" / "Expert Confirmé" / "Expert Prometteur" (mapped from composite_score tier)
  - Skills tags (from matched skills_needed)
  - Rate range: "€{rate_min} — €{rate_max} / heure"
  - Industries and project types as tags
  - Languages as tags
  - Placeholder avatar (generic silhouette — identical for all cards)

- [ ] AC4: Results header: "{N} experts correspondent à votre projet" above the card list. Top 3 matches are visually prominent (DEC-25). All remaining matches displayed below in standard-sized cards. No artificial limit on total matches shown.

- [ ] AC5: Email gate section below the match cards: headline "Découvrez le profil complet de vos experts", supporting text explaining value ("Accédez aux noms, parcours et disponibilités de vos experts"), email input field (single field — email only), "Débloquer les profils" button. Inline email format validation.

- [ ] AC6: On email submit: call `POST {CORE_API_URL}/api/prospects/{prospect_id}/identify` with `{ email, token }`. Loading state on button. On success: response contains full expert profiles with `{ display_name, headline, bio, skills, industries, rate_min, rate_max, score, booking_url }`.

- [ ] AC7: After identify success: anonymized cards are replaced by full expert cards showing:
  - Expert name + initial-based avatar (colored by quality tier)
  - Headline (one-line professional summary)
  - Bio (first 2 lines visible, expandable on click)
  - Full score breakdown (same as anonymous but with identity context)
  - "Réserver un appel" CTA button per expert → opens E03S06 booking widget inline with that expert's ID
  - Rate range and skills remain visible

- [ ] AC8: On identify error:
  - `409` (already identified): fetch profiles directly — use the response to reveal full cards
  - `403` (invalid/expired token): "Votre session a expiré. Veuillez recommencer." + link back to `/match`
  - `422` (invalid email format): inline field error
  - Other errors: "Une erreur est survenue." + retry button

- [ ] AC9: "No matches" state: if matches array is empty, display "Aucun expert ne correspond exactement à vos critères pour le moment." + two CTAs: "Modifier mes critères" → back to E03S02, "Parcourir le répertoire" → `/experts` (E03S04).

- [ ] AC10: PostHog events:
  - `satellite.matches_viewed` (satellite_id, prospect_id, match_count, top_score)
  - `satellite.match_card_expanded` (satellite_id, expert_rank) — when clicking to see full score details
  - `satellite.email_gate_submitted` (satellite_id, prospect_id)
  - `satellite.profiles_unlocked` (satellite_id, prospect_id, match_count)
  - `satellite.booking_cta_clicked` (satellite_id, prospect_id, expert_rank)

- [ ] AC11: Page URL changes to `/match/results` (or `/match?step=results`) for PostHog page tracking and browser history. Browser back → E03S02 (confirmation step), not E03S01.

- [ ] AC12: `<meta name="robots" content="noindex, nofollow">` — match results are prospect-specific and must not be indexed by search engines.

## Notes

- **Top-3 visual hierarchy (DEC-25):** top 3 matches are visually prominent. All others displayed in standard cards. No artificial limit on total count.
- **Score transparency** is the platform's core UX value proposition. Per-criterion breakdown should be immediately visible on each card, not hidden behind a tooltip or expand action.
- **Token TTL:** the JWT from E06S07 AC3 has 24h TTL. AC8 handles expiry gracefully.
- **booking_url field:** E06S07 AC7 returns `booking_url` but this references the deprecated Cal.com URL. E03S03 ignores `booking_url` and uses the E03S06 booking widget instead, which calls E06S11 APIs directly. The `expert_id` (or equivalent identifier) from the identify response is what the booking widget needs.
- **Email gate copy** is critical for conversion. The framing should emphasize value received ("See full profiles, read expert bios, book a call") not data extracted ("Enter your email"). A/B testable post-MVP via PostHog feature flags.
