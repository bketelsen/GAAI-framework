# Story E06S13: satellite_configs Schema Extension — Brand, Content & Theme Columns

## User Story

**As a** platform operator,
**I want** `satellite_configs` to carry the full visual identity and SEO content configuration for each satellite,
**so that** a single Worker can render any satellite's branded landing page by reading one DB row — without touching code.

---

## Context

The current `satellite_configs` schema (E06S02) holds `quiz_schema` and `matching_weights` — both Layer 2 concerns. The multi-tenant satellite Worker (E06S14) needs additional columns for: visual branding (design tokens), brand identity, SEO content (meta title, copy, value props), JSON-LD structured data, and an `active` flag to gate go-live.

This story is a **pure DB migration** — no application logic, no Worker changes. The satellite Worker (E06S14) depends on this schema being in place.

---

## Acceptance Criteria

- [ ] AC1: Supabase migration adds the following columns to `satellite_configs`:
  - `theme JSONB` — visual design tokens
  - `brand JSONB` — brand identity
  - `content JSONB` — SEO/page content (unique per satellite)
  - `structured_data JSONB` — JSON-LD schema markup
  - `active BOOLEAN NOT NULL DEFAULT false` — go-live gate; Worker returns 404 redirect for inactive satellites
  - `updated_at TIMESTAMPTZ DEFAULT now()` — set via trigger or application

- [ ] AC2: `theme` JSONB canonical shape (all fields optional with fallback defaults in the Worker):
  ```json
  {
    "primary": "#4F46E5",
    "accent": "#818CF8",
    "font": "Inter, sans-serif",
    "radius": "0.5rem",
    "logo_url": "https://..."
  }
  ```

- [ ] AC3: `brand` JSONB canonical shape:
  ```json
  {
    "name": "n8n Experts",
    "tagline": "L'expert n8n qu'un pair de confiance t'aurait recommandé."
  }
  ```

- [ ] AC4: `content` JSONB canonical shape (all string fields required when `active: true`):
  ```json
  {
    "meta_title": "Trouvez un expert n8n pré-qualifié | n8n Experts",
    "meta_description": "Décrivez votre projet d'automatisation n8n. Recevez des experts matchés à vos critères en quelques minutes.",
    "hero_headline": "L'expert n8n qu'un pair de confiance t'aurait recommandé.",
    "hero_sub": "Décris ton besoin. On te trouve l'expert qui correspond — budget, stade, secteur.",
    "value_props": [
      "Pré-qualifiés sur tes critères réels",
      "Call booké directement dans son agenda",
      "Résultats en moins de 2 minutes"
    ],
    "vertical_label": "experts n8n",
    "vertical_description": "Automatisez vos workflows métier avec un expert n8n pré-qualifié."
  }
  ```

- [ ] AC5: `structured_data` JSONB canonical shape (JSON-LD, rendered as `<script type="application/ld+json">` by the Worker):
  ```json
  {
    "@context": "https://schema.org",
    "@type": "ProfessionalService",
    "name": "n8n Experts",
    "description": "Plateforme de matching entre entreprises et experts n8n certifiés.",
    "url": "https://n8n-experts.io",
    "areaServed": "FR"
  }
  ```

- [ ] AC6: Existing `'default'` seed row updated with valid placeholder values for all new columns — `active: false`, theme using Callibrate brand colors (`primary: "#4F46E5"`), brand name `"Callibrate"`, coherent content and structured_data placeholders

- [ ] AC7: Migration applies cleanly on the existing staging database without data loss on existing columns (`id`, `label`, `domain`, `vertical`, `quiz_schema`, `matching_weights`, `created_at`)

- [ ] AC8: `supabase gen types typescript` re-run — `SatelliteConfig` TypeScript type updated to reflect new columns; generated file committed at `src/types/database.ts` in the `callibrate-core` repo

## Notes

- Columns are nullable at DB level (no `NOT NULL` except `active`) — the Worker applies fallback defaults when a field is missing, allowing partial configs during setup
- `updated_at` can be managed at application level (Worker sets it on updates) rather than via a DB trigger, to avoid migration complexity
- The `domain` column already exists (E06S02) — it is the hostname used by the satellite Worker for lookup (e.g. `"n8n-experts.io"`)
- Adding a new satellite = INSERT a new row with `active: false`, fill all fields, smoke test, then flip `active: true`
