# Story E07S05: PostHog MCP Integration + GAAI Analytics-Query Skill

## User Story

**As a** GAAI Discovery Agent analyzing growth opportunities,
**I want** direct access to PostHog data via MCP server and a governed analysis skill,
**so that** I can query funnels, diagnose drop-offs, and surface data-driven growth insights within the governed framework.

---

## Context

PostHog provides an official MCP server (`@posthog/mcp`) that exposes analytics queries, feature flags, experiments, and dashboards as MCP tools — directly usable in Claude Code sessions. This eliminates the need for a hand-rolled API wrapper.

The GAAI skill `analytics-query` complements the MCP server by providing:
- A governed analysis framework (what to query, when, in what sequence)
- Callibrate-specific query templates (prospect funnel, expert activation, business metrics)
- Interpretation guidelines (what signals to look for, what thresholds matter)
- The conversion optimization playbook as a repeatable diagnostic process

The MCP server executes. The skill governs what execution looks like and what to do with results.

## Notes

PostHog MCP server repo: https://github.com/PostHog/mcp

The skill is NOT a replacement for PostHog's built-in Max AI assistant (which operates inside the PostHog UI). The skill governs how GAAI agents use PostHog data within Discovery workflows — Max handles ad-hoc exploration in the PostHog dashboard.

PostHog API rate limits: 120 HogQL queries/hour, 2400 structured queries/hour. The skill documents these limits so agents plan query frequency appropriately.

### Conversion Optimization Playbook (embedded in skill)

```
Phase 1: Instrument (E07S01-S03)
Phase 2: Baseline — measure conversion at each step, set alerts for drops
Phase 3: Diagnose — funnel drop-off → session replay → correlation → rage clicks → path analysis
Phase 4: Hypothesize & Test — feature flag A/B test → measure funnel impact
Phase 5: Compound — ship winner, move to next drop-off, repeat
```

This loop is the core operating rhythm the skill encodes as repeatable procedures.
