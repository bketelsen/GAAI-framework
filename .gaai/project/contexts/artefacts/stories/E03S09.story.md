# Story E03S09: Freetext Input UX — Persistent Hint Text

## User Story

**As a** prospect arriving at the freetext input page,
**I want** always-visible guidance on what information to include in my description,
**so that** I provide enough detail for the matching engine to find the right expert without the guidance disappearing when I start typing.

---

## Context

DEC-119 identified that placeholder text (which disappears on first keystroke) is insufficient for guiding prospects. The freetext textarea (E03S01) needs persistent hints that remain visible throughout typing. The chosen pattern is industry-standard: label → hint text → textarea (Material Design, GOV.UK Design System).

This is a small UX improvement that can be delivered standalone or bundled with E03S08 (funnel restructure).

---

## Acceptance Criteria

- [ ] AC1: On the freetext input page (E03S01, `/match` route), replace any existing placeholder text with a persistent hint text block between the label and the textarea:
  - **Label:** "Decrivez votre projet"
  - **Hint text:** "Quelques pistes pour nous aider : votre probleme ou objectif · les outils concernes · votre budget approximatif · vos delais"
  - **Textarea:** Empty, no placeholder attribute.
  - Hint text is styled as muted/secondary text (smaller font, lighter color), clearly distinct from the label. Always visible regardless of textarea content.

- [ ] AC2: Hint text language matches the satellite's configured language (`config.locale` or `config.language`). If satellite locale is French → French hint text. If English → English hint text ("A few pointers: your problem or goal · tools involved · approximate budget · timeline"). If locale not available or not supported → English default.

- [ ] AC3: Conversational tone — "Quelques pistes pour nous aider" not "Vous devez indiquer". No imperative form. No bullet list — use middle dots (·) as inline separators for a lightweight feel.

- [ ] AC4: **Client-side soft warning:** If textarea content is less than 50 characters when the prospect pauses typing (debounce 1.5s), display a subtle inline message below the textarea: "Quelques mots de plus aideront a trouver le bon expert." Style: amber/warning color, small font. Not blocking — prospect can still submit with < 50 chars (the server-side validation in E06S40 AC2 handles minimum thresholds).

- [ ] AC5: Hint text is semantic HTML: `<p id="freetext-hint" class="...">` with `aria-describedby="freetext-hint"` on the textarea. Screen readers announce the hint when the textarea is focused.

- [ ] AC6: No visual regression on mobile. Hint text wraps naturally on narrow viewports. Textarea minimum height unchanged.

---

## Out of Scope

- Dynamic hints that change based on partial input (over-engineered for MVP)
- Clickable suggestion chips or templates
- Multi-language detection from browser (satellite config determines language, not browser locale)

---

## Dependencies

- E03S01 (freetext input page — the page being modified)

---

## Notes

- **Guard-rail philosophy (DEC-119):** Don't overcompensate at step 1 for what step 2 handles natively. The E03S02 confirmation questions (max 3) are designed to handle low-quality input. The hint text is a gentle nudge, not a hard gate.
- **Can be bundled:** This change is small enough to be included in E03S08 (funnel restructure) without a separate deployment.
