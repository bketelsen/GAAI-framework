# Story E06S02: Database Schema & Migrations

## User Story

**As a** developer building any feature on Callibrate,
**I want** all core tables, indexes, and RLS policies in place,
**so that** I can store and query data without schema rework.

---

## Context

Defines the complete data model for the MVP. Uses PostgreSQL JSONB + GIN indexes for matching criteria — flexible by design so that criteria can be added/modified without migrations. All tables created via Supabase migration files (version-controlled). TypeScript types generated from schema for type safety in Workers.

---

## Acceptance Criteria

- [ ] AC1: Supabase migration files created and applied cleanly on a fresh project for all 9 tables: `experts`, `prospects`, `matches`, `bookings`, `leads`, `satellite_configs`, `call_experience_surveys`, `project_satisfaction_surveys`, `lead_evaluations`
- [ ] AC2: `experts` table contains: `id UUID PK (FK auth.users)`, `display_name TEXT`, `headline TEXT`, `bio TEXT`, `avatar_url TEXT`, `rate_min INTEGER`, `rate_max INTEGER`, `availability TEXT CHECK IN ('available', 'limited', 'unavailable')`, `cal_username TEXT`, `profile JSONB`, `preferences JSONB`, `verified_at TIMESTAMPTZ`, `composite_score NUMERIC(5,2) DEFAULT 0`, `score_updated_at TIMESTAMPTZ`, `created_at TIMESTAMPTZ DEFAULT now()`
- [ ] AC3: `prospects` table contains: `id UUID PK DEFAULT gen_random_uuid()`, `satellite_id TEXT`, `email TEXT`, `utm_source TEXT`, `utm_campaign TEXT`, `quiz_answers JSONB`, `requirements JSONB`, `status TEXT CHECK IN ('anonymous', 'identified', 'matched', 'booked')`, `created_at TIMESTAMPTZ DEFAULT now()`
- [ ] AC4: `matches` table contains: `id UUID PK`, `expert_id UUID FK experts`, `prospect_id UUID FK prospects`, `score NUMERIC(5,2)`, `score_breakdown JSONB`, `status TEXT CHECK IN ('pending', 'revealed', 'booked', 'expired')`, `expires_at TIMESTAMPTZ`, `created_at TIMESTAMPTZ DEFAULT now()`
- [ ] AC5: `bookings` table contains: `id UUID PK`, `match_id UUID FK matches`, `cal_booking_id TEXT UNIQUE`, `cal_meeting_url TEXT`, `scheduled_at TIMESTAMPTZ`, `duration_min INTEGER DEFAULT 20`, `status TEXT CHECK IN ('confirmed', 'cancelled', 'completed', 'no_show')`, `created_at TIMESTAMPTZ DEFAULT now()`
- [ ] AC6: `leads` table contains: `id UUID PK`, `booking_id UUID FK bookings`, `expert_id UUID FK experts`, `prospect_id UUID FK prospects`, `amount INTEGER`, `ls_checkout_id TEXT`, `ls_order_id TEXT`, `status TEXT CHECK IN ('pending', 'billed', 'disputed', 'refunded')`, `billed_at TIMESTAMPTZ`, `created_at TIMESTAMPTZ DEFAULT now()`
- [ ] AC7: `satellite_configs` table contains: `id TEXT PK`, `label TEXT`, `domain TEXT`, `vertical TEXT`, `quiz_schema JSONB NOT NULL`, `matching_weights JSONB NOT NULL`, `created_at TIMESTAMPTZ DEFAULT now()`
- [ ] AC8: GIN indexes created: `experts(profile)`, `experts(preferences)`, `prospects(requirements)`, `matches(score_breakdown)`
- [ ] AC9: RLS policies: authenticated expert can SELECT/UPDATE only their own row in `experts`; service role bypasses all RLS; `prospects`, `matches`, `bookings`, `leads` accessible by service role only
- [ ] AC10: `supabase gen types typescript` produces valid types file committed to repo at `src/types/database.ts`
- [ ] AC11: One `satellite_configs` seed row inserted for `id: 'default'` with a valid `quiz_schema` and `matching_weights` (used for local dev and testing)
- [ ] AC12: `call_experience_surveys` table contains: `id UUID PK DEFAULT gen_random_uuid()`, `booking_id UUID FK bookings`, `prospect_id UUID FK prospects`, `score INTEGER CHECK (score BETWEEN 1 AND 5)`, `submitted_at TIMESTAMPTZ DEFAULT now()`
- [ ] AC13: `project_satisfaction_surveys` table contains: `id UUID PK DEFAULT gen_random_uuid()`, `booking_id UUID FK bookings`, `prospect_id UUID FK prospects`, `score INTEGER CHECK (score BETWEEN 1 AND 10)`, `submitted_at TIMESTAMPTZ DEFAULT now()`
- [ ] AC14: `lead_evaluations` table contains: `id UUID PK DEFAULT gen_random_uuid()`, `lead_id UUID FK leads`, `expert_id UUID FK experts`, `lead_quality_score INTEGER CHECK (lead_quality_score BETWEEN 1 AND 10)`, `conversion_declared BOOLEAN DEFAULT false`, `conversion_declared_at TIMESTAMPTZ`, `notes TEXT`, `submitted_at TIMESTAMPTZ DEFAULT now()`

## Notes

`profile` JSONB shape: `{ skills: string[], industries: string[], project_types: string[], languages: string[], tools: string[] }`. `preferences` JSONB shape: `{ min_budget: number, project_types: string[], excluded_industries: string[], company_size: string[], timeline: string[] }`. `requirements` JSONB shape: `{ challenge: string, skills_needed: string[], industry: string, budget_range: { min: number, max: number }, timeline: string, company_size: string, languages: string[] }`. These shapes are documented in code via TypeScript interfaces — not enforced at DB level (flexibility by design).
