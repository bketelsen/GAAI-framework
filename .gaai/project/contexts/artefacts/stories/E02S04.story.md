# Story E02S04: Google Calendar Connection Page

## User Story

**As a** registered expert,
**I want** to connect and manage my Google Calendar integration from my dashboard,
**so that** the platform can show my availability to prospects and book calls directly on my calendar.

---

## Context

Google Calendar integration is how Callibrate provides real-time availability and instant booking (E06S11). The expert connects their GCal via OAuth (E06S10), and the booking engine reads freebusy data + creates events with Google Meet links.

This page is accessible from the onboarding wizard (E02S02 Step 4) and from the profile settings page (E02S03 AC2 section 5). It manages the full GCal connection lifecycle: connect, view status, disconnect.

---

## Acceptance Criteria

### Connection Status

- [ ] AC1: GCal settings section (part of `/dashboard/settings` or dedicated sub-route `/dashboard/settings/calendar`). Loader fetches connection status via `GET /api/experts/:id/gcal/status` (E06S10). Displays: connected email address or "Non connecté".

- [ ] AC2: When connected: green status badge "Connecté", connected email displayed, "Déconnecter" button with confirmation dialog.

- [ ] AC3: When not connected: yellow status badge "Non connecté", "Connecter Google Calendar" primary CTA button, brief explanation text: "Connectez votre calendrier Google pour afficher vos disponibilités et recevoir des réservations automatiques."

### Connect Flow

- [ ] AC4: "Connecter Google Calendar" button: calls `GET /api/experts/:id/gcal/auth-url` via loader or client-side fetch → receives OAuth URL → redirects to Google consent screen (or opens in popup). After OAuth callback, GCal callback endpoint (E06S10) processes the code and redirects back to the dashboard settings page.

- [ ] AC5: After successful connection (detected by loader on return): status updates to "Connecté" with the connected email. Success toast "Google Calendar connecté avec succès".

### Disconnect Flow

- [ ] AC6: "Déconnecter" button opens a confirmation dialog: "Êtes-vous sûr de vouloir déconnecter votre Google Calendar ? Les prospects ne pourront plus réserver de créneaux sur votre calendrier." Confirm → calls `POST /api/experts/:id/gcal/disconnect` (E06S10) via action. On success: status updates to "Non connecté", toast "Google Calendar déconnecté".

### Availability Preview

- [ ] AC7: When connected: display next 5 available time slots (fetched via `GET /api/experts/:id/bookings/availability` from E06S11, next 7 days). Shows date + time range for each slot. "Actualisé à {time}" timestamp. Refresh button to re-fetch.

- [ ] AC8: If no available slots in next 7 days: "Aucun créneau disponible dans les 7 prochains jours. Vérifiez votre agenda Google."

### Quality

- [ ] AC9: `tsc --noEmit` — zero errors.

- [ ] AC10: Vitest tests: connected status display, disconnected status display, disconnect action with confirmation.

## Out of Scope

- Multiple calendar connections (one GCal account per expert at MVP)
- Calendar sync direction configuration (always read freebusy + write events)
- Working hours / availability rules configuration UI (uses GCal native settings)
- Outlook / Teams calendar integration (post-MVP — DEC-41)
- Manual availability overrides outside GCal

## Notes

- **OAuth popup vs redirect:** Delivery to decide. Popup keeps the dashboard context (better UX) but is more complex (postMessage coordination). Redirect is simpler but loses page state. Both are acceptable — popup preferred if implementation effort is reasonable.
- **Availability preview** is a confidence signal — it shows the expert that the platform can actually read their calendar. If slots look wrong, they know to check their GCal settings before prospects encounter issues.
- **GCal token encryption:** The OAuth tokens are stored encrypted in the DB (AES-256-GCM via E06S10 AC4). The dashboard never sees raw tokens — only the connection status and connected email.
- **PostHog events (Delivery must include):** `expert.gcal_connect_started` (clicked connect), `expert.gcal_connected` (successful callback), `expert.gcal_disconnected` (successful disconnect). Server-side capture in actions.
