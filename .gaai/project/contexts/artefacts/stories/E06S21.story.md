# Story E06S21: Vectorize Infrastructure + Embedding Pipeline

## User Story

**As a** platform operator,
**I want** expert profiles to be semantically indexed in Vectorize,
**so that** prospect requirements can be matched against experts using semantic similarity instead of exact keyword overlap.

---

## Context

Current matching relies on exact skill/industry string matching with a small alias map (30 entries) and industry proximity matrix (8 pairs). This fails on the long tail: "data science" ≠ "machine learning", "medtech" ≠ "biotech".

Vectorize + Workers AI (`@cf/baai/bge-base-en-v1.5`, 768 dimensions) enables semantic pre-filtering: embed expert profiles at registration time, query Vectorize with prospect requirements, and only score the top-K semantic candidates. This eliminates O(N) scoring while capturing semantic relationships.

## Notes

Embedding text format: `"Skills: react, typescript, nextjs. Industries: saas, fintech. Project types: web apps. Languages: english, french."` — derived from expert profile JSONB fields.

The embedding is generated inline during registration/profile PATCH (~50-100ms) but upserted to Vectorize via `ctx.waitUntil` to avoid blocking the HTTP response.
