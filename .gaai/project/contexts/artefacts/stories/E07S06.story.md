# Story E07S06: PostHog EU — Compte + Dashboards + MCP

## User Story

**As a** founder,
**I want** PostHog EU Cloud configured, dashboards created, and MCP server connected to Claude Code,
**so that** I can observe funnel conversion, expert activation, and business metrics from day 1 of staging traffic.

---

## Context

Les workers PostHog sont déployés (E07S01 proxy, E07S02 client-side, E07S03 server-side, E07S04 dashboard script, E07S05 MCP skill) mais aucun compte PostHog EU n'existe encore. Sans compte :

- `POSTHOG_API_KEY` (déjà settée en staging pour core + satellite) pointe vers un projet inexistant → les events capturés sont perdus
- Le script `scripts/posthog-setup-dashboards.ts` n'a jamais été exécuté
- Le MCP server PostHog n'est pas connecté dans Claude Code → la skill `analytics-query` est inutilisable
- `ph.callibrate.io` n'est pas routé vers le proxy worker

Cette story est **founder-executed** : aucun code à écrire.

---

## Acceptance Criteria

- [ ] AC1: Compte PostHog EU Cloud créé sur `eu.posthog.com` avec `ops@callibrate.io` — région EU vérifiée dans Settings → Organization
- [ ] AC2: Projet PostHog créé (nom : "Callibrate") — `POSTHOG_PROJECT_ID` noté (visible dans Settings → Project)
- [ ] AC3: `POSTHOG_API_KEY` (project API key) récupérée dans Settings → Project → API Keys → Project API Key (format `phc_xxx`) → vérifier que c'est la même valeur déjà settée en staging (sinon mettre à jour via `wrangler secret put POSTHOG_API_KEY --env staging` pour core + satellite)

- [ ] AC6: Script dashboards exécuté : `POSTHOG_PERSONAL_API_KEY=phx_xxx POSTHOG_PROJECT_ID=xxx npx tsx scripts/posthog-setup-dashboards.ts` → 3 dashboards créés (Prospect Conversion Funnel, Expert Activation Funnel, Business Overview) visibles dans PostHog UI
- [ ] AC7: Claude Code redémarré → `/mcp` confirme serveur `posthog` présent et connecté

## Notes

**Région EU obligatoire :** PostHog EU Cloud (`eu.posthog.com`) — pas US. Les workers envoient vers `eu.i.posthog.com`. Si le projet est créé sur US Cloud, les events seront perdus (host mismatch).

**API Key vs Personal API Key :** deux clés différentes.
- `POSTHOG_API_KEY` = project API key (format `phc_xxx`) → settée comme secret Worker → server-side event capture
- `POSTHOG_PERSONAL_API_KEY` = personal API key (format `phx_xxx`) → shell env var → MCP server + scripts

**script tsconfig :** le script utilise `scripts/tsconfig.json` → `npx tsx` doit être disponible (`npx tsx scripts/posthog-setup-dashboards.ts` depuis la racine du repo).
