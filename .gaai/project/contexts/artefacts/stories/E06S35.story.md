# Story E06S35: Monorepo restructure — move Core API into workers/api/

## User Story

**As a** developer working on the Callibrate platform,
**I want** the Core API Worker to live under `workers/api/` alongside the other Workers,
**so that** every Worker follows the same directory convention and the root is an unambiguous monorepo orchestrator.

---

## Context

The project currently has three Cloudflare Workers but an inconsistent layout:

- `src/` + `wrangler.toml` + `package.json` + `tsconfig.json` at root = Core API Worker
- `workers/satellite/` = Satellite Worker (own package.json, wrangler.toml, tsconfig.json)
- `workers/posthog-proxy/` = PostHog Proxy Worker (own package.json, wrangler.toml, tsconfig.json)

This happened because the project started as a single Worker (default `wrangler init` scaffold). When satellite and posthog-proxy were added later, they correctly went under `workers/`, but the original Core API was never moved.

The result: the root `package.json`, `tsconfig.json`, `wrangler.toml` all belong to one specific Worker, not to the monorepo. Running `npm install` or `npm run dev` at root only affects Core API. There is no unified workspace.

This story restructures the project into a proper npm workspaces monorepo.

---

## Acceptance Criteria

- [ ] AC1: `workers/api/` directory exists and contains: `src/`, `wrangler.toml`, `package.json`, `package-lock.json`, `tsconfig.json`, `d1/`, `.dev.vars.example`
- [ ] AC2: Root no longer contains Worker-specific files — no `src/`, no `wrangler.toml`, no Worker `tsconfig.json` at root level
- [ ] AC3: Root `package.json` is a workspace orchestrator: `"private": true`, `"workspaces": ["workers/*"]`, with orchestrator scripts (`dev:api`, `dev:satellite`, `dev:proxy`, `test`, `type-check`, `deploy:staging`, `deploy:prod`) that delegate to the appropriate workspace
- [ ] AC4: `npm install` at root installs all workspace dependencies (hoisted where possible). Each `workers/*/package-lock.json` is removed — a single root `package-lock.json` manages all workspaces.
- [ ] AC5: All three GitHub Actions workflows (`.github/workflows/deploy-staging.yml`, `deploy-production.yml`, `preview-deploy.yml`) updated: Core API job uses `workingDirectory: workers/api`, `cache-dependency-path` points to root `package-lock.json` for all jobs
- [ ] AC6: `workers/api/wrangler.toml` has `main = "src/index.ts"` — all bindings, envs, queues, workflows unchanged from the current root `wrangler.toml`
- [ ] AC7: `workers/api/tsconfig.json` has `"include": ["src/**/*"]` — compiler options unchanged
- [ ] AC8: All relative imports within `workers/api/src/` are unchanged (since `src/` moved as a unit)
- [ ] AC9: `SETUP.md` updated — all paths referencing root `wrangler.toml`, root `package.json`, root `npm install` corrected to reflect `workers/api/` or workspace root as appropriate
- [ ] AC10: `.gitignore` updated if needed — `.dev.vars` pattern covers `workers/api/.dev.vars`, `node_modules` pattern covers hoisted root `node_modules/` and any per-worker `node_modules/`
- [ ] AC11: `scripts/` directory stays at root (monorepo-level utilities, not Worker-specific)
- [ ] AC12: `supabase/`, `secrets/`, `.gaai/`, `.claude/`, `CLAUDE.md` stay at root — unaffected
- [ ] AC13: `tsc --noEmit` passes from `workers/api/` (and from each other worker directory)
- [ ] AC14: `vitest run` passes from `workers/api/` (and from `workers/posthog-proxy/`)
- [ ] AC15: `npm run deploy:staging` from root triggers all three Worker deploys (or each Worker deploy script works individually from its own directory)

---

## Out of Scope

- Shared packages / libraries extracted from Core API (e.g., `packages/shared-types/`) — premature until there is actual code duplication across Workers
- Turborepo, Nx, or other monorepo build tools — npm workspaces is sufficient for 3 Workers
- Moving `scripts/` into a dedicated workspace
- Renaming Worker names in Cloudflare dashboard (the deployed Worker names remain `callibrate-core-*`, `callibrate-satellite-*`, `callibrate-posthog-proxy-*`)

---

## Dependencies

- None — this is a structural refactoring with no functional dependency

## Notes

**Files that move to `workers/api/`:**
- `src/` (entire directory)
- `wrangler.toml`
- `package.json` (becomes the Worker-specific package.json)
- `package-lock.json` (deleted — replaced by root workspace lockfile)
- `tsconfig.json`
- `d1/` (D1 schema files, referenced by wrangler.toml)
- `.dev.vars.example`
- `.dev.vars` (gitignored, must be recreated locally)

**Files that stay at root:**
- `.gaai/`, `.claude/`, `.github/`, `.git/`, `.gitignore`, `.mcp.json`
- `CLAUDE.md`, `SETUP.md`
- `scripts/`, `supabase/`, `secrets/`, `temp/`

**New at root:**
- `package.json` (workspace orchestrator)
- `package-lock.json` (single lockfile for all workspaces)

**Risk: git history.** Use `git mv` for all moves to preserve file history. Do NOT copy+delete.

**Risk: .wrangler/ state directory.** The `.wrangler/` at root is local dev state for Core API. It can be deleted (regenerated on next `wrangler dev`). Ensure `.gitignore` covers `workers/api/.wrangler/`.

**Merge impact:** This story will conflict with any open PR that touches `src/` files or root config. Best delivered when staging is clean (no pending merges).
