## Quality Notes

### Script Idempotency

The smoke test script is fully idempotent:
- Expert register returns 409 on re-run → gracefully reuses USER_ID from JWT as EXPERT_ID
- Profile PATCH is idempotent by nature
- Prospect submit creates a new prospect_id each run (acceptable)
- No teardown required

### AC8 — Empty Match Results

The smoke-runner expert was created with a complete profile (skills: n8n, automation; industries: technology; budget_range: 5000-15000). However, the EXPERT_POOL KV cache was not yet populated for this user (KV population triggered by score-computation queue, which fires asynchronously post-register). The API returns HTTP 200 with empty results — explicitly acceptable per story notes: *"peut être vide si aucun expert ne matche — mais pas d'erreur 5xx"*.

### AC9 — 422 is Correct

The story notes explicitly state: *"le test passe si la réponse est un 200 avec `{ slots: [] }` ou un 4xx métier explicite (pas un 5xx infra)"*. HTTP 422 with `{"error":"gcal_not_connected"}` satisfies this criterion.
