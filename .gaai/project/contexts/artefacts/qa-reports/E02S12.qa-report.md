## Remediation Applied (Round 1)

### Backend API — `src/handlers/bookings/direct.test.ts`

**Issue 1:** `vi.mock('../../lib/db', ...)` used a plain arrow function for `createSql` (not a `vi.fn()`), so `vi.mocked(createSql).mockImplementation(...)` in the "returns 200 with booking_id on success" test failed at runtime with `mockImplementation is not a function`.

**Fix:** Changed the mock factory to wrap `createSql` in `vi.fn().mockImplementation(defaultFactory)` so per-test override works.

**Issue 2:** `global.fetch = mockFetch` — `global` is not defined in `@cloudflare/workers-types` environment (TS2304). Changed to `(globalThis as any).fetch = mockFetch`.

**Issue 3:** `mockSql.end = vi.fn()...` — TypeScript error TS2339 because `Mock<Procedure>` has no `.end` property. Changed to `(mockSql as any).end = vi.fn()...`.

**Issue 4:** `vi.mocked(createSql).mockImplementation(() => {...})` — return type mismatch (TS2345). Added `as any` cast to the implementation function.

### Backend API — `src/lib/milestones.test.ts`

**Issue:** `ExpertRow` type was updated in E02S12 to add three required fields (`direct_link_nonce`, `direct_submissions_this_month`, `direct_submissions_reset_at`). The `makeExpert` factory in milestones.test.ts predates E02S12 and didn't include these fields, causing TS2322.

**Fix:** Added the three E02S12 fields with default values to the `makeExpert` factory.

### Landing Worker — `src/pages/direct-booking.test.ts` and `src/pages/expert-profile.test.ts`

**Issue:** `global.fetch = vi.fn()...` used 9 times across two test files. Same TS2304 error as in backend.

**Fix:** Replaced all `global.fetch` with `(globalThis as any).fetch` in both files.
