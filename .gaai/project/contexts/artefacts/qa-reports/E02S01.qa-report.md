## AC5 Architecture Note

The AC spec says `_layout.tsx` should pass `{ user }` to child routes via Outlet context. The implementation uses a two-layer pattern instead:
- `_layout.tsx`: auth guard only — calls `requireSession()`, returns user as JSON loader data, renders `<Outlet />` without context
- `_layout.dashboard.tsx`: dashboard shell — also calls `requireSession()`, renders `<Outlet context={{ user }}>` with context

This means `dashboard.tsx` reads user from `_layout.dashboard.tsx`'s context (via `useOutletContext()`), not from `_layout.tsx`. The AC literal requirement is met by `_layout.dashboard.tsx`. The architecture is functionally sound: `_layout.tsx` is a pure auth guard, `_layout.dashboard.tsx` is the UI shell that provides context. Verdict: PASS (functional requirement satisfied, architectural variation acceptable).
