# E02S11 Memory Delta

## New Knowledge

- **Timezone conversion in CF Workers:** `Intl.DateTimeFormat` with `timeZone` option works correctly in Cloudflare Workers runtime. Strategy: compute UTC offset by comparing a reference UTC date with its representation in the target timezone using `toLocaleString('en-US', { timeZone })`. Apply inverse offset to convert local TIME values to UTC. Handles IANA timezone names including DST.
- **postgres.js TIME columns:** Returned as `'HH:MM:SS'` strings (with seconds). Always use `.slice(0, 5)` when comparing or returning HH:MM to API clients.
- **`expandRulesToWorkingHours()` pattern:** Converts `ExpertAvailabilityRuleRow[]` to `Record<string, [string, string]>` keyed by DAY_KEYS day names. Integrates with existing `mergeRules`/`computeFreeSlots` pipeline without modification.
- **Hybrid availability model (DEC-118 Layer 1):** DB rules → JSONB fallback → `not_configured`. GCal (Layer 2) is now non-blocking (try/catch, empty busyIntervals on error). `availability_status` field added to response metadata.
