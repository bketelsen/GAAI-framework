# Memory Delta — E02S05

## Story Delivered

**E02S05 — Lead Pipeline Page + Actions**

Route: `dashboard.leads.tsx`
Test file: `tests/leads-action.test.ts`

## Implementation Facts

### Architecture
- Single route file handles loader + three actions (confirm, flag, evaluate) using RR7 intent pattern.
- `<Form method="post">` with `<input type="hidden" name="intent" value="...">` dispatches to action branches.
- `useEffect` on `useActionData()` drives toast notifications — the standard pattern for this dashboard.

### Loader
- Calls `GET /api/experts/:userId/leads?status={status}&page={page}&per_page=20`.
- Graceful degradation: API failures return `{ leads: [], total: 0, page: 1, per_page: 20 }` — no error thrown to the UI.
- PostHog event: `expert.leads_viewed` with `status_filter`.

### Action — confirm
- Endpoint: `POST /api/leads/:id/confirm` with empty body `{}`.
- PostHog event: `expert.lead_confirmed` with `lead_id` + `price_cents`.
- Validation: leadId required (400 if missing).

### Action — flag
- Endpoint: `POST /api/leads/:id/flag` with `{ reason }`.
- Server-side validation: `flag_reason` must be >= 20 chars (422 if not).
- PostHog event: `expert.lead_flagged` with `lead_id` + `reason_length`.
- UI: Flag button only rendered when `flagWindowInfo()` returns non-null (i.e., window not expired).

### Action — evaluate
- Endpoint: `POST /api/leads/:id/evaluate` with `{ score, notes?, conversion_declared }`.
- Server-side validation: score 1–10 (422), notes <= 500 chars (422).
- Notes passed as `undefined` (not empty string) when not provided.
- PostHog event: `expert.lead_evaluated` with `lead_id`, `score`, `conversion_declared`.

### Helpers
- `flagWindowInfo(lead)`: Returns `{ daysLeft, urgent }` or `null`. Urgent = `hoursLeft < 24`. Used for both badge rendering and flag button gating.
- `formatPrice(cents)`: `Intl.NumberFormat` fr-FR EUR formatting.
- `relativeDate(isoString)`: Humanized relative time in French (< 1h / Nh / Nj).
- `statusBadgeClass(status)`: Tailwind class map for new/confirmed/flagged/default.
- `buildTabUrl` / `buildPageUrl`: URL builders for tab and pagination navigation.

### UI Patterns
- Status tabs: `<Link>` elements with active state via `border-b-2 border-primary`. Count badge on active tab only.
- Table click opens detail dialog; action cell uses `e.stopPropagation()` to prevent row-click conflict.
- Flag dialog is a separate `<Dialog>` controlled by `flagDialogOpen` state, separate from detail dialog.
- Evaluation form only shown inside detail dialog for `confirmed` leads with `evaluation_score === null`.
- Empty state: rendered when `leads.length === 0`, regardless of filter.

## Test Coverage Summary
- 19 tests across 4 suites.
- Loader: default params, status filter (new), status filter (confirmed), API failure fallback.
- Confirm: success path, missing leadId (400), API error (500).
- Flag: success with valid reason, short reason (422), missing leadId (400), API error (500).
- Evaluate: full fields success, score-only success, score > 10 (422), score = 0 (422), score NaN (422), notes > 500 (422), API error (500), unknown intent (400).

## Decisions / Patterns to Propagate

- **RR7 intent pattern**: All mutations in this dashboard use `intent` hidden field + single action function with if-branches. Consistent with E02S03/E02S04.
- **Flag window as render gate**: The `flagWindowInfo()` helper returning `null` for expired windows is a clean pattern for any time-gated action — hide rather than disable.
- **Graceful loader degradation**: API failure in loader yields empty state rather than error boundary. Adopted consistently across dashboard routes.
