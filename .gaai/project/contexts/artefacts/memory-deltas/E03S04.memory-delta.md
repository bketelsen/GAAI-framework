## New Knowledge

### 1. Satellite page rendering pattern (async)

Directory and profile pages are **async** unlike earlier satellite pages (landing, match, confirm, results). Pattern:

```typescript
export async function renderXxxPage(
  config: SatelliteConfig,
  posthogApiKey: string,
  coreApiUrl: string,
): Promise<string> {
  const res = await fetch(`${coreApiUrl}/api/...`);
  // render HTML
}
```

Route handlers use `await renderXxxPage(...)` and return `new Response(html, { status, headers })`.

### 2. Expert profile page returns `{ html, status }` — not just `html`

Unlike directory pages that always return 200, profile pages must propagate 404 (expert not found) and 500 (network error). Pattern:

```typescript
export async function renderExpertProfilePage(...): Promise<{ html: string; status: number }> {
  try {
    const res = await fetch(`${coreApiUrl}/api/experts/public/${slug}`);
    if (!res.ok) return { html: '', status: res.status };
    // render...
    return { html, status: 200 };
  } catch {
    return { html: '', status: 500 };
  }
}
```

### 3. Unlock flow from directory uses `create-from-directory` endpoint

The "Débloquer ce profil" CTA on profile pages calls `POST /api/prospects/create-from-directory` with `{ email, expert_slug }`. This creates a minimal `identified` prospect without quiz context, issues a `prospect:identify` JWT, then fires `booking-open` CustomEvent to open the booking widget. This is distinct from the quiz funnel path (`POST /api/prospects/submit`).

### 4. AC11 direct booking CTA is conditional on `direct_booking_url` field

The field is `null` until E02S12 delivers the DB column and server-side token generation. The CTA is hidden when `direct_booking_url === null`. No code change needed in E03S04 when E02S12 ships — the API field presence auto-enables the CTA.

### 5. Sitemap made async for expert slug inclusion

`renderSitemapXml` is now `async` with optional `coreApiUrl`. Pattern applied: pass `coreApiUrl` as second arg when calling from routes; tests that don't pass it continue to work (no slugs fetched). Expert slugs fetched via `GET /api/experts/public?per_page=50` and cached at CF edge with 86400s.
