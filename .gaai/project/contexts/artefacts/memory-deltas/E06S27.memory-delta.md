## New Knowledge

### 1. wrangler `[[ratelimits]]` format: `name` not `binding`

CF Workers Rate Limiting binding uses `name` (not `binding`) in wrangler.toml:
```toml
[[ratelimits]]
name = "RATE_LIMITER"   # ← correct
namespace_id = "1001"
simple = { limit = 30, period = 60 }
```
The E06S20 code used `binding = "RATE_LIMITER"` — this is incorrect and will fail wrangler validation. All PRs using E06S20's Rate Limiter binding format must be corrected before deployment.

### 3. Hyperdrive requires DB password — not available in any accessible secret store

The Supabase PostgreSQL direct connection password is NOT stored in:
- Wrangler secrets (only JWT API keys stored)
- `.dev.vars` (blocked by permissions)
- Supabase MCP (does not expose DB passwords)

**Action for future provisioning**: The founder must provide the DB password manually via Supabase Dashboard → Settings → Database → Connection string.

### 4. CF Workflows — no CLI creation

CF Workflows cannot be created via wrangler CLI. They require CF Dashboard action: Workers & Pages → Workflows → New Workflow. This should be documented as a permanent founder action, not attempted by agents.

### 5. D1 schema: staging vs production database naming convention

D1 databases were named `expert-pool-edge-staging` and `expert-pool-edge-prod` to match CF environment conventions. The `database_name` field in wrangler.toml must be set to these exact names. Code in E06S23 that references the database should use these exact names.
