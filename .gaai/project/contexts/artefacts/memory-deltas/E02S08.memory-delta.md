## Notes

### New patterns found

1. **recharts as charting library.** `recharts ^3.7.0` is now a production dependency of the dashboard worker. Components used: `RadarChart`, `BarChart`, `ResponsiveContainer`. This is the first charting library introduced in the frontend.

2. **Pure helper exports from route files for testability.** `getTierConfig`, `formatMonthLabel`, `safePct` are exported directly from `dashboard.analytics.tsx` so vitest can unit-test them without a component harness. This is a new pattern not previously present in the codebase — clean, recommended for future routes with non-trivial pure logic.

3. **`DashboardIndexResponse` extended with `monthly_history` and `month_stats`.** The dashboard index route (`dashboard.tsx`) now consumes `monthly_history: MonthHistory[]` and `month_stats: MonthStats` from the dashboard API. These fields were already returned by `GET /api/experts/:id/dashboard` but not previously used in the frontend.

4. **Composite score and tier shown on dashboard index.** The summary card for composite score on the dashboard index now renders tier badge + label inline, linking to `/dashboard/analytics`. This surfaces score context at the overview level.

### Conventions alignment confirmed

- Session guard pattern: `requireSession` → extract `session.user.id` → `apiGet` — matches all existing dashboard routes.
- `captureEvent(...).catch(() => {})` non-blocking PostHog pattern — consistent.
- `Response.json({ ... } satisfies LoaderData, { headers: responseHeaders })` — consistent.
- Graceful degradation via `.catch(() => defaultValue)` on all API calls — consistent.
- Route registration in `routes.ts` via `route()` inside dashboard layout — consistent.
