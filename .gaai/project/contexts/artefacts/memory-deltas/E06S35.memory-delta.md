## What Changed

The monorepo was restructured from a flat layout to a backend/frontend split:

| Before | After |
|--------|-------|
| Root = Core API Worker (src/, wrangler.toml, package.json) | Root = npm workspaces orchestrator |
| `workers/matching/` | `workers/backend/matching-engine/` |
| `workers/posthog-proxy/` | `workers/backend/posthog-proxy/` |
| `workers/satellite/` | `workers/frontend/satellites/` |
| Per-worker `package-lock.json` files | Single root `package-lock.json` |

```
- Worker scripts delegated via `npm -w <workspace-name>` (e.g. `npm -w callibrate-core run dev`)
```

## Key Architecture Decisions

### npm Workspaces as Orchestrator
- Root `package.json` has `"private": true` and `"workspaces"` array
- All dev dependencies (wrangler, typescript, vitest) are hoisted to root `node_modules/`
- Each worker `package.json` declares its own `name` (e.g. `callibrate-core`) used by `npm -w` commands
- CI runs `npm ci` at root, then uses `working-directory` or `workingDirectory` to target specific workers

### Git History Preserved
- All file moves done via `git mv` — git tracks as renames (0 byte changes in diff)
- Per-worker `package-lock.json` files deleted (replaced by single root lockfile)

### Remaining Documentation Debt
- `workers/frontend/satellites/README.md` still references `workers/satellite/` (old path) in 4 places
- GAAI historical artefacts (plans, stories, qa-reports, impl-reports) reference old paths — these are historical records and should NOT be updated

## CI/CD Pattern (Updated)

All 3 GitHub Actions workflows follow the same pattern:
1. Checkout
2. Setup Node 22
3. `npm ci` (at root — installs all workspaces)
4. Worker-specific steps use `working-directory` (shell steps) or `workingDirectory` (Actions inputs) to target the correct worker

The `deploy-production.yml` adds `tsc --noEmit` and `vitest run` gates for Core API before deploying.
