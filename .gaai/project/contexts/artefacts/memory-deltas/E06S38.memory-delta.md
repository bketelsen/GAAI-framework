# Memory Delta E06S38

## New Patterns

### 1. Dashboard read endpoint pattern (multi-query aggregation)
Dashboard endpoints with multiple independent aggregations (expert base data, counts, monthly stats) use sequential `await sql<>` calls rather than a single complex JOIN. This keeps individual queries simple and readable, at the cost of 5-7 round-trips. Acceptable for authenticated dashboard loads (not hot path). Pattern established in `handlers/experts/dashboard.ts`.

### 2. Window function for pagination total (`COUNT(*) OVER()`)
The bookings handler uses `COUNT(*) OVER() AS total_count` to retrieve the total alongside rows in a single query, avoiding a separate COUNT query. This is a deliberate optimization over the two-query pattern used in leads/billing. Both patterns are now present in the codebase — use window function when the query has a single WHERE variant; use separate COUNT when status/period branching makes a combined query awkward.

### 3. Monthly history zero-padding utility
When returning time-series data for analytics charts, generate the full 6-month array in TypeScript first, then fill from SQL results using a Map keyed by `YYYY-MM`. This avoids relying on SQL to produce empty rows for months with no data. Pattern: `padMonthlyHistory()` in `handlers/experts/dashboard.ts`.

### 4. LATERAL JOIN for latest match score
When a leads row needs the most recent match score (N matches may exist per expert+prospect pair), use a `LEFT JOIN LATERAL (SELECT score FROM matches WHERE expert_id = l.expert_id AND prospect_id = l.prospect_id ORDER BY created_at DESC LIMIT 1) m ON true`. This avoids a subquery-per-row and is idiomatic postgres.js.

### 5. Public endpoint route ordering
Unauthenticated routes that share a path prefix with an authenticated block (`/api/experts/public` within `/api/experts/`) must be registered BEFORE the authenticated block in `index.ts`. The router uses early returns — a route registered inside the auth block will require JWT even if the handler does not. Documented with a comment in `index.ts` line 290-291.

### 6. `ctx` as 5th parameter for handlers needing `waitUntil`
Handlers that fire PostHog events asynchronously via `ctx.waitUntil()` must receive `ctx: ExecutionContext` as an explicit parameter (not captured from closure). Router call site passes `ctx` from the top-level `fetch` handler. Pattern: `handleEvaluateLead(request, env, user, leadId, ctx)`.

### 7. Shared utility extraction pattern for quality tier
`deriveQualityTier` was extracted from `routes/matches.ts` into `lib/qualityTier.ts` to avoid duplication across dashboard and public endpoints. The utility defines its own `QualityTier` type (not re-exported from `types/matching.ts`) to avoid potential circular dependency. The type is a simple 4-value union — safe to duplicate at this scope.

### 8. Anonymized public endpoint shape
Public expert directory endpoints must exclude: `display_name`, `email`, `bio` (full). Permitted fields: `slug`, `headline`, `skills[]`, `industries[]`, `rate_min`, `rate_max`, `composite_score`, `quality_tier`, `completed_projects`, `languages[]`. Detail endpoint adds: `bio_excerpt` (first 200 chars), `availability_status`, `outcome_tags[]`. The `bio` field is fetched from DB for the excerpt computation but is never surfaced directly.
