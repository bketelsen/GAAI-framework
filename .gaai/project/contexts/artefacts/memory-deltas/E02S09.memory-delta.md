## New Knowledge

### Pattern: Landing Page PostHog Injection (E02S09)

The landing page PostHog injection pattern differs from the satellite worker pattern in two ways:

**Event naming:** Satellite uses `satellite.cta_clicked` with `{satellite_id, cta_text}`. Landing uses `landing.cta_clicked` with `{cta_location}`. The `cta_location` property (nav/hero/pricing/footer) provides funnel position context specific to landing pages, while satellite uses `cta_text` because satellites are multi-config.

**CTA tracking mechanism:** Landing uses `data-cta-location` HTML attribute + `querySelectorAll('.cta[data-cta-location]')` in body script. Satellite uses `data-cta-text`. Both use the same class `.cta` selector base.

**page_view properties:** Landing captures `{page:'landing', referrer, utm_source, utm_campaign, utm_medium, utm_content}`. Satellite captures `{satellite_id, referrer, utm_source, utm_campaign, utm_medium}`. Landing adds `utm_content` (T6 fix) and omits `satellite_id` (not applicable).

This pattern should be documented in `patterns/conventions.md` under the PostHog Analytics Patterns section.
