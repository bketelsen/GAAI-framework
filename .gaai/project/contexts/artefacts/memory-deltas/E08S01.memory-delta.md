## Patterns Worth Preserving (for future reference)

### PKCE + KV State in Cloudflare Workers

If future stories implement OAuth flows in CF Workers, the established pattern from E08S01 is:

1. **State token**: `crypto.getRandomValues(new Uint8Array(32))` → hex-encoded to 64 chars. Stored in SESSIONS KV as `oauth-state:{token}` with `expirationTtl: 600`.

2. **PKCE code_verifier**: `crypto.getRandomValues(new Uint8Array(32))` → BASE64URL-encoded (43 chars). Stored in the same KV value as `expert_id`.

3. **PKCE code_challenge**: `crypto.subtle.digest('SHA-256', new TextEncoder().encode(codeVerifier))` → BASE64URL-encoded. Passed as `code_challenge` + `code_challenge_method=S256` in auth URL.

4. **KV lifecycle**: Create (auth-url endpoint) → Read + Delete immediately (callback) → If absent, return error. This prevents replay attacks.

5. **Web Crypto API** (`crypto.getRandomValues`, `crypto.subtle`) is available natively in CF Workers — no polyfills or external libraries needed.

This pattern is correct, minimal, and well-tested. It can serve as the reference implementation for any future OAuth provider integrations.
