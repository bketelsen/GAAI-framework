# E06S12 — Migrate /api/extract from Anthropic to GPT-4o-mini: Execution Plan

## Scope Summary

Pure provider swap on a single route. No schema changes, no new DB tables, no new routes,
no new bindings beyond swapping one secret for another. All business logic (validation,
satellite context, confidence scoring, question capping, response shape) is unchanged.
Four files require modification — one is the main route, one is the type interface, one is
the test file, one is documentation.

#### 2c — Rewrite the fetch call section (AC2, AC7)

Remove the Anthropic fetch block. Replace with:

```javascript
const openaiBody = {
  model: MODEL,
  messages: [
    { role: 'system', content: systemPrompt },
    { role: 'user', content: text },
  ],
  tools: [EXTRACT_TOOL],
```

Key structural difference: OpenAI uses `messages` array with a `role: 'system'` entry
(the system prompt is no longer a separate top-level field). No `max_tokens` required.
No Anthropic-version header.

Variable rename: `anthropicRes` → `openaiRes`, `anthropicBody` → `openaiBody`.

Error strings (AC7):
- fetch catch block → `'AI service unreachable'` (was `'AI Gateway unreachable'`)
- non-200 branch → `'OpenAI API error'` (was `'Anthropic API error'`)

#### 2d — Rewrite the response parsing helper (AC3, AC7)

Remove the `extractToolUseBlock` function entirely (it parses Anthropic content array format).

Add a new internal function `extractOpenAIToolCall(data: unknown): ToolUseInput | null`:

```javascript
function extractOpenAIToolCall(data: unknown): ToolUseInput | null {
  if (typeof data !== 'object' || data === null) return null;
  const msg = data as Record<string, unknown>;
  const choices = msg['choices'];
  if (!Array.isArray(choices) || choices.length === 0) return null;
  const message = (choices[0] as Record<string, unknown>)['message'];
  if (typeof message !== 'object' || message === null) return null;
  const toolCalls = (message as Record<string, unknown>)['tool_calls'];
  if (!Array.isArray(toolCalls) || toolCalls.length === 0) return null;
  const fn = (toolCalls[0] as Record<string, unknown>)['function'];
  if (typeof fn !== 'object' || fn === null) return null;
  const args = (fn as Record<string, unknown>)['arguments'];
  if (typeof args !== 'string') return null;
  try {
    const parsed = JSON.parse(args);
    if (typeof parsed !== 'object' || parsed === null) return null;
    return parsed as ToolUseInput;
  } catch {
    return null;
  }
}
```

The `ToolUseInput` interface (`requirements`, `confidence`, `confirmation_questions?`) is unchanged.

#### 2e — Update call sites within `handleExtract`

- `anthropicData` → rename to `openaiData`
- Call `extractOpenAIToolCall(openaiData)` instead of `extractToolUseBlock(anthropicData)`
- Error string: `'Invalid response from OpenAI API'` (was `'Invalid response from Anthropic API'`)
- Error string: `'Unexpected response shape from OpenAI API'` (was `'Unexpected response shape from Anthropic API'`)

## Edge Cases per AC

### AC1 — Contract unchanged
- `confirmation_questions` is optional in both formats. Filtering logic is unchanged.
- If OpenAI `arguments` JSON parse fails → `extractOpenAIToolCall` returns null → 500
  with clear error. Caller never sees broken response shape.

### AC2 — OpenAI Chat Completions format
- `tool_choice: { type: "function", function: { name: "extract_requirements" } }` —
  named form is mandatory to guarantee `tool_calls` is populated. Generic form risks
  a text response instead of tool call.
- `tools` array uses `type: "function"` wrapper — verify constant restructuring is complete.

### AC3 — Response parsing path
- `choices` array can be empty on content policy refusal (HTTP 200). Guard:
  `choices.length === 0 → null → 500`. Correct behavior.
- `tool_calls[0]` taken without name-check. Acceptable for MVP (one tool declared).
- `arguments` type guard (`typeof args !== 'string'`) included.

### AC4 — Env interface
- Run `grep -r ANTHROPIC_API_KEY src/` after Step 1 to verify no other files reference it.
- `wrangler.toml` runtime secrets are set via `wrangler secret put`, not declared in TOML —
  no `wrangler.toml` changes needed.

### AC6 — Satellite vertical context preserved
- `systemPrompt` construction logic is unchanged.
- OpenAI `messages[0]` with `role: 'system'` delivers the same string. ~200 chars —
  negligible against gpt-4o-mini context window.

### AC7 — Error strings (4 strings total)
1. `'AI Gateway unreachable'` → `'AI service unreachable'` (fetch catch)
2. `'Anthropic API error'` → `'OpenAI API error'` (non-200)
3. `'Invalid response from Anthropic API'` → `'Invalid response from OpenAI API'` (JSON parse fail)
4. `'Unexpected response shape from Anthropic API'` → `'Unexpected response shape from OpenAI API'` (null tool block)
- Current tests check only `res.status`, not `body.error` string — low breakage risk.

### AC8 — Tests updated
- Mock `arguments` must be JSON string, not JS object. Use `JSON.stringify({...})`.
- Low-confidence mock keeps 4 questions → route caps at 3 → assertion `<= 3` passes.
