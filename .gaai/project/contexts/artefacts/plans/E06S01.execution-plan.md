### Phase 2 — Project Structure: File Scaffold (no wrangler auth needed)

**Step 2.1** — Create `.gitignore`
- Path: `.gitignore`
- Include: `node_modules/`, `.wrangler/`, `.env`, `.env.*`, `dist/`, `*.local`

**Step 2.2** — Create `package.json`
- Path: `package.json`
- Dependencies: `@cloudflare/workers-types`, `@supabase/supabase-js`
- DevDependencies: `wrangler`, `typescript`, `@types/node`
- Scripts: `dev`, `deploy:staging`, `deploy:prod`, `type-check`

**Step 2.3** — Create `tsconfig.json`
- Path: `tsconfig.json`
- Target: `ES2022`, `strict: true`, `moduleResolution: bundler`
- Types: `@cloudflare/workers-types`

**Step 2.4** — Create `wrangler.toml`
- Path: `wrangler.toml`
- Three environments: `[env.dev]`, `[env.staging]`, `[env.production]`
- Queue bindings: `email-notifications`, `lead-billing`, `matching-jobs` + DLQs
- KV bindings: `SESSIONS`, `RATE_LIMITING`, `FEATURE_FLAGS`
- ⚠️ Queue IDs and KV namespace IDs are placeholders — filled after `wrangler login`

**Step 2.5** — Create `src/index.ts` (main Worker entry)
- Path: `src/index.ts`
- Implements: `GET /api/health` → AC6
- Health check: lightweight Supabase REST ping (`SELECT 1` equivalent via REST API)
- Returns: `{ status: "ok", supabase: "connected", queues: ["email-notifications", "lead-billing", "matching-jobs"] }`

**Step 2.6** — Create `src/types/env.ts`
- Path: `src/types/env.ts`
- Defines `Env` interface with all bindings: `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_KEY`, `SESSIONS`, `RATE_LIMITING`, `FEATURE_FLAGS`, `EMAIL_NOTIFICATIONS`, `LEAD_BILLING`, `MATCHING_JOBS`

---

### Phase 3 — CI/CD: GitHub Actions (AC7)

**Step 3.1** — Create staging deploy workflow
- Path: `.github/workflows/deploy-staging.yml`
- Trigger: push to `main` branch
- Job: `wrangler deploy --env staging`
- Requires secret: `CLOUDFLARE_API_TOKEN`

**Step 3.2** — Create production deploy workflow
- Path: `.github/workflows/deploy-production.yml`
- Trigger: git tag matching `v*`
- Job: `wrangler deploy --env production`
- Requires secret: `CLOUDFLARE_API_TOKEN`

- Documents exact commands for:
  1. `wrangler login`
