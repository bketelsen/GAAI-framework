## Technical Decisions

- **TD-1:** Section dispatch via `intent` hidden field with values `"identite"`, `"expertise"`, `"marche"`, `"admissibilite"`. The action reads `formData.get("intent")` to branch into the correct validation path.
- **TD-2:** Action returns `Response.json({ success: true, section } | { success: false, section, errors, values })` — never a redirect. React Router 7 auto-revalidates the loader after a successful action, which refreshes all section data from server state.
- **TD-3:** Client-side `useEffect` watching `useActionData()` triggers Sonner toasts on each new action result.
- **TD-4:** `required_stack_overlap_min` is stored as a 0.0–1.0 fraction in the DB. UI displays 0–100%; the action divides the form integer by 100 before storage.
- **TD-5:** The GCal section is READ-ONLY — shows connection status and a link to `/dashboard/gcal` (E02S04). No form submission.
- **TD-6:** Zod schemas are defined in the settings route file (not extracted to shared module) — same isolation pattern as onboarding.tsx.
- **TD-7:** Partial PATCH payload: each section sends only its own fields. Expertise sends `{ profile: { skills, verticals }, rate_min, rate_max, outcome_tags }`. Marche sends `{ preferences }` as full JSONB replace. Admissibilite sends `{ admissibility_criteria }` as full JSONB replace.
- **TD-8:** `preferences` PATCH reconstructs the full object from section form fields — no merge on frontend.
- **TD-9:** Static arrays (PREDEFINED_SKILLS, PREDEFINED_VERTICALS, LANGUAGES, GEO_ZONES, INDUSTRIES, AVAILABILITY_OPTIONS, PROJECT_STAGES, CAREER_STAGES, WORK_MODES) copied from onboarding.tsx.
- **TD-10:** PostHog: `expert.profile_settings_viewed` fired in loader (fire-and-forget). `expert.profile_section_saved` fired in action after successful PATCH.

### Step 4 — `dashboard.settings.tsx`: Loader

```typescript
export async function loader({ request, context }: LoaderFunctionArgs) {
  const { session, responseHeaders } = await requireSession(request, context.cloudflare.env);
  const env = context.cloudflare.env;
  const userId = session.user.id;

  const [profile, gcalStatus] = await Promise.all([
    apiGet<ExpertProfile>(env, session.token, `/api/experts/${userId}/profile`).catch(() => null),
    apiGet<GcalStatus>(env, session.token, `/api/experts/${userId}/gcal/status`).catch(() => null),
  ]);

  captureEvent(env, `expert:${userId}`, "expert.profile_settings_viewed", {}).catch(() => {});

  return Response.json(
    { profile, gcalStatus, userId },
    { headers: responseHeaders },
  );
}
```

### Step 5 — `dashboard.settings.tsx`: Action (four section handlers)

The action reads `intent` and branches. For each section:
1. Extract fields from formData
2. Validate with section schema (safeParse)
3. On validation failure: return `{ success: false, section, errors, values }` with status 422
4. On validation success: call `apiPatch`
5. On API error: return `{ success: false, section, errors: { field: [msg] }, values }` with status 500
6. On API success: fire PostHog + return `{ success: true, section }`

**Section: identite** — sends `{ display_name, headline, bio }`
**Section: expertise** — sends `{ profile: { skills, verticals }, rate_min, rate_max, outcome_tags }`. Parse `rate_min`/`rate_max` with `parseInt(String(...))` — undefined if empty string.
**Section: marche** — sends `{ preferences: { career_stage, work_mode, availability, project_stage, languages, geo_zones, industries, budget_range? } }`. Reconstruct full preferences object.
**Section: admissibilite** — sends `{ admissibility_criteria }`. Divide `required_stack_overlap_min` input (0–100) by 100 before validation.
**Fallback** (unknown intent) — return 400.

4. **`AdmissibiliteSection({ profile, actionData })`**
   - Intro paragraph: "Ces critères filtrent automatiquement les prospects avant le matching."

### Step 8 — `dashboard.settings.tsx`: Main component

```typescript
export default function SettingsPage() {
  const { profile, gcalStatus } = useLoaderData<typeof loader>();
  const actionData = useActionData() as ActionData | undefined;

  useEffect(() => {
    if (!actionData) return;
    if (actionData.success) {
      toast.success("Profil mis à jour");
    } else {
      const firstError = Object.values(actionData.errors)[0];
      toast.error(firstError?.[0] ?? "Erreur lors de la sauvegarde.");
    }
  }, [actionData]);

  return (
    <div className="space-y-6">
      <div>
        <h1 className="text-2xl font-semibold">Paramètres du profil</h1>
        <p className="text-muted-foreground mt-1">Gérez vos informations et critères de matching.</p>
      </div>
      <IdentiteSection profile={profile} actionData={actionData} />
      <ExpertiseSection profile={profile} actionData={actionData} />
      <MarcheSection profile={profile} actionData={actionData} />
      <AdmissibiliteSection profile={profile} actionData={actionData} />
      <GcalSection gcalStatus={gcalStatus} />
    </div>
  );
}
```

## Edge Cases Per AC

| AC | Edge Case | Handling |
|----|-----------|----------|
| AC1 | `apiGet` for profile throws | `.catch(() => null)` — sections render in empty/default state |
| AC1 | gcalStatus API unreachable | `.catch(() => null)` — GCal section shows disconnected |
| AC3 | `rate_min`/`rate_max` submitted as empty strings | `parseInt("")` → `NaN` → guard: if NaN, set undefined |
| AC4 | `career_stage` submitted as invalid enum value | Zod returns field error, action returns 422 |
| AC5 | `useActionData()` undefined on initial load | `if (!actionData) return;` guard in useEffect |
| AC7 | `admissibility_criteria` is null in DB | Section renders with all fields empty/default |
| AC7 | `required_stack_overlap_min` null → slider default | `Math.round((v ?? 0) * 100)` for defaultValue |
| AC8 | Empty admissibility form | `{}` sent — API clears previous criteria (valid) |
| AC9 | TypeScript: `formData.get()` returns `FormDataEntryValue \| null` | Always wrap with `String(formData.get(...) ?? "")` |
| AC10 | captureEvent throws in loader | `.catch(() => {})` prevents loader failure |

| Risk | Severity | Mitigation |
|------|----------|------------|
| `useActionData()` typed as unknown in RR7 | Low | Cast to `ActionData \| undefined` at call site |
| `preferences` PATCH overwrites entire column | Medium | Reconstruct full preferences object from all form values before PATCH |

| `required_stack_overlap_min` float precision | Low | `Math.round(v * 100)` for display; divide by 100 in action |
