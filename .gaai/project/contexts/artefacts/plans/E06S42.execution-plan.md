# Execution Plan — E06S42: GDPR Data Retention: Purge Abandoned Funnels + OTP Cleanup

## Architecture Summary

**Key findings from codebase exploration:**

1. **`freetext`, `requirements`, `extraction_confidence` live on `prospect_projects`** (not `prospects`) — migrated by E06S41. The `prospects` table has `email`, `created_at`, `id`. The join `prospect_projects.prospect_id = prospects.id` is the correct relationship.

2. **KV key patterns (from E06S39 actual implementation):**
   - OTP codes: `otp:{prospectId}` (TTL 600s)
   - Rate limits: `otp:rate:{normalizedEmail}` (TTL 86400s)
   - Stored in `SESSIONS` KV binding (no separate `OTP_KV` — story spec mentions `OTP_KV` but E06S39 used `SESSIONS`)
   - Story spec mentions `otp-sends:*` but actual prefix is `otp:rate:*`

3. **Cron piggyback:** Existing `handleScheduled` in `handlers/bookings/cron.ts` dispatches `cron === '0 * * * *'` to `handleLsBillingCron`. Add `handleGdprCron` sequentially in same branch — no wrangler.toml changes needed.

4. **AC4 location:** `workers/frontend/satellites/src/pages/match.ts` — freetext submit button is the step 1 form submit.

## Implementation Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Cron trigger | Piggyback on existing `0 * * * *` | No new wrangler.toml trigger needed; same hourly cadence |
| Data column targets | `prospect_projects.freetext/requirements/extraction_confidence` | E06S41 migrated these here; `prospects` table has no `freetext` |
| KV namespace | `SESSIONS` | E06S39 actual implementation; `OTP_KV` binding doesn't exist |
| PostHog event | `system.abandoned_funnel_purged` with `{ count }` | No PII; matches DEC-65 observability patterns |
| Error isolation | `try/catch` per sub-function | GDPR cron failure must not abort billing cron |
| Idempotency | `AND (freetext IS NOT NULL OR ...)` guard | Prevents double-counting in audit log on repeated runs |

CREATE TABLE IF NOT EXISTS data_purge_log (

### Step 2 — New file: `gdprCron.ts`

**File:** `workers/backend/api/src/cron/gdprCron.ts`

Structure:
```typescript
import type { Env } from '../types/env';
import { createSql } from '../lib/db';
import { captureEvent } from '../lib/posthog';

export async function handleGdprCron(env: Env): Promise<void> {
  try { await purgeAbandonedFunnels(env); } catch (err) {
    console.error('handleGdprCron: purgeAbandonedFunnels failed', err);
  }
  try { await cleanupOtpKvKeys(env); } catch (err) {
    console.error('handleGdprCron: cleanupOtpKvKeys failed', err);
  }
}

async function purgeAbandonedFunnels(env: Env): Promise<void> {
  const sql = createSql(env);
  try {
    // AC2 safety: WHERE p.email IS NULL is the GDPR guard
    const abandoned = await sql<{ project_id: string }[]>`
      SELECT pp.id AS project_id
      FROM prospect_projects pp
      JOIN prospects p ON p.id = pp.prospect_id
      WHERE p.email IS NULL
        AND p.created_at < NOW() - INTERVAL '24 hours'
        AND (pp.freetext IS NOT NULL
          OR pp.requirements IS NOT NULL
          OR pp.extraction_confidence IS NOT NULL)
    `;

    if (abandoned.length === 0) return;

    const projectIds = abandoned.map(r => r.project_id);

    // AC1: soft purge — keep prospect row for aggregate analytics (DEC-129)
    await sql`
      UPDATE prospect_projects
      SET freetext = NULL, requirements = NULL, extraction_confidence = NULL
      WHERE id = ANY(${projectIds})
    `;

    // AC1: delete associated matches (no business value for unidentified prospects)
    await sql`DELETE FROM matches WHERE project_id = ANY(${projectIds})`;

    // AC5: audit trail
    await sql`
      INSERT INTO data_purge_log (purge_type, records_affected)
      VALUES ('abandoned_funnel', ${abandoned.length})
    `;

    // AC1: PostHog event — no PII
    await captureEvent(env, 'system', 'system.abandoned_funnel_purged', {
      count: abandoned.length
    });

    console.log(`purgeAbandonedFunnels: purged ${abandoned.length} abandoned projects`);
  } finally {
    await sql.end();
  }
}

async function cleanupOtpKvKeys(env: Env): Promise<void> {
  // AC3: scan SESSIONS KV for otp:* keys past their TTL
  const nowEpoch = Math.floor(Date.now() / 1000);
  const listed = await env.SESSIONS.list({ prefix: 'otp:' });

  if (!listed.list_complete) {
    console.warn('cleanupOtpKvKeys: list incomplete — volume may exceed 1000 keys');
  }

  const toDelete = listed.keys.filter(
    (k: { name: string; expiration?: number }) =>
      typeof k.expiration === 'number' && k.expiration < nowEpoch
  );

  for (const key of toDelete) {
    await env.SESSIONS.delete(key.name);
  }

  // AC5: audit trail (always insert, even on 0 deletions — proves cron ran)
  const sql = createSql(env);
  try {
    await sql`
      INSERT INTO data_purge_log (purge_type, records_affected)
      VALUES ('otp_kv_cleanup', ${toDelete.length})
    `;
  } finally {
    await sql.end();
  }

  if (toDelete.length > 0) {
    console.log(`cleanupOtpKvKeys: deleted ${toDelete.length} expired OTP KV keys`);
  }
}
```

## Edge Cases

| AC | Edge Case | Handling |
|----|-----------|---------|
| AC1 | No abandoned prospects found | Early return, no SQL writes, no audit log entry |
| AC1 | Already-purged prospects re-scanned | `AND (freetext IS NOT NULL OR ...)` guard prevents re-processing |
| AC2 | Prospect has email but no project data | `email IS NOT NULL` → excluded from query; not touched |
| AC3 | `otp:rate:{email}` keys appear in scan | Handled by TTL check — deleted if expired, retained if not |
| AC3 | Keys without expiration | Log warning, do not delete |
| AC3 | Volume > 1000 keys | Log warning, process available keys, defer cursor pagination |
| AC4 | French text only | Acceptable for Phase 0; EN variant deferred to i18n story |
| AC5 | Migration timestamp collision | Verify 20260228000009 is unused before apply |
