# E02S07 Execution Plan — Billing & Subscription Page

## Architecture Overview

Single-file route at `dashboard.billing.tsx` — same pattern as `dashboard.bookings.tsx` and `dashboard.settings.tsx`. One loader, one action with intent dispatch (`update_max_lead_price` / `update_spending_limit`), one default export component.

Key architectural decisions:
- No zod in the route file. Simple `parseFloat()` + guard checks.
- Lemon.js is client-side only, loaded via `useEffect` script injection.
- Checkout URL comes from `env.LS_CHECKOUT_URL` (optional, button disabled if null).
- Pagination via URL search params `?page=N` — `<Link>` navigation, no client state.
- Two action intents: `update_max_lead_price` / `update_spending_limit`.
- AC10 explicitly deferred (backend gap).
- Transaction date formatting done server-side in loader (SSR-safe).

### Step 4: Create `dashboard.billing.tsx`

**Imports:** Form, Link, useActionData, useLoaderData, useSearchParams from react-router; useState, useEffect from react; requireSession, apiGet, apiPatch, ApiError, captureEvent, toast; shadcn/ui: Button, Badge, Card/CardContent/CardHeader/CardTitle, Input, Label, Progress, Separator, Table/TableBody/TableCell/TableHead/TableHeader/TableRow.

**TypeScript globals:** `declare global { interface Window { createLemonSqueezy?: () => void; LemonSqueezy?: { Setup: (config: { eventHandler: (event: { event: string }) => void }) => void; openUrl: (url: string) => void; }; } }`

**Types:**
```
Transaction: { id, type, amount_cents, description, lead_id, created_at, formatted_date }
BillingResponse: { credit_balance, ls_subscription_status, ls_subscription_id, spending_limit, max_lead_price, monthly_spend, transactions (raw), total_transactions, page, per_page }
LoaderData: { billing: BillingResponse & { transactions: Transaction[] }, userId, lsCheckoutUrl: string | null }
ActionData: discriminated union on intent + success
```

**Helpers:**
- `formatTransactionDate(isoString: string | null): string` — fr-FR UTC date
- `centsToEur(cents: number): string` — divide by 100, toFixed(2)
- `buildPageUrl(page: number): string` — `/dashboard/billing?page=N`

**Loader (AC1, AC4, AC5, AC6):**
- requireSession → apiGet billing with page/per_page params
- Fallback on error: zeroed BillingResponse
- Pre-format transaction dates
- captureEvent `expert.billing_viewed`
- Response.json with lsCheckoutUrl from env

**Action (AC7, AC8, AC9):**
- intent `update_max_lead_price`: parse EUR → centimes → apiPatch profile → captureEvent `expert.spending_control_updated` with control_name + old_value + new_value
- intent `update_spending_limit`: same pattern
- old values passed as hidden form fields
- 422 on invalid/negative input, 500 on API error

**Component JSX structure:**
1. Page header: h1 "Facturation"
2. Subscription card (AC2/AC3): CTA if no ls_subscription_id, status badge if has
3. Credit balance card (AC4): large "{centsToEur}€", green/red based on value
4. Monthly spending card (AC5): progress bar if spending_limit set, else spend-only text
5. Spending controls card (AC7/AC8): two Forms with hidden old values
6. Transaction history card (AC6): Table + pagination or empty state
7. AC10 deferred comment

**Lemon.js useEffect:** Script injection, createLemonSqueezy, Setup with Checkout.Success → window.location.reload().

**Input initialization:** `useState(billing.max_lead_price !== null ? String(billing.max_lead_price / 100) : "")` directly (no useEffect needed).

**useEffect for actionData:** toast.success on success, toast.error on failure.

## Test Plan (12 tests)

## Edge Cases

| AC | Edge case | Handling |
|---|---|---|
| AC2 | lsCheckoutUrl null | Button disabled |
| AC2 | Lemon.js fails to load | openUrl never called (optional chaining), no crash |
| AC3 | Unknown subscription status | Gray badge with raw status string |
| AC4 | credit_balance = 0 or negative | Red color |
| AC5 | spending_limit null | No progress bar, only monthly_spend text |
| AC5 | monthly_spend > spending_limit | Clamp to 100%, red critical warning |
| AC6 | transactions empty | Empty state text |
| AC6 | created_at null | formatTransactionDate returns "—" |
| AC7/AC8 | Empty input | parseFloat("") = NaN → 422 |
| AC7/AC8 | Value = 0 | Valid, passes through |
| AC11 | Window global type | declare global block in file |
