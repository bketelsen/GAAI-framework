# E03S04 Execution Plan — Expert Directory + Profile Pages

## Complexity Assessment

**Tier:** 2 (complexity 6 — multiple new pages, async SSR, two worker touchpoints, SEO, tracking)
**Files affected:** 6 new + 4 modified
**Criteria count:** 11 ACs (AC1–AC11)

## Scope

Two server-rendered pages on the satellite Worker + one new Core API endpoint:

1. `GET /experts` — expert directory page (AC1–AC3, AC6–AC9)
2. `GET /experts/:slug` — expert profile detail page (AC4, AC7, AC8, AC11)
3. `POST /api/prospects/create-from-directory` — lightweight prospect creation (AC4 unlock flow)
4. Sitemap extension — expert slugs added to `/sitemap.xml` (AC7)

## Implementation Plan

### Step 1 — experts-directory.ts (satellite Worker)

New file: `workers/frontend/satellites/src/pages/experts-directory.ts`

- `async function renderExpertsDirectoryPage(config, posthogApiKey, coreApiUrl): Promise<string>`
- Fetches `GET /api/experts/public?vertical={config.vertical}&per_page=12&page=1` server-side
- Renders: branded header + nav, headline "{N} experts en {vertical_label}", filter bar (skill toggles from returned experts), 3-col card grid, load-more button, JSON-LD ItemList, PostHog events (directory_viewed, filter_applied)
- Empty state when `experts.length === 0` (AC9)
- Client-side JS: `filterCards()` by skill, fetch next page on "Charger plus" (AC6)
- SEO: `<meta name="robots" content="index, follow">`, canonical, title

### Step 2 — experts-profile.ts (satellite Worker)

New file: `workers/frontend/satellites/src/pages/experts-profile.ts`

- `async function renderExpertProfilePage(config, posthogApiKey, coreApiUrl, slug): Promise<{ html: string; status: number }>`
- Fetches `GET /api/experts/public/{slug}` server-side; returns `{ status: 404 }` on 404
- Renders: anonymized avatar (initial + color by tier), tier badge, composite score, headline, availability indicator ("Disponible" / "Disponible prochainement"), bio excerpt, skills/industries tags, rate range, completed projects, languages
- Primary CTA: "Vérifier la compatibilité" → /match
- Secondary CTA: "Débloquer ce profil" → inline email form → `POST /api/prospects/create-from-directory` → fires `booking-open` CustomEvent
- AC11: Direct booking CTA conditional on `expert.direct_booking_url ?? null` (hidden if null)
- JSON-LD Person, PostHog events, XSS protection in JSON-LD

### Step 3 — Core API: create-from-directory

Modified: `workers/backend/api/src/routes/prospects.ts` + `workers/backend/api/src/index.ts`

- `async function handleCreateFromDirectory(request, env, ctx): Promise<Response>`
- Body: `{ email: string, expert_slug: string }`
- Validates email regex, slug format (`exp-{8+ alphanumeric}`)
- Looks up expert by slug prefix: `SELECT id FROM experts WHERE LEFT(id::text, 8) = $1`
- Creates minimal prospect: `INSERT INTO prospects (satellite_id, email, status) VALUES (NULL, $1, 'identified') RETURNING id`
- Signs `prospect:identify` token, returns `{ prospect_id, token, expert_id }`
- Route: `POST /api/prospects/create-from-directory`

### Step 4 — Sitemap extension

Modified: `workers/frontend/satellites/src/pages/sitemap.ts`

- Changed from sync to async: `async function renderSitemapXml(config, coreApiUrl?): Promise<string>`
- Added `fetchExpertSlugs(coreApiUrl, vertical)` — fetches `/api/experts/public?per_page=50` slugs
- Appends `<url><loc>/experts/{slug}</loc></url>` entries

### Step 5 — Route registration

Modified: `workers/frontend/satellites/src/index.ts`

- Import `renderExpertsDirectoryPage`, `renderExpertProfilePage`
- `/experts` route: async, `Cache-Control: public, max-age=300, stale-while-revalidate=3600`
- `/experts/:slug` route: async, 404 handled with noindex inline HTML, 200 with 300s SWR cache
- Sitemap route made async

### Step 6 — Tests

New: `src/pages/experts-directory.test.ts` (20 tests)
New: `src/pages/experts-profile.test.ts` (30 tests)

Using `vi.stubGlobal('fetch', ...)` pattern consistent with project.

## Key Decisions

- **DEC-131** (existing): `/experts/:slug` = anonymized+indexed satellite pages. `callibrate.io/book/:slug?t={token}` = direct mode (E02S12 scope). AC11 CTA conditional on `direct_booking_url` field being non-null.
- **Prospect creation path**: New `create-from-directory` endpoint preferred over reusing quiz flow — quiz context not available here, and AC4 explicitly notes "lightweight prospect creation" semantics.
- **AC11 field**: `direct_booking_url` not yet populated by Core API (DB column added by E02S12). CTA hidden (`null`) until E02S12 delivers. No code change required when E02S12 lands — API field presence auto-enables the CTA.
- **Sitemap**: Made async with optional `coreApiUrl` parameter. Existing tests pass because `coreApiUrl` is optional (undefined → no expert slugs fetched).
