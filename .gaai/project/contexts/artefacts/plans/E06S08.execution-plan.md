## Implementation Summary

### Validation (AC1)
- `text` required, string, max 2000 chars → 422 on failure
- Invalid JSON → 400

### Satellite context (AC9)
- Optional `satellite_id` → queries `satellite_configs.vertical` from Supabase
- Non-blocking: errors proceed without context

### Claude API call (AC2, AC8)
- Model: `claude-haiku-4-5-20251001`
- Tool use: `extract_requirements` tool with JSON schema for all 7 fields + confidence + questions
- `tool_choice: { type: 'tool', name: 'extract_requirements' }` enforces structured output
- URL: `${CLOUDFLARE_AI_GATEWAY_URL}/v1/messages` (AI Gateway routing)
- Auth: `x-api-key: ${ANTHROPIC_API_KEY}`, `anthropic-version: 2023-06-01`

### Response computation (AC3–AC7)
- `needs_confirmation = fields where confidence < 0.7`
- `ready_to_match = needs_confirmation.length === 0`
- `confirmation_questions`: LLM-generated, filtered to low-confidence fields, max 3 (AC5)
- Omitted when `ready_to_match = true` (AC6)
- Stateless — no DB writes (AC7)

### Test coverage (AC10)
- 8 vitest tests covering AC1, AC2, AC3, AC5, AC6, AC8, AC10
- Mocked fetch (vi.stubGlobal) — no live API calls
