# E02S05 Execution Plan — Lead Pipeline Page + Actions

## Story Reference
**E02S05** — Lead Pipeline Page + Actions
**Status target:** `done`
**Files to produce:** 2 (route + test)

---

## 1. Architecture Overview

Single-file route pattern (same as `dashboard.gcal.tsx`, `dashboard.settings.tsx`). Loader + action + React component in one file. No sub-routes. Lead detail opens in a Dialog (no Sheet component available). URL search params are source of truth for filters and pagination.

Key design decisions:
- `useSearchParams` drives status filter and page (read-only from URL, written via `<Link>`)
- Loader: `requireSession` → `apiGet` → `Response.json(data, { headers: responseHeaders })`
- Action: `requireSession` → `formData.get('intent')` → switch on `confirm` / `flag` / `evaluate`
- Two Dialogs: one for lead detail (row click), one for flag reason form
- `useEffect` watches `actionData` for toast trigger
- PostHog events fired fire-and-forget with `.catch(() => {})`
- Test file: `tests/leads-action.test.ts` (matches naming convention)

### 2.3 Loader

```typescript
export async function loader({ request, context }: LoaderFunctionArgs): Promise<Response>
```

1. `requireSession(request, context.cloudflare.env)`
2. Read URL params: `status` (default `"all"`), `page` (default `"1"`)
3. Call `apiGet<LeadsResponse>(env, session.token, `/api/experts/${userId}/leads`, { status, page, per_page: "20" })`
4. Wrap in `.catch(() => ({ leads: [], total: 0, page: 1, per_page: 20 }))` — graceful degradation
5. Fire `captureEvent(env, `expert:${userId}`, "expert.leads_viewed", { status_filter: status }).catch(() => {})` — fire-and-forget, do NOT await
6. Return `Response.json({ leads, total, page, per_page, userId } satisfies LoaderData, { headers: responseHeaders })`

### 2.4 Action

```typescript
export async function action({ request, context }: ActionFunctionArgs): Promise<Response>
```

Switch on `formData.get("intent")`:

**case "confirm":**
- `leadId = String(formData.get("leadId") ?? "")`
- if empty → 400
- `apiPost<{ success: true }>(env, token, `/api/leads/${leadId}/confirm`, {})`
- `captureEvent(env, `expert:${userId}`, "expert.lead_confirmed", { lead_id: leadId, price_cents: Number(formData.get("price_cents")) || 0 }).catch(() => {})`
- return `Response.json({ success: true, intent: "confirm", leadId } satisfies ActionData)`
- catch ApiError → use `err.status`; else 500

**case "flag":**
- `leadId`, `reason = String(formData.get("flag_reason") ?? "").trim()`
- if empty leadId → 400
- if `reason.length < 20` → 422 `{ error: "La raison doit contenir au moins 20 caractères." }`
- `apiPost(env, token, `/api/leads/${leadId}/flag`, { reason })`
- `captureEvent(env, `expert:${userId}`, "expert.lead_flagged", { lead_id: leadId, reason_length: reason.length }).catch(() => {})`
- return success
- catch → 500

**case "evaluate":**
- `leadId`, `scoreRaw = Number(formData.get("score"))`, `notes = String(formData.get("notes") ?? "").trim()`, `conversion_declared = formData.get("conversion_declared") === "true"`
- if empty leadId → 400
- if `isNaN(scoreRaw) || scoreRaw < 1 || scoreRaw > 10` → 422 `{ error: "Score invalide (1–10 requis)." }`
- if `notes.length > 500` → 422 `{ error: "Notes limitées à 500 caractères." }`
- `apiPost(env, token, `/api/leads/${leadId}/evaluate`, { score: scoreRaw, notes: notes || undefined, conversion_declared })`
- `captureEvent(env, `expert:${userId}`, "expert.lead_evaluated", { lead_id: leadId, score: scoreRaw, conversion_declared }).catch(() => {})`
- return success
- catch → 500

**default:** → 400 `{ success: false, intent: "unknown", error: "Action inconnue." }`

## 5. Potential Pitfalls

- **Dialog open/close**: `open={!!selectedLead && !flagDialogOpen}` — flag dialog takes visual precedence
- **stopPropagation**: Action buttons inside `<TableRow onClick>` must call `e.stopPropagation()`
- **per_page as string**: `apiGet` params are `Record<string, string>` — pass `"20"` not `20`
- **score as number**: `formData.get("score")` → `Number()` → validate range
- **Test env**: `environment: "node"` in vitest config — no JSX rendering in tests
