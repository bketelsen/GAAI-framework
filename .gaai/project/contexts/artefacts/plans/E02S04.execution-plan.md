## Codebase Scan Findings

From reading existing implementation:

1. **`api.server.ts`** — supports GET, POST, PATCH only. Need to add `apiDelete` for the disconnect endpoint.
2. **Disconnect endpoint** — `DELETE /api/experts/:id/gcal/disconnect` (story AC6 says POST, but actual API is DELETE).
3. **Auth URL endpoint** — `GET /api/experts/:id/gcal/auth-url` returns `{ auth_url: string }` (not `authUrl`).
4. **OAuth callback redirect** — E06S10 callback redirects to `/onboarding/gcal-connected` (not `/dashboard/gcal`). Out of scope to modify E06S10. See TD-7.
5. **Availability endpoint** — `GET /api/experts/:id/availability` (story says `/bookings/availability`, actual route is `/availability`). Returns `{ slots: Array<{start: string, end: string}>, metadata: {...} }`.
6. **Availability access** — CORS-gated, no JWT. Server-to-server calls from dashboard loader are allowed (no Origin header = allow per CORS convention).
7. **GCal status response** — `{ connected: boolean, google_email: string | null, connected_at: string | null }`.

## Technical Decisions

- **TD-1: Route path** — `/dashboard/gcal` (matching the link set in E02S03's GcalSection read-only card).
- **TD-2: Connect flow** — Server action fetches `auth_url`, fires PostHog `expert.gcal_connect_started`, then returns redirect to OAuth URL. After OAuth, E06S10 redirects to `/onboarding/gcal-connected` (see TD-7).
- **TD-3: Disconnect method** — `DELETE`. Add `apiDelete` to `api.server.ts`. Method: `"DELETE"` added to `apiFetch` method type union.
- **TD-4: Availability route** — `GET /api/experts/:id/availability` (corrected from story's `/bookings/availability`). Loader fetches only when `gcalStatus.connected === true`. Graceful 422/502 → `null` slots.
- **TD-5: Success toast (AC5)** — Since E06S10 callback redirects to `/onboarding/gcal-connected` (not back to `/dashboard/gcal`), use `sessionStorage` flag pattern: connect action sets `sessionStorage.setItem('gcalPendingConnect', '1')` via `useEffect` on `connectTriggered` loader state. On mount, if flag present + `connected === true` → show toast + clear flag. Implementation via a `useEffect` in the React component.
- **TD-6: PostHog events** — `expert.gcal_connect_started` fired server-side in the `connect` action (before redirect). `expert.gcal_disconnected` fired server-side in the `disconnect` action (after successful DELETE). `expert.gcal_connected` is already fired by E06S10 backend callback — not duplicated here.
- **TD-7: OAuth callback behavior note** — E06S10 routes callback to `/onboarding/gcal-connected`. After connecting GCal from the dashboard (post-onboarding), the user will land on the onboarding success page. This is an existing limitation of E06S10 (out of scope). When user navigates back to `/dashboard/gcal`, the connected state will be visible.
- **TD-8: Disconnect confirmation dialog** — Use shadcn `Dialog` component (already in `components/ui/dialog.tsx`). Controlled state via `useState`.
- **TD-9: Availability refresh button** — Uses `useFetcher` with `load("/dashboard/gcal")` to reload the loader without full navigation. Alternatively, `window.location.reload()`. Use `useFetcher` for cleaner UX.
- **TD-10: Availability slots** — `slots.slice(0, 5)` to show first 5. Display each slot as locale date + time range.
- **TD-11: Test approach** — Mirror `settings-action.test.ts` pattern. Test loader (connected state, disconnected state, availability fetch). Test disconnect action (success, API error). Test connect action (redirect to auth_url).

### Step 1 — `api.server.ts`: Add `apiDelete`

Add `"DELETE"` to the method type union in `apiFetch`, and export `apiDelete`:

```typescript
async function apiFetch<T>(
  env: Env,
  token: string,
  method: "GET" | "POST" | "PATCH" | "DELETE",
  path: string,
  params?: Record<string, string>,
  body?: unknown,
): Promise<T>

export function apiDelete<T>(
  env: Env,
  token: string,
  path: string,
): Promise<T> {
  return apiFetch<T>(env, token, "DELETE", path, undefined, undefined);
}
```

### Step 4 — `dashboard.gcal.tsx`: Loader

```typescript
export async function loader({ request, context }: LoaderFunctionArgs) {
  const { session, responseHeaders } = await requireSession(request, context.cloudflare.env);
  const env = context.cloudflare.env;
  const userId = session.user.id;

  const gcalStatus = await apiGet<GcalStatus>(
    env, session.token, `/api/experts/${userId}/gcal/status`
  ).catch(() => null);

  let slots: AvailabilitySlot[] | null = null;
  let generatedAt: string | null = null;

  if (gcalStatus?.connected) {
    try {
      const avail = await apiGet<AvailabilityResponse>(
        env, session.token, `/api/experts/${userId}/availability`
      );
      slots = avail.slots.slice(0, 5);
      generatedAt = avail.metadata.generated_at;
    } catch {
      // Non-blocking — slots unavailable but page still renders
    }
  }

  return Response.json(
    { gcalStatus, slots, generatedAt, userId },
    { headers: responseHeaders },
  );
}
```

### Step 5 — `dashboard.gcal.tsx`: Action

The action handles two intents: `"connect"` and `"disconnect"`.

```typescript
export async function action({ request, context }: ActionFunctionArgs) {
  const { session } = await requireSession(request, context.cloudflare.env);
  const env = context.cloudflare.env;
  const userId = session.user.id;

  const formData = await request.formData();
  const intent = String(formData.get("intent") ?? "");

  if (intent === "connect") {
    try {
      const { auth_url } = await apiGet<{ auth_url: string }>(
        env, session.token, `/api/experts/${userId}/gcal/auth-url`
      );
      captureEvent(env, `expert:${userId}`, "expert.gcal_connect_started", {}).catch(() => {});
      return redirect(auth_url);
    } catch {
      return Response.json({ success: false, action: "connect", error: "Impossible de démarrer la connexion." }, { status: 500 });
    }
  }

  if (intent === "disconnect") {
    try {
      await apiDelete(env, session.token, `/api/experts/${userId}/gcal/disconnect`);
      captureEvent(env, `expert:${userId}`, "expert.gcal_disconnected", {}).catch(() => {});
      return Response.json({ success: true, action: "disconnect" });
    } catch (err) {
      const status = err instanceof ApiError ? err.status : 500;
      return Response.json(
        { success: false, action: "disconnect", error: "Impossible de déconnecter le calendrier." },
        { status }
      );
    }
  }

  return Response.json({ success: false, action: "disconnect", error: "Action inconnue." }, { status: 400 });
}
```

### Step 6 — `dashboard.gcal.tsx`: Page component

Main component with:
1. Status badge (AC2/AC3)
2. Connect button (AC3/AC4) — Form method="post" with intent="connect"
3. Disconnect button (AC2) — opens Dialog for confirmation
4. Availability preview (AC7/AC8)
5. Refresh button for availability (AC7) — useFetcher
6. `useEffect` for:
   - AC5 success toast: detect sessionStorage flag + connected state
   - AC6 success toast: actionData.success + action="disconnect"
   - Error toast: actionData.success===false

```typescript
export default function GcalPage() {
  const { gcalStatus, slots, generatedAt, userId } = useLoaderData<LoaderData>();
  const actionData = useActionData() as ActionData | undefined;
  const fetcher = useFetcher();
  const [disconnectOpen, setDisconnectOpen] = useState(false);

  // AC5: Detect return from OAuth connect (sessionStorage flash)
  useEffect(() => {
    if (typeof window === "undefined") return;
    const pending = sessionStorage.getItem("gcalPendingConnect");
    if (pending && gcalStatus?.connected) {
      toast.success("Google Calendar connecté avec succès");
      sessionStorage.removeItem("gcalPendingConnect");
    }
  }, [gcalStatus?.connected]);

  // Set sessionStorage flag when connect action is initiated
  // This is done via a data attribute on the form submit button, handled in onClick
  // The form action will redirect the page — we set the flag synchronously before form submit

  // AC6 + error toasts from actionData
  useEffect(() => {
    if (!actionData) return;
    if (actionData.success && actionData.action === "disconnect") {
      toast.success("Google Calendar déconnecté");
      setDisconnectOpen(false);
    } else if (!actionData.success) {
      toast.error(actionData.error);
    }
  }, [actionData]);

  const refreshingSlots = fetcher.state !== "idle";

  return (
    <div className="space-y-6">
      <div>
        <h1 className="text-2xl font-semibold">Google Calendar</h1>
        <p className="text-muted-foreground mt-1">
          Connectez votre calendrier pour recevoir des réservations automatiques.
        </p>
      </div>

      <Card>
        <CardHeader>
          <CardTitle>Statut de connexion</CardTitle>
          <CardDescription>
            Votre calendrier Google est utilisé pour afficher vos disponibilités aux prospects.
          </CardDescription>
        </CardHeader>
        <CardContent className="space-y-4">
          {gcalStatus?.connected ? (
            <>
              <div className="flex items-center gap-3">
                <Badge variant="default" className="bg-green-600 hover:bg-green-600">
                  Connecté
                </Badge>
                <span className="text-sm text-muted-foreground">{gcalStatus.google_email}</span>
              </div>
              <Button
                variant="destructive"
                onClick={() => setDisconnectOpen(true)}
                type="button"
              >
                Déconnecter
              </Button>
            </>
          ) : (
            <>
              <div className="flex items-center gap-3">
                <Badge variant="outline" className="border-yellow-500 text-yellow-600">
                  Non connecté
                </Badge>
              </div>
              <p className="text-sm text-muted-foreground">
                Connectez votre calendrier Google pour afficher vos disponibilités et recevoir des réservations automatiques.
              </p>
              <Form method="post">
                <input type="hidden" name="intent" value="connect" />
                <Button
                  type="submit"
                  onClick={() => {
                    if (typeof window !== "undefined") {
                      sessionStorage.setItem("gcalPendingConnect", "1");
                    }
                  }}
                >
                  Connecter Google Calendar
                </Button>
              </Form>
            </>
          )}
        </CardContent>
      </Card>

      {gcalStatus?.connected && (
        <Card>
          <CardHeader>
            <div className="flex items-center justify-between">
              <div>
                <CardTitle>Prochains créneaux disponibles</CardTitle>
                {generatedAt && (
                  <CardDescription>
                    Actualisé à{" "}
                    {new Date(generatedAt).toLocaleTimeString("fr-FR", { hour: "2-digit", minute: "2-digit" })}
                  </CardDescription>
                )}
              </div>
              <fetcher.Form method="get" action="/dashboard/gcal">
                <Button variant="outline" size="sm" type="submit" disabled={refreshingSlots}>
                  {refreshingSlots ? "Actualisation..." : "Actualiser"}
                </Button>
              </fetcher.Form>
            </div>
          </CardHeader>
          <CardContent>
            {slots && slots.length > 0 ? (
              <ul className="space-y-2">
                {slots.map((slot) => (
                  <li key={slot.start} className="text-sm flex gap-2">
                    <span className="font-medium">
                      {new Date(slot.start).toLocaleDateString("fr-FR", { weekday: "long", month: "long", day: "numeric" })}
                    </span>
                    <span className="text-muted-foreground">
                      {new Date(slot.start).toLocaleTimeString("fr-FR", { hour: "2-digit", minute: "2-digit" })}–
                      {new Date(slot.end).toLocaleTimeString("fr-FR", { hour: "2-digit", minute: "2-digit" })}
                    </span>
                  </li>
                ))}
              </ul>
            ) : (
              <p className="text-sm text-muted-foreground">
                Aucun créneau disponible dans les 7 prochains jours. Vérifiez votre agenda Google.
              </p>
            )}
          </CardContent>
        </Card>
      )}

      {/* Disconnect confirmation dialog */}
      <Dialog open={disconnectOpen} onOpenChange={setDisconnectOpen}>
        <DialogContent>
          <DialogHeader>
            <DialogTitle>Déconnecter Google Calendar</DialogTitle>
            <DialogDescription>
              Êtes-vous sûr de vouloir déconnecter votre Google Calendar ? Les prospects ne pourront plus réserver de créneaux sur votre calendrier.
            </DialogDescription>
          </DialogHeader>
          <DialogFooter>
            <Button variant="outline" onClick={() => setDisconnectOpen(false)}>
              Annuler
            </Button>
            <Form method="post">
              <input type="hidden" name="intent" value="disconnect" />
              <Button variant="destructive" type="submit">
                Déconnecter
              </Button>
            </Form>
          </DialogFooter>
        </DialogContent>
      </Dialog>
    </div>
  );
}
```
