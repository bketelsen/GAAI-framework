# Execution Plan — E07S01: PostHog Reverse Proxy CF Worker

## Tier

Tier 2 (complexity 3 — new worker scaffold, simple proxy logic, config updates)

## Approach

PostHog's officially documented CF Worker reverse proxy pattern. Approach explicitly specified in the Story — no approach evaluation required. Hono already established in the project (E06S14 satellite worker).

## Implementation Sequence

1. `workers/posthog-proxy/package.json` — dependencies: hono, wrangler, vitest, typescript, @cloudflare/workers-types

## Key Design Decisions

- **Proxy pattern:** `proxyTo(targetBase, request)` — rewrites Host header, buffers body with `arrayBuffer()` (avoids ReadableStream duplex issues), preserves path + query string
- **CORS:** Hono `cors()` middleware with `origin` function checking `hostname === 'callibrate.io' || hostname.endsWith('.callibrate.io')`. No Origin header → `null` (CORS not relevant for server-to-server)
- **No secrets:** Proxy is transparent pass-through. PostHog API key travels in client payload — never stored in Worker (AC6)
- **CF zone routes:** `ph.callibrate.io/*` and `ph.staging.callibrate.io/*` defined in wrangler.toml `[[env.*.routes]]`
- **Worker naming:** `callibrate-posthog-proxy-{dev/staging/prod}` per DEC-32 convention

## Edge Cases

- Query strings preserved: `new URL(pathname + search, targetBase)`
- OPTIONS preflight: handled by Hono cors middleware (returns 204 with ACAO, no fetch call)
- Unknown routes: 404 from Hono default handler
- Non-callibrate.io origins: no ACAO header (browser blocks, Worker still runs — no secrets to protect)
- Proxy errors (PostHog down): upstream status code passed through transparently

## Risk Register

- Low: Route deployment requires CF API token scope to include `callibrate-posthog-proxy*` — documented in SETUP.md
- Low: DNS record needed for `ph.callibrate.io` — documented as human action
