## Implementation Sequence

### Step 1 — Create migration file

**File:** `supabase/migrations/20260223000001_satellite_configs_brand_content_theme.sql`

```sql
ALTER TABLE satellite_configs
  ADD COLUMN theme           JSONB,
  ADD COLUMN brand           JSONB,
  ADD COLUMN content         JSONB,
  ADD COLUMN structured_data JSONB,
  ADD COLUMN active          BOOLEAN NOT NULL DEFAULT false,
  ADD COLUMN updated_at      TIMESTAMPTZ DEFAULT now();
```

Column comments documenting canonical JSONB shapes (developer reference — not DB constraints per Story notes).

**Checkpoint:** Migration file valid SQL, all 6 columns correct types.

### Step 2 — Seed update (in same migration file)

```sql
UPDATE satellite_configs
SET
  theme = '{"primary": "#4F46E5", "accent": "#818CF8", "font": "Inter, sans-serif", "radius": "0.5rem", "logo_url": null}'::jsonb,
  brand = '{"name": "Callibrate", "tagline": "..."}'::jsonb,
  content = '{"meta_title": "...", "meta_description": "...", "hero_headline": "...", "hero_sub": "...", "value_props": [...], "vertical_label": "...", "vertical_description": "..."}'::jsonb,
  structured_data = '{"@context": "https://schema.org", "@type": "ProfessionalService", ...}'::jsonb,
  active = false,
  updated_at = now()
WHERE id = 'default';
```

**Checkpoint:** UPDATE targets WHERE id = 'default'. All 7 content fields present. active = false.

### Step 3 — Apply migration to Supabase staging

`mcp__supabase__apply_migration` with the SQL from Step 1+2.

**Checkpoint:** Migration applies without error. Existing rows + columns preserved (AC7).

### Step 4 — Regenerate TypeScript types

`mcp__supabase__generate_typescript_types` → write result to `src/types/database.ts`.

**Checkpoint:** `satellite_configs.Row` reflects all 6 new columns with correct types.

### Step 5 — Commit

Atomic commit: migration file + updated `database.ts`.

## Edge Cases

| AC | Edge Case | Mitigation |
|----|-----------|------------|
| AC1 | `active BOOLEAN NOT NULL DEFAULT false` — NOT NULL requires DEFAULT | DEFAULT false provided → safe for existing rows |
| AC1 | `updated_at` nullable — no trigger required | Set to `now()` via seed UPDATE; application-managed thereafter (per Story notes) |
| AC6 | Seed row `id = 'default'` may not exist if DB was reset | If UPDATE affects 0 rows, migration still succeeds (no error); operator responsible for seed presence |
| AC7 | ADD COLUMN on non-empty table | PostgreSQL ADD COLUMN is non-locking for nullable cols; active has DEFAULT so also safe |
| AC8 | Type generator returns all columns | Verify `satellite_configs.Row` has all 6 new fields before committing |
