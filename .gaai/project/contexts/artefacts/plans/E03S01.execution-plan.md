# Execution Plan — E03S01: Prospect Freetext Input Page (/match)

## Tier Assessment

**Tier 2 — Core Team (Planning + Implementation + QA)**
- Complexity: 4 (10 ACs, 1 new file, 3 file modifications, client-side JS, CSP update)
- Files affected: 6 (1 new, 4 modified, 1 test)
- No new infrastructure, no DB changes, no migration
- Risk: Low-Medium (CSP update affects all satellite pages; client-side JS state machine is primary complexity)

## Approach Rationale

Follows the exact conventions established by `landing.ts` and validated across E06S14. All rendering is server-side HTML template literals. All client-side logic is vanilla JS embedded in a `<script>` tag at the bottom of `<body>` — no external JS dependencies, no bundler. CORE_API_URL is injected at render time as `window.__SAT__ = { apiUrl, satelliteId }` (inline script variable, not `data-api-url` attribute — preferred for immediate availability without DOM traversal). sessionStorage used for AC10 (correct over history.state since transition to /confirm is via window.location.href).

## Implementation Sequence

### Step 1 — Modify `workers/frontend/satellites/src/types/env.ts`
Add `CORE_API_URL: string` to the `Env` interface after `POSTHOG_API_KEY`. Done first because match.ts and index.ts both depend on the type.

**Checkpoint:** `npx tsc --noEmit` passes.

### Step 2 — Modify `workers/frontend/satellites/wrangler.toml`
Add `CORE_API_URL` as a `[vars]` entry for each environment:
- Root (dev): `[vars]` block with `CORE_API_URL = "https://callibrate-core-staging.frederic-geens-consulting.workers.dev"`
- `[env.staging.vars]`: `CORE_API_URL = "https://callibrate-core-staging.frederic-geens-consulting.workers.dev"`
- `[env.production.vars]`: `CORE_API_URL = "https://api.callibrate.io"`

Note: `CORE_API_URL` is not a secret — it is a public URL, safe to commit in wrangler.toml.

### Step 3 — Create `workers/frontend/satellites/src/pages/match.ts`
Export `renderMatchPage(config: SatelliteConfig, posthogApiKey: string, coreApiUrl: string): string`.

#### 3a — Head section
- `<title>`: `${brand?.name ?? 'Callibrate'} — Décrivez votre projet` (no match-specific content field in SatelliteContent)
- `<meta name="robots" content="index, follow">`
- `<link rel="canonical" href="https://${config.domain}/match">`
- OG tags: og:title, og:description (from config.content?.meta_description), og:url (`https://${config.domain}/match`), og:type=website
- JSON-LD: WebPage structured data (url, name, description from config) — always emitted, independent of config.structured_data
- PostHog head snippet: same minified snippet as landing.ts, gated by `config.tracking_enabled !== false && posthogApiKey`
- CSS vars block: same `:root` pattern from config.theme

#### 3b — Inline config script (before </body>)
`window.__SAT__ = { apiUrl: {coreApiUrl JSON-stringified}, satelliteId: {config.id JSON-stringified} }` — both values escaped with `.replace(/</g, '\\u003c')`.

#### 3c — Form HTML (body)
```
<main class="container">
  [logo if theme?.logo_url]
  <h1>[brand name]</h1>
  <h2>[fallback "Décrivez votre projet"]</h2>
  <form id="match-form">
    <textarea id="match-text" maxlength="2000" placeholder="..." rows="6" aria-describedby="char-counter match-error"></textarea>
    <div id="char-counter" aria-live="polite">0/2000</div>
    <div id="match-error" role="alert" aria-live="assertive" style="display:none"></div>
    <button type="submit" id="submit-btn" disabled>Analyser mon projet</button>
  </form>
  <div id="loading-msg" style="display:none" aria-live="polite">Analyse en cours...</div>
</main>
```

#### 3d — Client-side JS IIFE (body script)
Six concerns handled in one IIFE:
1. **sessionStorage restore (AC10)**: on DOMContentLoaded, read `match:text` from sessionStorage; restore textarea value + trigger counter update. Wrapped in try/catch (private browsing graceful degradation).
2. **Character counter (AC2)**: on `input` event — update counter text, toggle `counter--warn` class at >1800, enable/disable submit button based on 30≤len≤2000.
3. **Form validation on submit (AC2)**: preventDefault always; show inline error if <30 chars; proceed otherwise.
4. **Loading state (AC4)**: on valid submit — `textarea.readOnly = true`, `submitBtn.disabled = true`, show spinner in button, show `#loading-msg`, hide prior error. Store text in sessionStorage (covers AC10 on failure/back).
5. **API call + success (AC3, AC5, AC7)**: `fetch(window.__SAT__.apiUrl + '/api/extract', { method:'POST', headers:{'Content-Type':'application/json'}, body: JSON.stringify({text, satellite_id}) })`. On ok: store full response in `sessionStorage.setItem('match:extraction', JSON.stringify(data))`, fire `satellite.extraction_completed`, navigate to `/confirm`.
6. **Error handling (AC6)**: on catch/non-ok — log to console, fire `satellite.extraction_error`, restore form (readOnly=false, re-enable button, restore button text), hide loading, show error div.
7. **PostHog events (AC7)**: `satellite.match_page_viewed` on DOMContentLoaded; `satellite.match_form_submitted` on valid submit before fetch. All PostHog calls guarded by `typeof posthog !== 'undefined'`.

#### 3e — CSS
Key values:
- `textarea#match-text`: `width:100%`, `min-height:120px`, `max-height:400px`, `resize:vertical`, `overflow-y:auto`, border using `--color-primary`, `border-radius: var(--radius-card, 0.5rem)`, `padding:0.75rem`
- `#char-counter`: right-aligned, `font-size:0.875rem`, `color:#888`, `margin-top:0.25rem`
- `#char-counter.counter--warn`: `color:#dc2626`
- `#match-error`: `color:#dc2626`, `font-size:0.875rem`, error background tint, hidden by default
- `button#submit-btn`: same as `.cta` on landing — `background: var(--color-primary, #4F46E5)`, `color:#fff`, `padding:0.875rem 2rem`, `border-radius: var(--radius-card, 0.5rem)`, `font-size:1.0625rem`, `font-weight:600`, `min-height:44px` (AC9), `width:100%` for mobile
- `button:disabled`: `opacity:0.5`, `cursor:not-allowed`
- Spinner: `@keyframes spin` + inline SVG in button during loading
- Mobile `@media (max-width: 640px)`: reduce h2 font-size, reduce container padding

**Checkpoint:** `npx tsc --noEmit` passes after Step 3.

### Step 4 — Modify `workers/frontend/satellites/src/index.ts`
- Import: `import { renderMatchPage } from './pages/match';`
- Route: after landing page route, before `export default`:
  ```typescript
  app.get('/match', (c) => {
    const config = c.get('config');
    return new Response(renderMatchPage(config, c.env.POSTHOG_API_KEY, c.env.CORE_API_URL), {
      status: 200,
      headers: { 'Content-Type': 'text/html; charset=utf-8', 'Cache-Control': 'no-store' },
    });
  });
  ```
  Cache-Control: `no-store` (not `public, max-age=300`) — match page contains injected CORE_API_URL + satellite_id, should not be CDN-cached.

**Checkpoint:** `npx tsc --noEmit` passes after Step 4.

### Step 5 — Modify `workers/frontend/satellites/src/lib/securityHeaders.ts`
**CRITICAL:** Current `connect-src 'self' https://ph.callibrate.io` blocks the client-side fetch() to Core API.

Update HTML_CSP `connect-src` to include `https://*.callibrate.io`:
```
connect-src 'self' https://ph.callibrate.io https://*.callibrate.io
```

Rationale: matches the CORS allowlist in Core API (`host.endsWith('.callibrate.io')`); future-proofs for additional satellite-facing endpoints; security tradeoff minimal (already within callibrate.io scope).

This change affects all satellite pages (additive only, non-breaking).

## Edge Cases Per AC

- **AC1**: `config.theme` null → CSS vars block empty, fallback colors in CSS properties; `config.brand` null → h1 renders "Callibrate" default
- **AC2**: paste exceeding 2000 → `maxlength` attribute + JS fallback; exactly 1800 → warn color, button still enabled
- **AC3**: `coreApiUrl` with trailing slash → strip or hardcode `/api/extract` path carefully; empty string → fetch fails → AC6 error handler
- **AC5**: `/confirm` 404 → accepted per out-of-scope note; `confirmation_questions` undefined → `JSON.stringify` handles gracefully
- **AC6**: HTTP 422/502/network offline → all caught by `.catch()` + non-ok branch; all show retry UI
- **AC7**: `posthog` global undefined when tracking disabled → all `posthog.capture()` calls gated by `typeof posthog !== 'undefined'`
- **AC10**: sessionStorage unavailable (private mode) → try/catch, graceful degradation without restoration
