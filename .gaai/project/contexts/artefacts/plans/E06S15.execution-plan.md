# E06S15 — Email deliverability hardening: Execution Plan

## Scope Summary

Five files require modification. The changes are purely additive to the existing email-sending infrastructure: two new env vars, an expanded function signature, plain text versions for three email sends, and test updates. No database migrations. No new routes. No new bindings beyond two non-secret `[vars]` entries in `wrangler.toml`.

The Resend HTTP API accepts `reply_to` (string or string array) and `text` (string) as optional top-level fields in the POST `/emails` payload. Both are snake_case.

---

## Implementation Sequence

### Step 1 — Extend `src/types/env.ts` (AC7)

Add two new properties to the `Env` interface:

```typescript
// Email deliverability (non-secret vars in wrangler.toml)
EMAIL_FROM_DOMAIN: string;
EMAIL_REPLY_TO: string;
```

Place after `N8N_WEBHOOK_URL`, grouped under a new comment block for email config.

### Step 2 — Update `wrangler.toml` (AC8)

Add to both `[env.staging.vars]` and `[env.production.vars]`:

```toml
EMAIL_FROM_DOMAIN = "send.callibrate.io"
EMAIL_REPLY_TO = "support@callibrate.io"
```

### Step 3 — Update `sendResendEmail()` and callers (AC4, AC5, AC6)

**3a.** Expand signature: add `text`, `fromDomain`, `replyTo` parameters.

**3b.** Update payload: `from: Callibrate <notifications@${fromDomain}>`, add `text`, add `reply_to: replyTo`.

**3c.** `sendExpertRegisteredEmail()` — add plain text version + pass `env.EMAIL_FROM_DOMAIN`, `env.EMAIL_REPLY_TO`.

**3d.** `sendBookingConfirmedEmails()` expert email — add plain text version + pass env vars.

**3e.** `sendBookingConfirmedEmails()` prospect email — add plain text version + pass env vars.

### Step 4 — Update `SETUP.md` (AC9)

Add section documenting `send.callibrate.io` domain setup, DMARC record, and the two new env vars.

### Step 5 — Update tests (AC10)

**5a.** `makeEnv()` — add `EMAIL_FROM_DOMAIN` and `EMAIL_REPLY_TO`.

**5b.** Test 1 (expert.registered) — add assertions for `from`, `reply_to`, `text` fields.

**5c.** Test 2 (booking.confirmed) — add assertions for both Resend payloads.

**5d-5g.** Tests 3-6 — no changes needed (n8n/retry/DLQ/idempotency paths unaffected).

---

## Edge Cases

- **Missing `EMAIL_FROM_DOMAIN` in `.dev.vars`:** Runtime fallback `env.EMAIL_FROM_DOMAIN || 'callibrate.io'` in callers.
- **Missing `EMAIL_REPLY_TO`:** Runtime fallback `env.EMAIL_REPLY_TO || 'support@callibrate.io'`.
- **`display_name` null:** Existing `?? 'Expert'` fallback covers plain text.
- **Resend rejects `reply_to`:** Standard email format, handled by existing retry/DLQ.
- **Plain text empty string:** Never passed — all templates have substantive content.
