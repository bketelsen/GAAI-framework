# Implementation Report — E03S10: Return Visit (Cookie Auth + Magic Link)

## Summary

Implemented 7-day persistent authentication for return visits via two channels:
1. **Cookie/session token** — issued after OTP verify, validated server-side by satellite on return visits
2. **Magic link email** — sent after identify, enables cross-device access via `GET /results/{id}?token=...`

## Architecture Notes

- Cookie is set on BOTH: Core API domain (SameSite=None for browser auto-send) AND satellite domain (via magic link flow, SameSite=Lax)
- localStorage fallback: client JS stores `callibrate:session:token` and `callibrate:session:prospect_id` for same-device same-tab continuity
- Magic link email: fired from `handleProspectIdentify` as `ctx.waitUntil` (non-blocking); looks up `satellite_configs.domain` for URL construction
- Session validation is server-side in satellite Worker (no token exposure to client JS until embedded in HTML)
