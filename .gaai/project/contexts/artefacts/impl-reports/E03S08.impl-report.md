# E03S08 Implementation Report

## Status: COMPLETE

## Summary

E03S08 merges the 3-page satellite funnel (match → confirm → results) into a 2-page funnel (match → results). The confirm page becomes a transparent redirect. The results page absorbs all former confirm-page responsibilities — confirmation questions, Turnstile bot-gate, extraction summary, match cards, email/OTP gate, full profiles, and booking widget — in a single unified server-rendered HTML page driven by sessionStorage state.

A new backend endpoint (`POST /api/prospects/:id/requirements`) is added to handle re-match requests when a prospect edits their extracted requirements from the summary panel.

## Key Design Decisions

### Single-page state machine
The results page reads `match:extraction` from sessionStorage on load. If missing, it redirects to `/match`. The extraction object drives the page into one of two branches: AC1 (`ready_to_match=true`) fires Turnstile immediately; AC2 (`needs_confirmation=true`) shows the questions section first. Back-navigation is handled by checking `match:prospect_id` — if already set, Turnstile and prospect creation are skipped.

### turnstileSiteKey via window.__SAT__
The `window.__SAT__` object previously held only `apiUrl` and `satelliteId`. It now also carries `turnstileSiteKey`, injected from `c.env.TURNSTILE_SITE_KEY` at the `/results` Worker route. This avoids a separate fetch and keeps all per-satellite config in one client-accessible location.

### handleProspectRequirements — dynamic import to avoid circular deps
The re-match trigger uses `import('./matches')` dynamically at call time (inside `ctx.waitUntil`) rather than a static import. This avoids a circular dependency between `prospects.ts` and `matches.ts`, and ensures the SQL connection can be closed before the async compute job is dispatched.

### KV rate-limiting for re-match
Re-match requests are capped at 5 per prospectId per 24h using a KV counter key `rematch-count:{prospectId}`. On each successful call the counter is written with `expirationTtl: 86400`. The check reads the key and returns 429 if the value exceeds 5.

### Confirm page as pure redirect
The `/confirm` route and `renderConfirmPage` function are preserved in the codebase (rather than deleted) to avoid breaking any external links or bookmarks that may have been shared. The page emits both a `<meta http-equiv="refresh">` and a `window.location.replace` for maximum compatibility, and carries `noindex, nofollow` to prevent indexing.

## Issues Encountered

### Missing node_modules in worktree
The git worktree at `/E03S08-workspace/` was created without installed dependencies. Running `tsc --noEmit` initially failed with:

```
error TS2688: Cannot find type definition file for '@cloudflare/workers-types'.
```

Fixed by running `npm install` in both `workers/frontend/satellites/` and `workers/backend/api/`. After install both `tsc --noEmit` checks passed clean with no output.

Note: the worktree has no `pnpm-lock.yaml`, so `pnpm install --frozen-lockfile` was not an option; `npm install` was used instead.
