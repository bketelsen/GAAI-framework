## Change Detail

### confirm.ts

**AC1 — #loading-msg CSS:**
- Removed `font-size: 0.875rem` (muted small text)
- Changed `color: #555` to `color: #1a1a2e` (body text color, full contrast)

**AC1 — HTML position reorder:**
- `#loading-msg` moved to appear AFTER `#confirm-btn` in DOM order (previously before)
- Default aria text updated to `"Recherche en cours parmi nos experts qualifiés"`

**AC1 — JS loading text:**
- Updated `loadingMsg.textContent` in `renderTurnstile()` callback to match new AC spec text

**AC4 — catch-all error specificity:**
- Replaced generic `"Une erreur est survenue. Veuillez réessayer."` with specific `"La connexion au serveur a échoué. Vérifiez votre connexion internet et réessayez."`

**AC7 — PostHog matching_error in handleSubmitError:**
- Added PostHog `satellite.matching_error` capture at the top of the error handler
- Classifies error into `validation_422`, `rate_limit_429`, `turnstile`, or `network`
- Fires before the per-status error display logic

### results.ts

**AC2 — State variables:**
- Added `waitStart=0` (timestamp when first 202 received)
- Added `networkErrorRetried=false` (silent retry guard)

**AC2 — Computing msg HTML:**
- `<p>` now has `id="computing-msg-text"` for dynamic text updates
- Default text changed to `"Nous affinons les correspondances, quelques secondes…"`

**AC2 — handle202() dynamic text:**
- Sets `waitStart=Date.now()` on first call (retryCount===0)
- If >10 seconds elapsed, updates `computing-msg-text` to reassurance text: `"Cela prend un peu plus de temps que prévu. Votre recherche est en cours."`
- On MAX_RETRIES exhaustion: fires `satellite.matching_error` with `error_type:'computing_timeout'`

**AC3 — Silent retry in .catch():**
- First network failure: sets `networkErrorRetried=true`, silently retries after 2000ms without showing error UI
- Second failure: shows error UI with specific messages
- retry-btn click handler now also resets `networkErrorRetried=false`

**AC5 — Error specificity:**
- Detects `err.status >= 500` to distinguish server vs network errors
- Server: `"Nos serveurs sont momentanément indisponibles. Réessayez dans quelques instants."`
- Network: `"Connexion interrompue. Vérifiez votre connexion et réessayez."`
- Added secondary link `id="fetch-error-browse"` pointing to `/experts`

**AC6 — Computing timeout fallback (no-available-msg):**
- Comment updated from "AC2: No experts available" to "AC6: Computing timeout fallback"
- New copy: `"Le calcul des correspondances prend plus de temps que prévu. Vous recevrez vos résultats par email dès qu'ils sont prêts."`
- Added secondary link `href="/experts"`: `"En attendant, parcourir le répertoire"`

**AC7 — PostHog matching_error in results:**
- `handle202()` fires on timeout: `error_type:'computing_timeout'`
- `.catch()` fires after second failure: `error_type:'server_5xx'` or `'network'`
