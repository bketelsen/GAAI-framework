## AC Checklist

### GET /api/experts/:id/leads

- [x] **AC1**: `handleGetLeads(request, env, user, expertId)` in `handlers/experts/leads.ts`. JWT auth via existing `authenticate()` in router + ownership check (`user.id !== expertId` → 403).
- [x] **AC2**: Query params `?page=1&per_page=20&status=new|confirmed|flagged|all` (default `all`, per_page max 50). Returns `{ leads, total, page, per_page }`.
- [x] **AC3**: Each lead includes: `id`, `status`, `price_cents` (aliased from `amount`), `created_at`, `confirmed_at`, `flagged_at`, `flag_reason`, `flag_window_expires_at` (computed as `created_at + interval '7 days'`), `evaluation_score`, `evaluation_notes`, `conversion_declared`, `evaluated_at`, `prospect` (embedded: `{ id, email, requirements }`), `match_score` (LATERAL JOIN to latest match score), `booking` (embedded: `{ id, starts_at, status }`).
- [x] **AC4**: Results ordered by `created_at DESC`. Offset-based pagination (`LIMIT per_page OFFSET (page-1)*per_page`).

### GET /api/experts/:id/bookings

- [x] **AC5**: `handleGetBookings(request, env, user, expertId)` in `handlers/experts/bookings.ts`. JWT auth + ownership check.
- [x] **AC6**: `?period=upcoming|past|all&page=1&per_page=20` (default: `upcoming`). `upcoming` = `start_at > NOW() AND status NOT IN ('cancelled')`. `past` = `start_at <= NOW() OR status = 'cancelled'`.
- [x] **AC7**: Each booking includes: `id`, `starts_at` (mapped from `start_at`), `ends_at` (mapped from `end_at`), `status`, `meeting_url`, `cancel_reason`, `prospect` (embedded: `{ id, email, name }` from denormalized `prospect_id`, `prospect_email`, `prospect_name` columns), `created_at`.
- [x] **AC8**: `upcoming` ordered by `start_at ASC`, `past` ordered by `start_at DESC`, `all` ordered by `start_at ASC`.

### GET /api/experts/:id/billing

- [x] **AC9**: `handleGetBilling(request, env, user, expertId)` in `handlers/experts/billing.ts`. JWT auth + ownership check.
- [x] **AC10**: Returns `{ credit_balance, ls_subscription_status, ls_subscription_id, spending_limit, max_lead_price, monthly_spend, transactions, total_transactions, page, per_page }`.
- [x] **AC11**: `monthly_spend` = SUM of debit credit transactions since `date_trunc('month', NOW())`.
- [x] **AC12**: `?page=1&per_page=20`. Each transaction: `{ id, type, amount_cents, description, lead_id, created_at }`. Ordered by `created_at DESC`.

### GET /api/experts/:id/dashboard

- [x] **AC13**: `handleGetDashboard(request, env, user, expertId)` in `handlers/experts/dashboard.ts`. JWT auth + ownership check.
- [x] **AC14**: Returns `{ unread_leads, upcoming_bookings, credit_balance, composite_score, quality_tier, month_stats, monthly_history }`.
- [x] **AC15**: `unread_leads` = COUNT leads WHERE `status = 'new'`. `upcoming_bookings` = COUNT bookings WHERE `start_at > NOW() AND status NOT IN ('cancelled')`. `quality_tier` derived via `deriveQualityTier(composite_score)`. `bookings_total` = bookings this month. `conversions_declared` = COUNT leads WHERE `conversion_declared = true` this month.
- [x] **AC15b**: `monthly_history`: 6 months always (padded with zeros for empty months). Each entry: `{ month (YYYY-MM), leads_received, leads_confirmed, leads_flagged, bookings_total, bookings_completed, conversions_declared }`.

## Deviations from Plan

| # | Deviation | Rationale |
|---|---|---|
| 1 | `handleEvaluateLead` uses `ctx: ExecutionContext` as 5th parameter (not in router signature) | The plan called `ctx.waitUntil()` for PostHog — added `ctx` as required parameter. Router call in `index.ts` passes `ctx` correctly. |
| 2 | bookings handler uses `COUNT(*) OVER()` window function instead of separate COUNT query | Reduces DB round-trips by 1 per request while providing total in a single query. Same approach works for all 3 period variants. |
| 3 | `POSTHOG_API_KEY` omitted in test `mockEnv` (instead of full PostHog constructor mock) | Worktree vitest environment has module resolution differences causing `vi.fn().mockImplementation()` constructor mocks to fail in SSR transform mode. Omitting the API key causes `captureEvent` to return early (no-op) — same behavior as the existing test pattern. No production impact (PostHog is always no-op when key absent). |
| 4 | `lib/qualityTier.ts` defines its own `QualityTier` type instead of re-exporting from `types/matching.ts` | `types/matching` imports domain-specific types that could create circular dependencies. The type definition is identical (4-value union) and was already duplicated in `routes/matches.ts`. The new lib file is intentionally self-contained. |
| 5 | Skills filtering for public endpoint done in TypeScript after SQL fetch | As noted in the execution plan: "simpler to filter in app layer for MVP". JSONB array containment in postgres.js tagged templates requires type casting complexity. App-layer filtering is correct for MVP scale. |
