## Known Risks / Limitations

1. **OAuth callback redirect mismatch.** E06S10 redirects to `/onboarding/gcal-connected` after OAuth success. When expert connects GCal from the dashboard (post-onboarding), they land on the onboarding success page instead of returning to `/dashboard/gcal`. Workaround: sessionStorage flash fires toast when user navigates back. To fully fix: E06S10 callback URL would need to be configurable (separate story).

2. **sessionStorage not available in SSR.** Guard: `typeof window !== "undefined"` in all sessionStorage access. No runtime errors.

3. **Availability endpoint is CORS-gated (satellite-facing).** Dashboard loader calls it server-to-server (no Origin header) which bypasses CORS. This is correct and intentional — the endpoint exists for satellite consumption but is also consumable by the dashboard loader.
