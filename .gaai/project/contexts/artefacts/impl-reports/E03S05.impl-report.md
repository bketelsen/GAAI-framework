### AC2 — robots.txt served with correct cache header ✅ PASS

**File:** `workers/satellite/src/index.ts` (route `GET /robots.txt`)

```typescript
app.get('/robots.txt', (c) => {
  const config = c.get('config');
  return new Response(renderRobotsTxt(config), {
    status: 200,
    headers: {
      'Content-Type': 'text/plain',
      'Cache-Control': 'public, max-age=86400',
    },
  });
});
```

- Per-request generation by satellite Worker ✅
- `Cache-Control: public, max-age=86400` → CF edge cache 24h = functionally equivalent to static ✅
- `Content-Type: text/plain` ✅

### AC3 — sitemap.xml generated, linked, correct pages, no profile URLs ✅ PASS

**Files:**
- `workers/satellite/src/pages/sitemap.ts` — sitemap renderer
- `workers/satellite/src/index.ts` — route `GET /sitemap.xml`

Sitemap content (from `renderSitemapXml`):
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url><loc>https://{domain}/</loc></url>
  <url><loc>https://{domain}/match</loc></url>
  <url><loc>https://{domain}/experts</loc></url>
</urlset>
```

- Homepage `/` ✅
- Funnel entry `/match` (static content page) ✅
- Expert listing `/experts` (category page — NOT an individual profile URL) ✅
- Individual profile URLs (`/experts/:id`, `/profiles/:slug`) NOT included ✅
- Sitemap linked in robots.txt via `Sitemap: https://${config.domain}/sitemap.xml` ✅
- Served with `Cache-Control: public, max-age=86400` ✅

### AC4 — Expert profile pages do not carry noindex ✅ PASS

**File:** `workers/satellite/src/pages/landing.ts`

The landing page explicitly opts into indexing:
```html
<meta name="robots" content="index, follow">
```

No `noindex` directive exists anywhere in the satellite Worker's HTML output.
No expert profile page templates exist at this stage of the project. The `index, follow`
pattern on the landing page establishes the correct convention for future profile pages.
