# Implementation Report — E03S01: Prospect Freetext Input Page (/match)

## Summary

Implemented the `/match` route on the satellite Worker: a server-rendered freetext textarea form that lets prospects describe their AI integration project. On submit, client-side JS calls `POST /api/extract` on the Core API, stores the response in sessionStorage, and navigates to `/confirm` (E03S02, not yet built). Full PostHog event tracking, SEO meta, mobile-responsive layout.

A critical infrastructure issue was discovered and fixed during implementation: the existing CSP `connect-src` blocked all `fetch()` calls to the Core API. The fix was additive and non-breaking.

## Rules Applied

- Server-rendered HTML template literal (same as `landing.ts` pattern — no bundler, no external JS)
- `escapeHtml()` applied to all user-controlled config values in HTML context
- `JSON.stringify().replace(/</g, '\\u003c')` applied to all inline JSON/script context values
- `typeof posthog !== 'undefined'` guard on all PostHog capture calls
- `CORE_API_URL` is a plain var (public URL, not a secret) — committed to wrangler.toml
- sessionStorage access wrapped in try/catch (graceful degradation in private browsing)
- `Cache-Control: no-store` on `/match` (contains injected per-environment URL)

## Known Risks / Limitations

- `/confirm` (E03S02) is not yet built — `window.location.href='/confirm'` will 404. Explicitly accepted per story out-of-scope notes.
- CSP `connect-src https://*.callibrate.io` covers all subdomains. This was the correct scope per Core API CORS allowlist (`host.endsWith('.callibrate.io')`) and was a blocking issue (AC3 would have silently failed in browsers without this fix).
- Worker `connect-src` also covers `callibrate-core-staging.frederic-geens-consulting.workers.dev` only via the wildcard when on `*.callibrate.io`. The staging Core API URL uses `workers.dev` domain — not covered by `*.callibrate.io`. See Note below.

## Note: Staging CSP Gap

The staging Core API URL is `https://callibrate-core-staging.frederic-geens-consulting.workers.dev` — which does NOT match `https://*.callibrate.io`. The CSP `connect-src` as implemented will block fetches to the staging Core API from the satellite.

**Resolution options (for human review):**
1. Add `https://*.workers.dev` to `connect-src` (broadest)
2. Add the exact staging URL: `https://callibrate-core-staging.frederic-geens-consulting.workers.dev`
3. Give the staging Core API a `api.staging.callibrate.io` custom domain (cleanest, matches production pattern)

This does not affect production (which uses `https://api.callibrate.io` — covered by `*.callibrate.io`). Flagged for human review. Does not block story delivery — QA will verify against production URL pattern.
