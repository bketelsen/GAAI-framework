## Changes by Acceptance Criterion

### AC1 — Migration applied cleanly
- Migration `initial_schema` applied to fresh schema via `mcp__supabase__apply_migration`.
- Confirmed via `list_migrations`: `[{"version":"20260220003751","name":"initial_schema"}]`.

### AC2 — `experts` table
All required columns present: `id UUID PK REFERENCES auth.users`, `display_name`, `headline`, `bio`, `avatar_url`, `rate_min INTEGER`, `rate_max INTEGER`, `availability TEXT CHECK IN (available/limited/unavailable)`, `cal_username`, `profile JSONB`, `preferences JSONB`, `verified_at`, `composite_score NUMERIC(5,2) DEFAULT 0`, `score_updated_at`, `created_at DEFAULT now()`.

### AC3 — `prospects` table
All required columns present: `id UUID PK DEFAULT gen_random_uuid()`, `satellite_id`, `email`, `utm_source`, `utm_campaign`, `quiz_answers JSONB`, `requirements JSONB`, `status TEXT CHECK IN (anonymous/identified/matched/booked)`, `created_at DEFAULT now()`.

### AC4 — `matches` table
All required columns present: `id UUID PK`, `expert_id UUID FK experts`, `prospect_id UUID FK prospects`, `score NUMERIC(5,2)`, `score_breakdown JSONB`, `status TEXT CHECK IN (pending/revealed/booked/expired)`, `expires_at`, `created_at DEFAULT now()`.

### AC5 — `bookings` table
All required columns present: `id UUID PK`, `match_id UUID FK matches`, `cal_booking_id TEXT UNIQUE`, `cal_meeting_url`, `scheduled_at`, `duration_min INTEGER DEFAULT 20`, `status TEXT CHECK IN (confirmed/cancelled/completed/no_show)`, `created_at DEFAULT now()`.

### AC6 — `leads` table
All required columns present: `id UUID PK`, `booking_id UUID FK bookings`, `expert_id UUID FK experts`, `prospect_id UUID FK prospects`, `amount INTEGER`, `ls_checkout_id`, `ls_order_id`, `status TEXT CHECK IN (pending/billed/disputed/refunded)`, `billed_at`, `created_at DEFAULT now()`.

### AC7 — `satellite_configs` table
All required columns present: `id TEXT PK`, `label`, `domain`, `vertical`, `quiz_schema JSONB NOT NULL`, `matching_weights JSONB NOT NULL`, `created_at DEFAULT now()`.

### AC8 — GIN indexes
4 GIN indexes created and verified via `pg_indexes`:
- `experts_profile_gin ON experts USING gin (profile)`
- `experts_preferences_gin ON experts USING gin (preferences)`
- `prospects_requirements_gin ON prospects USING gin (requirements)`
- `matches_score_breakdown_gin ON matches USING gin (score_breakdown)`

### AC9 — RLS policies
RLS enabled on all 9 tables. Two user-facing policies on `experts`:
- `experts_select_own`: FOR SELECT TO authenticated USING (auth.uid() = id)
- `experts_update_own`: FOR UPDATE TO authenticated USING (auth.uid() = id)
Service role bypasses RLS by default. No user-facing policies on remaining tables → service role only access.

### AC10 — TypeScript types
Generated via `mcp__supabase__generate_typescript_types`. Written to `src/types/database.ts`. Includes `Database` type with all 9 tables, Row/Insert/Update types, FK relationships, and utility types (`Tables`, `TablesInsert`, `TablesUpdate`, `Enums`, `CompositeTypes`).

### AC11 — Seed data
`satellite_configs` row `id: 'default'` inserted:
- `label`: "Default — AI Consulting"
- `domain`: "callibrate.ai"
- `vertical`: "ai-consulting"
- `quiz_schema`: 7 fields (challenge, budget_range, timeline, company_size, industry, skills_needed, languages)
- `matching_weights`: 5 dimensions summing to 1.00 (skills_overlap: 0.35, budget_fit: 0.25, timeline_fit: 0.20, language_match: 0.10, composite_score_boost: 0.10)

### AC12 — `call_experience_surveys` table
All required columns: `id UUID PK DEFAULT gen_random_uuid()`, `booking_id UUID FK bookings`, `prospect_id UUID FK prospects`, `score INTEGER CHECK (BETWEEN 1 AND 5)`, `submitted_at DEFAULT now()`.

### AC13 — `project_satisfaction_surveys` table
All required columns: `id UUID PK DEFAULT gen_random_uuid()`, `booking_id UUID FK bookings`, `prospect_id UUID FK prospects`, `score INTEGER CHECK (BETWEEN 1 AND 10)`, `submitted_at DEFAULT now()`.

### AC14 — `lead_evaluations` table
All required columns: `id UUID PK DEFAULT gen_random_uuid()`, `lead_id UUID FK leads`, `expert_id UUID FK experts`, `lead_quality_score INTEGER CHECK (BETWEEN 1 AND 10)`, `conversion_declared BOOLEAN DEFAULT false`, `conversion_declared_at`, `notes TEXT`, `submitted_at DEFAULT now()`.

## Known Risks / Limitations

- `availability` CHECK constraint allows NULL values (column is nullable). Downstream code must handle NULL as "no status set".
- TypeScript types reflect schema at time of generation — must be regenerated after future schema migrations.
- No INSERT policy on `experts` via authenticated role (only SELECT/UPDATE) — expert creation must go through service role (correct for registration flow in E06S03).
