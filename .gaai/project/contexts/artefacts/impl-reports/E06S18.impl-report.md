## Summary

This story migrates all Cloudflare Worker DB queries from the Supabase JS client (HTTP-based PostgREST) to **postgres.js v3.4.4** (native PostgreSQL wire protocol) via **Cloudflare Hyperdrive** connection pooling. It also adds **Smart Placement** and fixes two production bugs (B1: unlimited match inserts, B2: missing expert pool cache in match computation).

## Implementation Details

### postgres.js Query Patterns Used

```typescript
// Simple SELECT with destructuring
const [row] = await sql<PickType[]>`SELECT col FROM table WHERE id = ${id}`;
if (!row) return errorResponse('Not found', 404);

// Array IN using ANY()
const rows = await sql<T[]>`SELECT id FROM bookings WHERE match_id = ANY(${matchIds})`;

// JSONB parameter
await sql`INSERT INTO ... VALUES (${JSON.stringify(obj)}::jsonb)`;

// INSERT RETURNING
const [inserted] = await sql<Pick<T, 'id'>[]>`INSERT INTO t (...) RETURNING id`;

// Named params RPC (AC8)
const [result] = await sql<ExpertRow[]>`SELECT * FROM merge_expert_profile(
  p_id := ${expertId}, p_bio := ${bio ?? null}, ...)`;

// 23505 unique violation error code
} catch (err) {
  if ((err as { code?: string }).code === '23505') { /* handle duplicate */ }
}
```

### Bug Fixes

**B1 — prospects.ts unlimited match inserts:**
Before: all expert matches inserted without limit.
After:
```typescript
const top20Rows = matchRows
  .sort((a, b) => (b.score ?? 0) - (a.score ?? 0))
  .slice(0, 20);
for (const row of top20Rows) { await sql`INSERT INTO matches ...`; }
```

**B2 — matches.ts direct DB expert query:**
Before: `supabase.from('experts').select(...)` — bypassed KV cache, missed `total_leads`.
After: `const expertPool = await loadExpertPool(env)` — uses KV-cached pool with pre-computed `total_leads`.

### Test Mock Architecture

All test files now use the same pattern:
```typescript
vi.mock('../lib/db', () => ({ createSql: vi.fn() }));
import { createSql } from '../lib/db';
import { type Mock } from 'vitest';

// In test:
const mockSql = vi.fn() as Mock;
(createSql as Mock).mockReturnValue(mockSql);
mockSql
  .mockResolvedValueOnce([/* first query result */])
  .mockResolvedValueOnce([/* second query result */]);
```

postgres.js returns arrays directly (not `{ data, error }`), so `mockResolvedValueOnce([...])` returns the array the handler destructures with `const [row] = await sql\`...\``.

## Hyperdrive Configuration Notes

- Dev environment uses `localConnectionString` (required for `wrangler dev` to connect to Supabase without a real Hyperdrive config ID)
- Staging and production use `id = "PLACEHOLDER_HYPERDRIVE_CONFIG_ID"` — must be replaced with the real Hyperdrive config ID once created via `wrangler hyperdrive create`
- The `prepare: true` option in `createSql` enables prepared statements, which are supported by Hyperdrive
