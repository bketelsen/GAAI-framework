# Implementation Report — E06S03

## Summary

Implemented the expert registration and profile management API for the Cloudflare Worker. Delivered a JWT authentication middleware, a POST /api/experts/register endpoint with Zod validation and email queue notification, GET and PATCH /api/experts/{id}/profile endpoints with JSONB merge via a Supabase RPC, and a KV-backed per-IP rate limiter. A `merge_expert_profile` Supabase RPC was also deployed via migration.

## Rules Applied

- Zod used for all input validation (AC3, AC9) — no manual type narrowing on request bodies.
- `createServiceClient()` centralised — all handlers share one factory, never construct clients inline.
- `SECURITY DEFINER` on the RPC ensures the merge function runs with elevated privileges regardless of caller role.
- `MATCHING_JOBS` removed from `Env` per DEC-33 — no references remain in any file.
- `cal.setup` queue message not included per DEC-34 — deferred to E06S04.
- TypeScript fix applied: `z.record(z.string(), z.unknown())` for Zod v3 compatibility (two-argument form required); `profileMatch[1]!` non-null assertion justified because the capture group is guaranteed by regex structure.

## Known Risks or Limitations

- The rate limiter uses a fixed-window TTL reset (KV `expirationTtl`), not a true sliding window. This is acceptable for the current load profile but may allow up to 2x the intended burst at window boundaries.
- `CF-Connecting-IP` header is trusted; in local dev or non-Cloudflare environments this header may be absent, causing all requests to share the `unknown` key.
- `PGRST116` (PostgREST "no rows found") is the only explicitly handled Supabase error for GET profile; other PostgREST error codes fall through to the generic 500.

## Deviations from Plan

- `z.record(z.unknown())` replaced with `z.record(z.string(), z.unknown())` to satisfy Zod v3's two-argument requirement for `z.record()` — the plan's snippet used the single-argument form which is only valid in Zod v4+.
- `profileMatch[1]` replaced with `profileMatch[1]!` in `src/index.ts` to satisfy TypeScript's `RegExpMatchArray` typing — the plan's snippet passed the value directly, which causes a `string | undefined` type error.
- Both deviations are minimal, type-correct fixes that do not alter runtime behaviour.
