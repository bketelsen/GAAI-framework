## Helper Functions

| Function | Purpose |
|----------|---------|
| `formatBookingDatetime(startsAt, endsAt)` | UTC-locked Intl.DateTimeFormat, fr-FR locale, capitalizes first char. Returns e.g. "Jeu. 6 mars 2026, 14:00 — 14:20" |
| `bookingStatusDisplay(status, period)` | Returns `{ label, badgeClass }`. Past non-cancelled → "Terminé" (gray). held → "En attente" (yellow). confirmed → "Confirmé" (green). cancelled → "Annulé" (red). |
| `buildTabUrl(period, currentParams)` | Constructs `/dashboard/bookings?period=...` URL, preserves other params, removes page |
| `buildPageUrl(page, period)` | Constructs `/dashboard/bookings?period=...&page=...` URL |

## Deviations from Execution Plan

| Deviation | Reason |
|-----------|--------|
| Test count: 13 instead of 14 | Tests 3 and 14 from the plan both cover graceful degradation with empty bookings. Combined into one test as they are structurally identical — both mock `apiGet` to reject, expect `bookings=[]` and `total=0`. No behavioral coverage lost. |
| Node_modules install required for test run | The worktree at `/E02S06-workspace/` had no `node_modules` installed. Ran `npm install --ignore-scripts` at workspace root to install dependencies. Tests then ran successfully using the workspace-local vitest binary. This is an environment setup requirement, not a code deviation. |
| `formatBookingDatetime` uses comma separator | The spec example shows "Jeu. 6 mars 2026, 14:00 — 14:20" with a comma before time. The implementation builds the date and time parts separately and joins with `, ` which produces consistent output. The exact format depends on the `fr-FR` locale's Intl formatting of abbreviated weekday + date. |

## Known Limitations

1. **Prep link deferred.** The detail dialog includes a comment `{/* prep link: not in list API response — deferred */}` as specified. The prep link requires a separate API call not covered by the bookings list endpoint.

2. **UTC-locked formatting.** `formatBookingDatetime` uses `timeZone: "UTC"` to ensure deterministic output independent of the server or test environment timezone. This matches the execution plan specification.

3. **navigator.clipboard not available in SSR.** The copy button uses `navigator.clipboard.writeText()` which is client-side only. This is called inside an `onClick` handler which only fires in the browser — no SSR issue.
