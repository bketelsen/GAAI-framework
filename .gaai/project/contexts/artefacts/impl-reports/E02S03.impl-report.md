## Known Risks / Limitations

1. **`CAREER_STAGE_LABELS` / `WORK_MODE_LABELS` are defined but not rendered.** They are per-spec and silenced with `void`. A future AC could use them in read-only display views.

2. **`ExpertiseSection` custom skill addition** is not included in this implementation. The spec focused on toggling predefined skills (which is the primary UX). Custom skill input (as in onboarding Step 2) can be added as an enhancement if needed — out of scope for E02S03.

3. **No loading/pending state on form submit buttons.** RR7 provides `useNavigation()` for this pattern; not specified in E02S03 ACs. Can be added in a future enhancement story.

4. **GCal route `/dashboard/gcal` does not yet exist.** The link in `GcalSection` points forward — this is expected; the route is out-of-scope for this story.

5. **`availablity` field in `ExpertProfile` type.** The `ExpertProfile.availability` field is typed but the API loader does not explicitly populate a separate `availability` field at root level (it lives in `preferences.availability`). The type is correct per the DB schema; the form reads from `preferences.availability`.
