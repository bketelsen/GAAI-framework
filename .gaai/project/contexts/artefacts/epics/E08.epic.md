# Epic: Security Hardening — Production Readiness

## Purpose

Les experts confient à Callibrate leur calendrier Google, leurs données de facturation et leurs informations de profil. Les prospects soumettent des descriptions de projet et des emails. La plateforme doit garantir que ces données sont protégées contre les attaques connues avant le go-live.

Cet Epic remédie les 17 findings identifiés dans l'audit SEC-AUDIT-001. Il ne crée pas de nouvelles fonctionnalités — il durcit l'existant.

---

## Scope

- Remédiation des 2 findings CRITICAL (OAuth CSRF, booking hold DoS)
- Remédiation des 5 findings HIGH (PKCE, XSS, security headers, admin auth, input validation)
- Remédiation des 7 findings MEDIUM (secrets config, JWT claims, CORS optimization, prompt injection guard)
- Remédiation des 3 findings LOW (token transport, RPC validation, Content-Type check)

---

## Out of Scope

- Penetration testing externe (recommandé post-Epic)
- WAF rules Cloudflare (configuration dashboard, hors codebase)
- SOC2 compliance framework
- Audit de l'infrastructure Supabase (RLS déjà validé, pas de modification)
- Secrets rotation process (documentation hors codebase)
