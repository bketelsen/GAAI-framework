# Epic E06: Platform Foundation — Infrastructure & Technical Plumbing

## Purpose

All technical infrastructure, data models, APIs, and async workers are in place so that E02–E05 user-facing epics can be built on a stable, governed foundation with no rework.

---

## Scope

- Project bootstrap: Supabase + Cloudflare Workers + Queues + KV
- Database schema and migrations (all core tables + feedback tables, JSONB flexible model, GIN indexes, RLS)
- Auth middleware + Expert registration and profile APIs
- Google Calendar OAuth layer (E06S10) + Booking Engine (E06S11) — replaces Cal.com (DEC-41)
- Matching engine scoring function (JSONB-based, configurable weights, bi-directional, composite score tiebreaker)
- Async queue workers: email-notifications, lead-billing, score-computation
- CF Workflows: durable time-delayed workflows for survey triggers (E06S16) — replaces n8n (DEC-59)
- Satellite funnel API: config, prospect submission, match reveal, email capture
- AI freetext extraction service: natural language → ProspectRequirements JSONB (GPT-4o-mini — DEC-48, E06S12)
- Expert composite score computation: tri-directional feedback aggregation → composite_score
- Survey submission endpoints: feedback loop activation (E06S17)
- Satellite infrastructure: satellite_configs schema (E06S13), multi-tenant Worker (E06S14)
- Email deliverability hardening (E06S15)
- Scalable matching engine: Hyperdrive connection pooling, postgres.js tagged templates, Smart Placement (E06S18)
- Cache API L1 per-datacenter + D1 edge serving with Cron sync (E06S23 — E06S19 merged)
- Security hardening: CF Rate Limiting binding + Turnstile CAPTCHA (E06S20)
- Vectorize semantic matching: embedding pipeline + scoring integration (E06S21, E06S22)
- Service Bindings Worker split + Durable Objects write coordinator (E06S24, E06S25)
- Analytics Engine matching observability (E06S26)

---
