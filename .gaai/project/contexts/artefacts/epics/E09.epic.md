# Epic: CI/CD Test Hardening — Deployment Confidence

## Purpose

As a founder deploying stories to staging, I need confidence that merged code actually works on the live environment — without manual verification after every deploy.

Today, staging deploys run zero tests before deploying (core, satellite, posthog-proxy), the matching-engine has no CI/CD pipeline at all, and the existing smoke test script is manual-only. This means regressions and infrastructure wiring issues are caught by human testing in the browser — too late, too slow, too unreliable.

This epic closes the gap between "code merged" and "code verified on staging."

---

## Scope

- Pre-deploy quality gates (type-check + vitest) on all workers in `deploy-staging.yml`
- Matching-engine CI/CD pipeline (deploy + tests)
- Post-deploy automated smoke testing against live staging URLs
- Matching-engine unit test coverage (currently zero)
- Extended API regression suite covering all done stories' endpoints
- Frontend E2E regression suite (Playwright) covering satellite funnel and expert dashboard

---

## Out of Scope

- Integration tests using `@cloudflare/vitest-pool-workers` (future investment — separate epic)
- Production deploy hardening (already has test gates)
- Synthetic monitoring / uptime checks (operational concern, not CI/CD)
- Coverage thresholds / coverage reporting (nice-to-have, not blocking)
- Visual regression testing (screenshot comparison)
- Multi-browser E2E testing (Chromium-only is sufficient)
