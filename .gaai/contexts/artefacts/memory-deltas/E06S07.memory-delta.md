### New Patterns (worth persisting)

1. **JWT pattern (Web Crypto API)**: `signProspectToken()` / `verifyProspectToken()` via `crypto.subtle` (HMAC-SHA256). No external library needed — CF Workers provides `crypto.subtle` natively. Located at `src/lib/jwt.ts`. Pattern: base64url encode header+payload, HMAC-SHA256 sign, verify on each request. Claims: `{ prospect_id, exp }`.

2. **KV cache-aside pattern**: `src/lib/expertPool.ts` — try KV first (`get(key, { type: 'json' })`), fall back to DB on miss, write-back with `expirationTtl`. Non-blocking write failure (try/catch). Used for expert pool to avoid full-table scan per prospect submission.

3. **CORS middleware pattern**: `src/lib/cors.ts` — `handleCors()` returns `{ allowed, origin, preflight }` discriminated result. Caller checks `preflight` first (OPTIONS), then `allowed`, then adds CORS headers to actual response via `addCorsHeaders()`. Blocked origins receive 403 WITH CORS headers (so browser can read error body). No Origin header → allow (server-to-server).

4. **Dynamic CORS from DB**: `validateOrigin()` queries `satellite_configs.domain` to allow satellite-domain origins. DB stores bare hostnames (`callibrate.ai`, not `https://callibrate.ai`). CORS check extracts `new URL(origin).hostname` for comparison.

5. **quiz_schema validation pattern**: `extractRequiredKeys()` supports two formats — Format A (`{ questions: [{ key, required }] }`) and Format B (`{ fieldKey: { required: true } }`). Liberal validation — unknown formats return no required keys (no 422 for unrecognized schema format).
