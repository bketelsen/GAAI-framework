## New Patterns Worth Persisting

### 1. PostHog Injection Pattern (Satellite Worker)

**Context:** Cloudflare Worker generating server-side HTML for satellite landing pages.

**Pattern:**
- Inject the PostHog JS minified stub into `<head>` via an inline `<script>` tag (no `defer` or `src` attribute on the outer tag — the stub itself creates an async script element internally with `p.async=!0`).
- Use `capture_pageview:false` in `posthog.init()` to prevent any blocking network call at init time.
- Fire `page_view` manually in a separate `<body>` script after DOM is available.
- `api_host` points to `https://ph.callibrate.io` (reverse proxy — do not use the PostHog direct endpoint).
- `ui_host` is set to `https://eu.posthog.com` (EU dashboard region).
- `persistence:"memory"` for cookieless mode (GDPR compliance).
- API key is sourced from `c.env.POSTHOG_API_KEY` (Cloudflare secret), never hardcoded.
- Both head snippet and body script are guarded: `config.tracking_enabled !== false && posthogApiKey`.

**Security note:** Both `posthogApiKey` and `config.id` are wrapped in `JSON.stringify()` before interpolation into inline scripts, preventing XSS via malicious config values.

---

### 2. Per-Satellite Tracking Toggle Pattern

**Context:** Multi-tenant satellite architecture where individual satellites may need tracking disabled (GDPR opt-out, debug mode, white-label clients).

**Pattern:**
- Add `tracking_enabled BOOLEAN NOT NULL DEFAULT true` to `satellite_configs`.
- Check `config.tracking_enabled !== false` (strict false check, not falsy) to allow `undefined` to default to enabled — important during KV cache transition periods when old cached entries may not yet include the new field.
- Default `true` means no action needed for existing satellites at migration time.

---

### 3. Analytics Event Schema for Satellites

**Events fired:**

| Event | Properties | Trigger |
|---|---|---|
| `page_view` | `satellite_id`, `referrer`, `utm_source`, `utm_campaign`, `utm_medium` | On page load (body script) |
| `satellite.cta_clicked` | `satellite_id`, `cta_text` | On `.cta` element click |

UTM params default to `null` (not `undefined`) when absent from `URLSearchParams`.
`referrer` uses `document.referrer || null`.
