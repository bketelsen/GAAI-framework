# Memory Delta — E03S06

## NEW_PATTERNS

### 1. Inline booking widget via CustomEvent + DOM injection
The booking widget pattern uses a `CustomEvent('booking-open', {detail: {expertId, expertName}})` dispatched on `window`. The widget listens and inserts a `div.bw-container` inline into the matching `.match-card` DOM node, then animates open via CSS class addition (`bw-container--open`). This decouples the results page logic from the booking widget logic while keeping the widget inline (no modal, no navigation).

**File:** `workers/frontend/satellites/src/pages/booking-widget.ts`

### 2. Server-side CSS injection from TypeScript module
Booking widget styles are authored as a TypeScript function returning a CSS string (`getBookingWidgetStyles(): string`). The function is imported into `results.ts` and injected into the `<style>` block server-side. This keeps the satellite a single-file worker with no separate CSS assets.

**Pattern:** All satellite page CSS shipped inline via `${getBookingWidgetStyles()}` inside a `<style>` tag.

### 3. State machine pattern for multi-step widget
The widget uses an explicit `STATE` variable (`IDLE | DATE_STEP | SLOT_STEP | HOLD_STEP | SUCCESS | EXPIRED`) to control rendering logic. Each step re-renders `containerEl.innerHTML`. This avoids show/hide complexity and keeps each step's DOM independent.

### 4. Slot grouping by local timezone for calendar
`buildSlotsByDate()` uses `Intl.DateTimeFormat('fr-CA', {timeZone: tz, ...}).formatToParts()` to convert UTC slot timestamps to local YYYY-MM-DD keys. The calendar then maps by those keys, ensuring the prospect sees dates in their own timezone — not UTC.

### 5. Countdown timer with expiry callback
`startCountdown()` uses `setInterval(1000ms)` comparing `Date.now()` to `new Date(heldUntil).getTime()`. When `remaining <= 0`, it clears the interval and calls `handleHoldExpired()`. This is the authoritative hold-expiry mechanism on the client side, independent of server-side 410 detection.

### 6. prep_token in confirm response
`confirm.ts` returns `prep_token: booking.prep_token` in its success JSON. The widget consumes this to construct the prep link: `https://{hostname}/prep/{encodeURIComponent(prepToken)}`. This is the canonical pattern for prep link construction from the booking widget.

## DRIFT_DETECTED

None. All patterns align with established satellite conventions (IIFE scripts, server-side rendering, escHtml, PostHog guarded capture, sessionStorage for prospect session, window.__SAT__ for config).
