### Pattern 1 — Anthropic Tool Use for Structured Output

**Context:** E06S08 uses Anthropic tool_use API to enforce JSON schema output from Claude. No SDK dependency.

**Pattern:**
```typescript
const tool = {
  name: 'extract_requirements',
  description: '...',
  input_schema: { type: 'object', properties: { ... }, required: [...] },
};

const body = {
  model: 'claude-haiku-4-5-20251001',
  tools: [tool],
  tool_choice: { type: 'tool', name: 'extract_requirements' }, // forces schema compliance
  messages: [{ role: 'user', content: userText }],
};
```

**Why:** `tool_choice: { type: 'tool', name }` forces Claude to always call the specified tool, guaranteeing JSON schema compliance. No need for prompt engineering to enforce output format.

### Pattern 2 — CF AI Gateway Routing for Anthropic

**Context:** All Claude API calls route through Cloudflare AI Gateway for usage visibility and cost logging.

**Pattern:**
```typescript
// CLOUDFLARE_AI_GATEWAY_URL = https://gateway.ai.cloudflare.com/v1/{account}/{gateway}/anthropic
const apiUrl = `${env.CLOUDFLARE_AI_GATEWAY_URL}/v1/messages`;

await fetch(apiUrl, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': env.ANTHROPIC_API_KEY,       // NOT Authorization: Bearer
    'anthropic-version': '2023-06-01',
  },
  body: JSON.stringify(body),
});
```

**Note:** Anthropic uses `x-api-key` header, not `Authorization: Bearer`. The gateway URL includes the Anthropic provider path — append `/v1/messages` for the messages endpoint.

### Pattern 3 — Non-Blocking Optional DB Enrichment

**Context:** Some endpoints accept an optional ID to enrich their processing (e.g., vertical context for extraction). Enrichment failure must never block the core operation.

**Pattern:**
```typescript
let verticalContext: string | null = null;

if (typeof satellite_id === 'string' && satellite_id) {
  try {
    const { data } = await supabase
      .from('satellite_configs')
      .select('vertical')
      .eq('id', satellite_id)
      .maybeSingle();

    if (data?.vertical && typeof data.vertical === 'string') {
      verticalContext = data.vertical;
    }
  } catch {
    // Non-blocking: proceed without enrichment
  }
}
```

**Why:** Optional enrichment wrapped in try/catch. Null check after DB read. Core logic proceeds regardless of enrichment result.
