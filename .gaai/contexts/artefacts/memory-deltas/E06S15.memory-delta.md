# Memory Delta — E06S15

## New Patterns Established

### Email deliverability configuration via env vars
- Sending domain (`EMAIL_FROM_DOMAIN`) and reply-to address (`EMAIL_REPLY_TO`) are externalized as non-secret `[vars]` in `wrangler.toml`, not hardcoded in source.
- Fallback defaults are applied at the call site: `env.EMAIL_FROM_DOMAIN || 'callibrate.io'` (root domain as safe fallback).
- This is the first instance of non-secret, non-KV, non-Queue vars in `wrangler.toml`. Establishes precedent for future config vars.

### Plain text email companion pattern
- Every `sendResendEmail()` call now requires both `html` and `text` parameters.
- Plain text versions use `\n\n` for paragraph separation, plain URLs (no `<a>` tags), and no HTML tags.
- Tests explicitly assert `not.toContain('<p>')` and `not.toContain('<a ')` to catch regressions.
- Future email types must follow this dual-format pattern.

### Subdomain isolation for transactional email
- Transactional emails send from `send.callibrate.io`, not the root domain.
- This reserves the root domain for future marketing email subdomain without reputation cross-contamination.
- DMARC starts at `p=none` (monitoring), with documented upgrade path to `p=reject`.

## Existing Patterns -- No Update Needed

- `sendResendEmail()` function signature growth (now 7 params). If further parameters are added (e.g., `cc`, `bcc`, `attachments`), refactor to an options object. Current state is acceptable at 7 params.
- `SETUP.md` documentation pattern: new infrastructure sections are appended before the "Verify Local Dev" step, with an `(E06Sxx)` tag in the heading. Consistent with `Google Calendar OAuth Secrets (E06S10)` pattern.

## Drift from Documented Conventions

None detected. Implementation aligns with:
- Env var naming convention (SCREAMING_SNAKE_CASE)
- wrangler.toml structure (vars in `[env.{stage}.vars]`)
- Test assertion pattern (parse JSON body, assert field-by-field)
- SETUP.md documentation structure
