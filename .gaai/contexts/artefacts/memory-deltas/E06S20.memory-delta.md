## New Knowledge

### Cloudflare Rate Limiting Binding Pattern

The project uses the Cloudflare Workers Rate Limiting API (binding type `RateLimit`) rather than a KV-based counter. The binding is declared in `wrangler.toml` via `[[rate_limiting]]` blocks (one per env: dev/staging/production). The `Env` interface type for the binding is `RateLimit` (not `KVNamespace`). Invocation: `env.RATE_LIMITER.limit({ key: ip })` returns `{ success: boolean }`. This pattern is zero-cost for KV operations and enforced at the Cloudflare edge.

Namespace IDs in use:
- dev: 1001
- staging: 1002
- production: 1003

### Turnstile Test Key

The Cloudflare Turnstile always-pass test secret key `1x0000000000000000000000000000000AA` is the standard documented test key. It is safe to commit in `wrangler.toml` under `[env.staging.vars]`. Production secret must be set via `wrangler secret put TURNSTILE_SECRET_KEY --env production` (never committed).

### Dev Env Setup Gap

`.dev.vars.example` does not include `TURNSTILE_SECRET_KEY`. Developers running `wrangler dev` locally must add `TURNSTILE_SECRET_KEY=1x0000000000000000000000000000000AA` to their `.dev.vars` file manually.
