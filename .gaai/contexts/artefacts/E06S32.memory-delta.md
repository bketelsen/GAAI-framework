## New Knowledge

### Pattern: Supabase RPC for atomicity without postgres.js (AC3 pattern)
When postgres.js/Hyperdrive is not available in a Worker (because the dependency PR is not yet merged), PostgreSQL functions via `supabase.rpc()` provide equivalent ACID atomicity:
- Function uses SELECT FOR UPDATE to lock the row
- INSERT + UPDATE operations run in the same plpgsql transaction
- Returns jsonb with `{success, reason?, ...}` for structured error handling
- Tested via mock: `vi.fn().mockResolvedValue(rpcResult)` on `supabase.rpc`

### Pattern: Dependency branch merge in worktree
When a dependency story (E06S31) is done but its PR is not yet merged to staging, merge the dependency branch directly into the story branch in the worktree:
```bash
git -C ../E06S32-workspace merge origin/story/E06S31 --no-edit
```
The PR description should note the merge dependency order.

### Pattern: `expert.billing.insufficient_balance` email type
New `EmailNotificationMessage` variant added. Email consumer currently stubs it — template implementation deferred to E06S33+.

### Pattern: exactOptionalPropertyTypes workaround for optional object spread
With `exactOptionalPropertyTypes: true`, use spread instead of direct assignment:
```ts
// BAD: skills: requirements?.skills_needed  (string[] | undefined not assignable to string[])
// GOOD:
...(requirements?.skills_needed ? { skills: requirements.skills_needed } : {})
```
