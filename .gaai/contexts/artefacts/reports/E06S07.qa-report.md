## Remediation Cycle

**Initial QA verdict: FAIL** (1 item)

**Failure**: `corsForbidden()` returned 403 with no CORS headers — blocked origins received an opaque browser CORS failure rather than a readable 403. Satellite developers debugging a missing `satellite_configs.domain` entry would see a network error with no diagnostic information.

**Fix applied**: `src/lib/cors.ts` `corsForbidden()` now accepts optional origin parameter; `src/index.ts` passes `corsResult.origin`. Blocked origins receive `Access-Control-Allow-Origin: {blocked-origin}` header so the 403 body is readable in the browser.

**Items reviewed but NOT changed (QA over-reads)**:

1. **AC5/AC7 response wrappers** (`{ matches: [...] }`, `{ experts: [...] }`): QA initial pass flagged these as missing from the AC spec. ACs specify the *card/profile shape*, not the top-level response envelope. Codebase convention is wrapper objects (see `handleMatchCompute` → `{ computed, top_matches }`, `handleMatchGet` → `{ results, page, page_size }`). Wrappers kept as-is.

2. **EXPERT_POOL placeholder KV IDs**: QA flagged as "deployment-blocking". This is intentional — KV IDs require human provisioning (see SETUP.md Step 3). Code falls back to DB on KV miss — fully functional without provisioned KV. Not a code defect.

3. **satellite_configs.domain format**: QA raised ESCALATE concern about bare hostname vs full URL. Resolved by DB query: confirmed `domain = "callibrate.ai"` (bare hostname). CORS code correctly extracts `originUrl.hostname` for comparison. No issue.
