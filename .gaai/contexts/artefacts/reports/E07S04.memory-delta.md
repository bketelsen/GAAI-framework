# Memory Delta — E07S04: PostHog Funnel Dashboards via API

## Verdict: NEW_KNOWLEDGE_FOUND

New patterns introduced that should be recorded in conventions.

---

## New Patterns

### 1. `scripts/` directory — standalone Node.js TypeScript setup scripts

- **Pattern:** One-time operator scripts (setup, migration, data seeding outside Supabase) live in `scripts/` at the repo root.
- **Runtime:** `npx tsx scripts/{name}.ts` — no compilation step, no output files.
- **Env vars:** `process.env["VAR_NAME"]` (bracket notation — compatible with `noUncheckedIndexedAccess`). Fail fast with `process.exit(1)` if required vars missing.
- **TypeScript config:** `scripts/tsconfig.json` — independent from root `tsconfig.json` which only includes `src/**/*`. Use `moduleResolution: "bundler"` + all strict flags. No `@cloudflare/workers-types`.
- **Type check:** `npx tsc --project scripts/tsconfig.json --noEmit` — NOT the root tsconfig (wrong scope).

### 2. PostHog Dashboard + Insights API — first use

- **Base URL (EU):** `https://eu.posthog.com`
- **Auth:** `Authorization: Bearer {POSTHOG_PERSONAL_API_KEY}` — Personal API Key required (not Project API Key) for Dashboard/Insight write operations
- **Create dashboard:** `POST /api/projects/{projectId}/dashboards/` → `{ name, description }` → `{ id, name }`
- **List dashboards:** `GET /api/projects/{projectId}/dashboards/?limit=100` → `{ results: [{ id, name }], next }`
- **Create insight:** `POST /api/projects/{projectId}/insights/` → `{ name, filters: {...}, dashboards: [dashboardId] }`
- **Insight shape (FUNNELS):** `filters.insight: "FUNNELS"`, `filters.events: [{ id, name, order, type: "events" }]`, optional: `filters.breakdown`, `filters.breakdown_type: "event"`, `filters.date_from`, `filters.funnel_window_interval`, `filters.funnel_window_interval_unit`
- **Insight shape (TRENDS):** `filters.insight: "TRENDS"`, `filters.events: [{ id, name, order, type: "events", math?: "total" }]`, `filters.date_from`, `filters.interval: "day"`, optional: `filters.formula` (e.g. `"B/A*100"`), `filters.display`
- **Idempotency:** Check existing by exact name match before creating. At dashboard level — scope for one-time setup scripts. Insight-level idempotency not needed.
