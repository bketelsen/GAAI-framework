## Key Design Decisions

- **Tool schema includes `confirmation_questions`** in same call (AC5 requirement). LLM is instructed to generate questions for fields with confidence < 0.7. Post-processing filters to max 3 questions targeting actual low-confidence fields.
- **Non-blocking satellite enrichment**: `satellite_configs` DB read wrapped in try/catch. Any error → proceed without vertical context. Prevents satellite_id lookup failure from blocking extraction.
- **AI Gateway URL + `/v1/messages`**: `CLOUDFLARE_AI_GATEWAY_URL` is the Anthropic-provider base URL (format: `https://gateway.ai.cloudflare.com/v1/{account}/{gateway}/anthropic`). `/v1/messages` appended in code.
- **`challenge` and `company_size` added to `ProspectRequirements`**: These fields are in the extraction output but weren't in the original matching type. Added as optional fields — matching engine ignores unknown fields.

## Known Risks / Limitations

- `budget_range.min` is optional in `BudgetRange` (pre-existing type). AC4 specifies `{ min: number, max: number }`. LLM may omit `min` for single-value budgets — this is acceptable at MVP.
- No rate limiting on `POST /api/extract` (not specified in ACs). To be added if misuse is observed.
- Test uses mocked Claude responses — real accuracy depends on haiku-4-5 quality in production.
