## Known Risks (from impl-report, acknowledged not escalated)

1. **LS_STORE_ID / LS_VARIANT_ID placeholders** (`lead-billing.ts` lines 8–9): hardcoded `'YOUR_LS_STORE_ID'` / `'YOUR_LS_VARIANT_ID'`. Functional gap until Lemon Squeezy account is configured. This is a pre-production configuration item, not a code defect, and the TODO is explicitly documented. Does not affect test validity (tests mock the fetch response).

2. **Duplicate email risk on retry** (`booking.confirmed`): if Resend succeeds but n8n webhook fails, on retry Resend is called again. KV mark is only set after full success. Duplicate emails possible. Noted in impl-report as MVP-acceptable. Not a data integrity issue.

3. **Expert email falls back to gcal_email**: if expert has not completed GCal OAuth, `gcal_email` is null and the consumer throws + retries. Intentional design — incentivizes GCal connection completion. Acceptable for MVP.
