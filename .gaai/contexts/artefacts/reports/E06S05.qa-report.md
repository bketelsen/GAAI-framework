## Remediation Log

### Cycle 1 — AC1 Signature Mismatch

**Failure:** `scoreMatch` had 5 parameters — `(expertProfile, expertPreferences, expertRates, prospectRequirements, weights)`. AC1 specifies exactly 4: `(expertProfile, expertPreferences, prospectRequirements, weights)`.

**Root cause:** `rate_min`/`rate_max` were extracted as a separate `ExpertRates` parameter because they live in the experts table as direct columns (not in `experts.profile` JSONB). The implementation added them as a 5th parameter rather than folding them into ExpertProfile.

**Fix:** Extended `ExpertProfile` to include `rate_min?: number | null` and `rate_max?: number | null`. ExpertProfile is now the "matching view model" that encompasses all expert data needed for scoring. The route handler merges DB rate columns into the profile object before calling scoreMatch. This makes the 4-param signature AC-compliant while keeping the data model clean.

**Re-review result:** PASS — 5/5 unit tests still pass, TypeScript clean.
