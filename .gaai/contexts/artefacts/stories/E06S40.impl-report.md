## Defense Layer Order in extract.ts

The 6 layers are ordered cheapest-first to reject garbage before it costs anything:

1. **Parse body** ‚Äî 400 on invalid JSON
2. **Honeypot** (`_hp` non-empty) ‚Äî fake 200 (AC3)
3. **Timing** (`_ts < 3000ms`) ‚Äî fake 200 (AC4)
4. **Global rate limit** (RATE_LIMITER binding) ‚Äî 429 (pre-existing)
5. **Tiered KV rate limit** (SESSIONS, per-IP) ‚Äî 4/min or 10/10min ‚Üí 429, +24h block (AC5)
6. **Text validation** (`validateFreetext`) ‚Äî 422 with reason (AC2)
7. **Circuit breaker** (`isCircuitOpen`) ‚Äî 503 if cost > $10/hr (AC8)
8. **Extraction count** (cookie `_extract_count` + IP KV fallback) ‚Äî 403 (AC9)
9. **OpenAI call** (max_tokens: 500) + cost tracking + flow token signing ‚Üí 200 (AC10, AC11)

## Notable Decisions

### AC6 ‚Äî Turnstile placement
The story said to "move Turnstile FROM extract TO confirmation step." In practice, Turnstile was never on `POST /api/extract` ‚Äî it was on `POST /api/prospects/submit` from E06S20. AC6 is satisfied: extract has no Turnstile (layers 1-6 replace it), and prospects/submit retains Turnstile. The flow token (AC11) is the stronger gate on submit ‚Äî it proves the client completed a real extract ‚Üí confirm ‚Üí submit flow.

### mockEnv test isolation
`checkExtractRateLimit` writes to `SESSIONS` KV on every call, even when the request fails at a later layer (text validation, etc.). Tests using a module-level `const mockEnv` accumulated rate-limit counts across tests, causing 13 false-429 failures. Fix: changed to `let mockEnv` reset in `beforeEach` to give each test a fresh KV store.
