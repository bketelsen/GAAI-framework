# Story E06S07: Satellite Funnel API

## User Story

**As a** prospect arriving from a niche satellite site,
**I want** to submit my project needs, see matched experts, and book a call,
**so that** I can find the right AI expert without leaving the context of my industry.

---

## Context

The API layer that powers all satellite sites. Each satellite loads its quiz schema and matching weights dynamically — no code changes needed to add a new niche. The prospect flow is: submit quiz → synchronous matching (scoreMatch() inline, DEC-40) → see anonymized results → provide email → get full profiles + booking links. CORS is configured for satellite domains. The token-gated reveal mechanic ensures email capture before profile unlock.

---

## Acceptance Criteria

- [ ] AC1: `GET /api/satellites/:id/config` returns `{ id, label, domain, vertical, quiz_schema, matching_weights }` for the given satellite — returns `404` if satellite not found
- [ ] AC2: `POST /api/prospects/submit` accepts `{ satellite_id, quiz_answers, utm_source?, utm_campaign? }` — validates that `quiz_answers` contains all required question keys defined in `satellite_configs.quiz_schema` for that satellite — returns `422` with missing fields if validation fails
- [ ] AC3: Valid `POST /api/prospects/submit` normalizes `quiz_answers` to `requirements JSONB`, creates a `prospects` row with `status: 'anonymous'`, loads the active expert pool from `EXPERT_POOL` KVNamespace, calls `scoreMatch()` synchronously for each expert, INSERTs all scored results into the `matches` table, returns `{ prospect_id, token, token_expires_at }` — token is a signed JWT (24h TTL, `PROSPECT_TOKEN_SECRET` secret) *(DEC-40 — matching-jobs supprimée DEC-33)*
- [ ] AC4: `GET /api/prospects/:id/matches?token=xxx` validates the token (signature + expiry + matches prospect_id) — returns `403` if invalid or expired
- [ ] AC5: Valid `GET /api/prospects/:id/matches?token=xxx` returns all matches from `matches` table ranked by `score` descending, as anonymized expert cards: `{ rank, score, score_breakdown, skills, industries, project_types, languages, rate_min, rate_max }` — no `display_name`, `avatar_url`, `cal_username`, or contact info. No artificial limit on number of results.
- [ ] AC6: `GET /api/prospects/:id/matches?token=xxx` returns `{ status: "computing", estimated_seconds: 5 }` with `202` if the `matches` table is empty for this `prospect_id` *(garde défensive — ne se déclenche pas dans le flux nominal depuis DEC-40 ; couvre les cas d'erreur ou d'expert pool vide)*
- [ ] AC7: `POST /api/prospects/:id/identify` accepts `{ email, token }` — validates token, validates email format, updates `prospects.email` and `prospects.status` to `'identified'`, returns full expert profiles: `{ display_name, headline, bio, skills, industries, rate_min, rate_max, score, booking_url }` where `booking_url = https://cal.com/{cal_username}/intro-call`
- [ ] AC8: `POST /api/prospects/:id/identify` returns `409` if prospect already identified (email already set)
- [ ] AC9: CORS configured on all `/api/satellites/*` and `/api/prospects/*` routes: allow origins matching `*.callibrate.io`, `app.callibrate.io`, and any domain listed in `satellite_configs.domain` — all other origins return `403` *(callibrate.ai supprimé — DEC-39)*
- [ ] AC10: All endpoints return consistent error shape `{ error: string, details?: object }` and correct HTTP status codes

## Out of Scope

- Satellite site frontend (built separately per satellite in Astro or WordPress)
- Prospect account creation or login (anonymous at MVP)
- Saved prospect searches or history
- Pagination (all matching experts returned in single response at MVP — paginate post-MVP if pool grows significantly)

## Notes

The token returned from `POST /api/prospects/submit` is a short-lived JWT (not stored in DB) — validated by the Worker on each request. This avoids a DB lookup per reveal request. Matching is synchronous (DEC-40) — the submit endpoint returns only after matches are inserted; the satellite frontend calls `GET /api/prospects/:id/matches?token=xxx` directly after submit, no polling required. The `202` guard (AC6) covers edge cases only (expert pool empty, insert error). Satellite domain whitelist should be checked against `satellite_configs` table at request time — not hardcoded in Worker config.
