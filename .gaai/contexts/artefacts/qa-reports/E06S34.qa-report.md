### AC5: Monthly spend query uses index on `credit_transactions(expert_id, created_at)`

**PASS**

`src/lib/billingFilter.ts` (lines 84-90):
```sql
SELECT expert_id, COALESCE(SUM(ABS(amount)), 0)::integer AS monthly_spend
FROM credit_transactions
WHERE type = 'lead_debit'
  AND expert_id = ANY(${expertIds})
  AND created_at >= ${monthStart.toISOString()}
GROUP BY expert_id
```

- `AND expert_id = ANY(${expertIds})` — present, enables index on `(expert_id, created_at)`.
- `AND created_at >= ${monthStart.toISOString()}` — range constraint using the index.
- `COALESCE(SUM(ABS(amount)), 0)` — handles NULL for experts with no transactions.

`workers/matching/src/index.ts` (lines 132-138): identical query structure with same three conditions.
