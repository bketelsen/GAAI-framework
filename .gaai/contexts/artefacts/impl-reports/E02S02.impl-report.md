## Rules Applied

- Layer 3 → Layer 2 only: all data via `apiGet`/`apiPatch`/`apiPost`
- Zod safeParse pattern (E06S03)
- React Router v7 loader/action pattern (E02S01)
- PostHog direct fetch pattern (new for dashboard; posthog-node used in core API)
- No secrets in wrangler.toml vars

---

## Known Limitations

- Welcome toast in dashboard (AC9) requires JavaScript — shows only after client hydration. Acceptable for MVP.
- Step 3 validation (required radio fields career_stage/work_mode) relies on HTML `required` attribute for browser-level prevention; Zod validates server-side. No JS disable of submit button (complexity vs value tradeoff at MVP).
- GCal OAuth redirect returns to wherever Google's callback sends the user (handled by core API E06S10 callback). After OAuth completes, user is responsible for navigating back to `/onboarding?step=4` to see connected status.
