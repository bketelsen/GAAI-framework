## Known Risks and Limitations

1. **DNS records not yet created.** The `send.callibrate.io` subdomain requires Resend domain verification (DKIM + SPF DNS records) and a `_dmarc.callibrate.io` TXT record. These are manual ops documented in SETUP.md but not yet provisioned.
2. **Fallback domain.** If `EMAIL_FROM_DOMAIN` is unset (e.g., local dev without `.dev.vars` update), the code falls back to `callibrate.io`. This is intentional for backward compatibility but means local dev emails still use the root domain unless `.dev.vars` is updated.
3. **DMARC starts at `p=none`.** The SETUP.md recommends starting with monitoring mode. Enforcement (`p=reject`) should be enabled after 72h of clean DMARC reports, which is a manual follow-up step.
4. **Click/open tracking.** SETUP.md documents disabling Resend's click/open tracking, but this is a manual dashboard action, not enforced by code.
