## Rules Applied

- AC5 non-blocking enforced: `notifyExpertPoolDO` returns `void` synchronously, all async work in `ctx.waitUntil`
- Graceful degradation: `EXPERT_POOL_DO`, `EXPERT_DB`, `AI`, `VECTORIZE` all optional — no panic if not provisioned
- Named export from `index.ts` required for CF DO runtime class discovery (per CF Workers pattern)
- `[[migrations]]` top-level only in wrangler.toml (per CF Workflows conventions — same rule applies to DOs)

## Known Risks / Limitations

- **In-memory pending events**: if the DO is evicted before the alarm fires, pending events are lost. The cron sync (*/5 min) via Hyperdrive is the safety net for this scenario.
- **vectorize.ts duplication**: this file is also being added by story/E06S21. When the founder merges both PRs, a trivial conflict will need to be resolved. The files are identical in purpose and implementation.
- **Human action required**: no new CF resources needed beyond what E06S21 (Vectorize) and E06S23 (D1) require. DO is automatically provisioned on first deploy via `[[migrations]] tag = "v1"`. No manual `wrangler` commands needed.
