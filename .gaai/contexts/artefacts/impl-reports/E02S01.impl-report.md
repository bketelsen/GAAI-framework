## Rules Applied

- **Layer rule enforced:** Dashboard routes never import from Supabase DB directly. All data goes through `CORE_API_URL`. Supabase used only for auth via `@supabase/ssr`.
- **Cookie-based auth:** `@supabase/ssr` `createServerClient()` with `getAll`/`setAll` API used throughout. No external cookie library.
- **No client-side data fetching:** All data loading via React Router v7 loaders. No TanStack Query, Zustand, or client-side fetch.
- **Env typing:** `Env` interface in `worker-configuration.d.ts`. `AppLoadContext` declared in `workers/app.ts`.
- **Virtual module pattern:** `virtual:react-router/server-build` stubbed in `worker-configuration.d.ts` so `tsc -b` passes before build-time generation.

## Known Risks / Limitations

1. **Vite version deviation:** Plan specified `"vite": "^6.0.0"` but implementation uses `"^7.0.0"` (resolved to 7.3.1). Reason: root monorepo already has Vite v7 hoisted (from another workspace), and `@cloudflare/vite-plugin` v1.15.3 bundles Vite v7 as a peer. Using v6 created TS type conflicts on `hotUpdate` signature. QA should confirm Vite v7 compatibility with `@react-router/dev@7.9.6`.

2. **shadcn/ui CLI installed to `~/components/ui/`:** The shadcn CLI v3.8.5 resolved `~/` as a literal path (`~` directory) instead of expanding to `app/`. Files were copied to the correct `app/components/ui/` location. The `~` directory remains untracked and gitignored. It can be manually deleted.

3. **shadcn/ui added `next-themes` and `radix-ui`:** The CLI added these dependencies to `package.json` (beyond what the plan specified). These are shadcn/ui's current dependency model. `next-themes` is used by the Sonner wrapper for theme detection. `radix-ui` (v1.4.3) replaces individual `@radix-ui/*` packages.

4. **`requireSession` in `_layout.tsx` vs `_layout.dashboard.tsx`:** Both call `requireSession`. This is by design — the auth layout (`_layout.tsx`) guards all `_layout.*` routes, and the dashboard layout (`_layout.dashboard.tsx`) independently validates the session to extract user data for the sidebar. Double validation on authenticated requests is acceptable at this scaffold stage.

5. **`.react-router/` types not generated:** The `tsc -b` passes because the virtual module stub covers the `virtual:react-router/server-build` reference. Once `react-router typegen` is run (at dev/build time), it generates `.react-router/types/` which provides route-specific types. QA should run `npm run dev:dashboard` to trigger typegen.

6. **`SUPABASE_URL` / `SUPABASE_ANON_KEY` are placeholders:** Auth will fail at runtime until real values are substituted. Acceptable for scaffold stage.
