## Rules Applied

- Naming convention DEC-32: `callibrate-satellite-{env}` for Workers, `callibrate-satellite-kv-config-{env}` for KV
- KV cache-aside pattern (consistent with callibrate-core expertPool.ts pattern)
- HTML escaping on all config-sourced text content (defensive XSS prevention)
- Non-blocking KV write failures (consistent with callibrate-core patterns)
- Separate Worker deployment (independent from callibrate-core per DEC-53/54)

## Known Risks

- KV namespace IDs are PLACEHOLDER — human must provision and update wrangler.toml before first deploy
- Supabase RLS on satellite_configs must allow anon SELECT for the Worker to query on KV miss (current schema has RLS enabled with no policies — service role bypasses, but anon key used here needs verification)
- GitHub Actions `CLOUDFLARE_API_TOKEN` scope must include `callibrate-satellite-*` Worker names (may need token update)
