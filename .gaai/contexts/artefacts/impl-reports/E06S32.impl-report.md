## DB Migration

Migration file: `supabase/migrations/20260223100001_lead_credit_rpc.sql`

- `debit_lead_credit(p_expert_id, p_booking_id, p_prospect_id, p_amount)` → jsonb
  - SELECT FOR UPDATE on expert row (lock)
  - INSERT leads (ON CONFLICT booking_id: idempotent)
  - If balance < amount: return `{success:false, reason:'insufficient_balance', lead_id}`
  - If balance sufficient: UPDATE experts SET credit_balance -= amount + INSERT credit_transactions
  - Returns `{success:true, lead_id, balance_after}`
- `restore_lead_credit(p_expert_id, p_lead_id, p_amount, p_flag_reason)` → jsonb
  - SELECT FOR UPDATE on expert row
  - UPDATE experts SET credit_balance += amount
  - INSERT credit_transactions (type='lead_credit')
  - UPDATE leads SET status='flagged', flagged_at=NOW(), flag_reason
  - Returns `{success:true, balance_after}`

## Atomicity Note (AC3)

E06S18 (postgres.js/Hyperdrive) is not yet merged. To satisfy AC3 without E06S18, atomicity is implemented via PostgreSQL RPC functions called through `supabase.rpc()`. The PostgreSQL functions execute BEGIN/COMMIT internally (plpgsql SECURITY DEFINER), providing the same ACID guarantees as postgres.js `sql.begin()`.
