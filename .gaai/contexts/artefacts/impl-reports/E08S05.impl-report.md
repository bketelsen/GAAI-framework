## Known Risks and Issues

### AC7 — RLS on `satellite_configs`
The migration `20260224200001_satellite_configs_anon_select.sql` enables RLS and creates a permissive `anon_read_domain` policy with `USING (true)` — all rows visible to anon role. This is intentional (domain lookup requires reading all configured domains) but means satellite config metadata is publicly readable. If `satellite_configs` stores sensitive configuration, a more restrictive policy (e.g. `USING (active = true)`) should be considered in a future story.

### AC6 — `localConnectionString` in `wrangler.toml` removal
Removing `localConnectionString` from both `wrangler.toml` files means `wrangler dev` will fail to connect to the DB unless the developer manually re-adds it locally. The SETUP.md section documents this. The risk is developer confusion during onboarding. A future mitigation would be to switch to Supabase local dev (Docker) or use a `.gitignore`-safe overlay mechanism if Cloudflare adds support.

### AC9 — L2 webhook idempotency ordering
The L2 `SELECT webhook_events` check happens before the switch but the `INSERT webhook_events` happens after the switch completes. If the switch itself throws (e.g. LS API call fails in `handleSubscriptionCreated`), we fall into the catch block which returns 500 — and `webhook_events` is NOT inserted. On retry, the L2 check finds nothing and the event is reprocessed. This is correct behavior (we want to retry on failures), but means the L2 check only deduplicates successfully processed events, not failed ones. This is consistent with the KV L1 behavior and is the desired semantics.

### Audience validation breaks existing `prospect:submit` tokens in flight
After deployment, any JWT signed before the `aud` claim was added will fail `verifyProspectToken` because `claims.aud` will be `undefined` (not equal to `'prospect:matches'` or `'prospect:identify'`). Tokens have a 24h TTL, so this window is bounded. Prospects who submitted forms in the 24h before deployment will see a 403 on `/matches` or `/identify`. This is an acceptable tradeoff for the security improvement — the team should be aware of this brief breakage window at deploy time.
