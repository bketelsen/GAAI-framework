## Summary

Implemented PostHog client-side analytics instrumentation for satellite landing pages.

The work comprises:

1. A DB migration adding `tracking_enabled BOOLEAN NOT NULL DEFAULT true` to `satellite_configs`, enabling per-satellite opt-out of tracking (GDPR / debug mode).
2. Regenerated `src/types/database.ts` to reflect the new column (plus synced two other diffs from the remote schema: `comment` field on both survey tables, and `isOneToOne: true` on `lead_evaluations_lead_id_fkey`).
3. Extended the satellite worker's `SatelliteConfig` interface and `Env` interface with the new fields.
4. Documented `POSTHOG_API_KEY` as a wrangler secret in `wrangler.toml`.
5. Updated `renderLandingPage` to accept and inject the PostHog JS SDK (minified snippet) into `<head>` and a page-view + CTA-click capture script into `<body>`, both guarded by `tracking_enabled` and `posthogApiKey` truthiness.
6. Updated the call site in `index.ts` to pass `c.env.POSTHOG_API_KEY`.

SDK is configured for: reverse-proxy host `ph.callibrate.io`, cookieless persistence (`"memory"`), autocapture enabled, session replay enabled (`disable_session_recording: false`), and manual pageview capture with UTM and referrer properties.

## Post-Deployment Human Actions Required

1. **Set the wrangler secret on staging:**
   ```
   wrangler secret put POSTHOG_API_KEY --env staging
   ```
   Enter the PostHog project API key when prompted.

2. **Set the wrangler secret on production (when ready to promote):**
   ```
   wrangler secret put POSTHOG_API_KEY --env production
   ```

3. **Configure PostHog reverse proxy** (`ph.callibrate.io`) to forward requests to `eu.posthog.com`. This is required before events will be received. The SDK will fail silently without the proxy, but no errors surface to the user.

4. **Purge KV cache** for any satellite that already has a cached config (the new `tracking_enabled` field is `NOT NULL DEFAULT true`, so existing rows in the DB are already correct; however KV-cached entries will not include the field until TTL expires or cache is purged):
   ```
   POST /admin/cache/purge
   x-admin-secret: <ADMIN_SECRET>
   {"domain": "<satellite-domain>"}
   ```

5. **Verify in PostHog** that `page_view` events are appearing with correct `satellite_id`, `utm_*`, and `referrer` properties after deploying.
