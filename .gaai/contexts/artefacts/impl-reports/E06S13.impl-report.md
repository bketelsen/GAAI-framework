### AC1 — Migration adds 6 new columns ✅

```sql
ALTER TABLE satellite_configs
  ADD COLUMN theme           JSONB,
  ADD COLUMN brand           JSONB,
  ADD COLUMN content         JSONB,
  ADD COLUMN structured_data JSONB,
  ADD COLUMN active          BOOLEAN NOT NULL DEFAULT false,
  ADD COLUMN updated_at      TIMESTAMPTZ DEFAULT now();
```

## Known Limitations

- `updated_at` is application-managed (set by Worker on updates). No DB trigger. Per Story notes: "updated_at can be managed at application level rather than via a DB trigger, to avoid migration complexity."
- Seed UPDATE targets `WHERE id = 'default'`. If the 'default' row does not exist (e.g., after DB reset), the UPDATE is a no-op (0 rows affected). Not an error, but seed may be absent.
- `logo_url: null` in seed theme — operator must set this when configuring a satellite.
