## What Was Implemented

### 1. `scripts/smoke-test-staging.sh`

Created an idempotent, macOS-compatible smoke test script covering AC1–AC9. Key design decisions:

- **Persistent smoke-runner user** (`smoke-runner@callibrate.io` / `SmokeRunner!2026`): Uses sign-in (not sign-up) to avoid Supabase email domain validation failures on each run. Expert register handles 409 gracefully (user already exists → reuses `USER_ID` from JWT as `EXPERT_ID`).
- **macOS-compatible curl parsing**: Uses `mktemp` + `-o "$TMP_FILE"` + `-w "%{http_code}"` pattern instead of `head -n -1` (not supported on BSD head).
- **Turnstile bypass**: Staging uses Cloudflare test secret key `1x0000000000000000000000000000000AA` — any token string passes verification. Script uses literal `"SMOKE.TEST.TOKEN.E06S29"`.
- **No-Origin header**: CORS middleware allows requests with no `Origin` header (server-to-server mode). All curl calls omit Origin.
- **Satellite quiz payload**: `POST /api/prospects/submit` requires `quiz_answers` keyed by `question_id` from the satellite config. Script fetches `GET /api/satellites/default/config` to extract quiz schema dynamically, building the quiz_answers payload at runtime.

### 2. Supabase smoke-runner user setup (one-time, via MCP)

The smoke test user was pre-created in the staging Supabase project via admin SQL:

```sql
-- auth.users row with bcrypt password hash for SmokeRunner!2026
INSERT INTO auth.users (instance_id, id, aud, role, email, encrypted_password, ...)
  VALUES ('00000000-0000-0000-0000-000000000000', '36916832-7f19-478c-875c-497e9457169c',
          'authenticated', 'authenticated', 'smoke-runner@callibrate.io', '$2a$10$...', ...);

-- auth.identities record (required for password sign-in)
INSERT INTO auth.identities (provider_id, user_id, identity_data, provider, ...)
  VALUES ('smoke-runner@callibrate.io', '36916832-7f19-478c-875c-497e9457169c', ...);

-- Fix NULL token columns (Supabase auth requires empty string, not NULL)
UPDATE auth.users SET
  confirmation_token = '', recovery_token = '', email_change_token_new = '',
  email_change_token_current = '', phone_change_token = '', reauthentication_token = '',
  email_change = ''
WHERE email = 'smoke-runner@callibrate.io';
```

User ID: `36916832-7f19-478c-875c-497e9457169c`

## Issues Encountered and Resolved

| Issue | Root Cause | Fix |
|-------|-----------|-----|
| `head -n -1` fails on macOS | BSD head doesn't support negative line numbers | Switched to `mktemp` temp file pattern |
| Email domain rejected by Supabase | `@smoke.callibrate.io` has no MX records | Created `@callibrate.io` smoke user (valid domain) |
| Sign-in HTTP 500 "Database error" | `confirmation_token` was NULL — Supabase auth requires `''` | `UPDATE auth.users SET confirmation_token = '', ...` |
| Sign-in HTTP 500 (persisting) | Missing `auth.identities` record | Inserted identity row for email provider |
