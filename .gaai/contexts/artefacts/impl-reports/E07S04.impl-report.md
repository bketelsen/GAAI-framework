## Rules Applied

- Strict TypeScript — `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`, all strict flags
- `process.env` access uses optional property pattern with explicit undefined checks (AC5 fail-fast)
- No external HTTP libraries — native `fetch` (Node 18+, already in project environment)
- No CF Worker patterns (env bindings, KV, Queues) — this is a plain Node.js script
- Idempotency implemented at dashboard level (not insight level) — correct for a one-time setup tool
- Error surface: `throw new Error('PostHog API error {status} at {path}: {body}')` on non-2xx; main catches and exits with code 1

## Known Limitations / Notes

- **Insight idempotency:** When a dashboard already exists, its insights are NOT inspected or deduplicated. Idempotency operates at the dashboard level (skip entire dashboard if name exists). This is the correct behavior for a one-time setup script — if insights need adjustment, the founder modifies them in the PostHog UI.
- **Pagination:** `listAllDashboards()` fetches `?limit=100` — sufficient for this project (3 dashboards expected). A project with >100 dashboards would require pagination, but this is not a realistic scenario here.
- **Business Overview conversion rate:** Formula `B/A*100` (booking.confirmed / prospect.form_submitted × 100). Event order matters for formula mapping: A = first event (index 0), B = second event (index 1).
- **`tsx --check` flag:** Not supported in tsx v4.21. TypeScript validation performed via `npx tsc --project scripts/tsconfig.json --noEmit` → EXIT:0 (clean).
