# Execution Plan — E06S26: Analytics Engine — Matching Observability

## Tier
Tier 1 — MicroDelivery (complexity 2, files 4, criteria 5)

## Approach

Analytics Engine is a Cloudflare-native, fire-and-forget observability primitive. Implementation requires:

1. **wrangler.toml** — add `[[analytics_engine_datasets]]` binding `MATCHING_ANALYTICS` for dev, staging, and production environments.
2. **src/types/env.ts** — add `MATCHING_ANALYTICS?: AnalyticsEngineDataset` (optional for graceful no-op degradation).
3. **src/lib/matchingAnalytics.ts** — create `writeMatchingDataPoint()` helper: wraps `writeDataPoint()` in try/catch, uses optional chaining for AC4.
4. **src/routes/matches.ts** — instrument `handleMatchCompute`: capture `startTime`, compute `topScore`/`meanScore` from scored array, call `writeMatchingDataPoint` before returning.
5. **src/routes/prospects.ts** — instrument `handleProspectSubmit`: same pattern, using `scoredResults` array captured from the scoring map.

## Data Schema

- `blobs[0]` = satellite_id
- `blobs[1]` = endpoint (`/api/matches/compute` or `/api/prospects/submit`)
- `doubles[0]` = latency_ms
- `doubles[1]` = pool_size (number of experts scored)
- `doubles[2]` = top_score
- `doubles[3]` = mean_score

## No-op Pattern (AC4)

```typescript
env.MATCHING_ANALYTICS?.writeDataPoint({ blobs: [...], doubles: [...] });
```
Optional chaining makes this a no-op when binding is absent. Entire call is also wrapped in `try/catch`.

## Test Strategy

No new tests required — existing 79 vitest tests cover the handlers. `MATCHING_ANALYTICS` is undefined in test mocks, verifying the no-op behavior implicitly.
