## Approach Rationale — AES-256-GCM Encryption Format

**Specified by the Story:** `crypto.subtle.encrypt` (AES-256-GCM) with `GCAL_TOKEN_ENCRYPTION_KEY`. IV stored alongside ciphertext (concatenated, base64-encoded).

**Storage format:**
```
base64(IV || ciphertext)
```
Where:
- `IV` = 12-byte random Initialization Vector (AES-GCM standard — 96-bit IV per NIST SP 800-38D)
- `ciphertext` = output of `crypto.subtle.encrypt({ name: 'AES-GCM', iv }, key, plaintext)`
- `||` = byte concatenation
- Result: standard base64 for DB text field storage

**Key import:** `GCAL_TOKEN_ENCRYPTION_KEY` Worker secret = base64-encoded 32-byte key (256 bits).
```
crypto.subtle.importKey('raw', base64Decode(env.GCAL_TOKEN_ENCRYPTION_KEY), { name: 'AES-GCM' }, false, ['encrypt', 'decrypt'])
```

**Encryption helper:** Encapsulated in `src/lib/gcalCrypto.ts` as `encryptToken(plaintext, rawKey)` and `decryptToken(stored, rawKey)`. Both async, both accept the raw base64-string key directly (import inside the function). Reusable by callback handler, refresh utility, and disconnect handler.

### Step 4 — Encryption Utility (AC2, AC5, AC7, AC8)

**File:** `src/lib/gcalCrypto.ts` (NEW)

**Exports:**

`encryptToken(plaintext: string, rawKey: string): Promise<string>`
1. Decode `rawKey` from base64 → `Uint8Array`
2. Import as AES-GCM CryptoKey with `['encrypt']` usage
3. Generate 12-byte random IV: `crypto.getRandomValues(new Uint8Array(12))`
4. Encrypt: `crypto.subtle.encrypt({ name: 'AES-GCM', iv }, key, new TextEncoder().encode(plaintext))`
5. Concatenate: `new Uint8Array([...iv, ...new Uint8Array(ciphertext)])`
6. Return `btoa(String.fromCharCode(...combined))`

`decryptToken(stored: string, rawKey: string): Promise<string>`
1. Base64-decode `stored` → `Uint8Array`
2. Split: first 12 bytes = IV, remainder = ciphertext
3. Import key with `['decrypt']` usage
4. Decrypt: `crypto.subtle.decrypt({ name: 'AES-GCM', iv }, key, ciphertext)`
5. Return `new TextDecoder().decode(decrypted)`
6. Wrap in try/catch → throw typed `GcalDecryptionError` on failure (token corruption)

**Edge cases:**
- Invalid/corrupted stored data → `crypto.subtle.decrypt` throws → `GcalDecryptionError`
- `rawKey` not valid base64 → `atob` throws → let propagate (misconfiguration)

### Step 5 — Token Refresh Utility (AC5, AC6 contract)

**File:** `src/lib/gcalRefresh.ts` (NEW)

**Export:** `refreshGcalToken(expertId: string, env: Env): Promise<string>`

Steps:
1. Query DB: `supabase.from('experts').select('gcal_refresh_token').eq('id', expertId).single()`
2. If not found or `gcal_refresh_token` is null → throw `GcalNotConnectedError`
3. Decrypt stored refresh token via `decryptToken(storedRefreshToken, env.GCAL_TOKEN_ENCRYPTION_KEY)`
4. POST to `https://oauth2.googleapis.com/token`:
   ```
   { refresh_token, client_id: env.GOOGLE_CLIENT_ID, client_secret: env.GOOGLE_CLIENT_SECRET, grant_type: 'refresh_token' }
   ```
5. If non-200 from Google → throw `GcalRefreshError`
6. Extract `access_token` and `expires_in` from response
7. Compute `new_expiry_at = new Date(Date.now() + (expires_in ?? 3600) * 1000).toISOString()`
8. Encrypt new access token via `encryptToken(access_token, env.GCAL_TOKEN_ENCRYPTION_KEY)`
9. Update DB: `{ gcal_access_token: encryptedToken, gcal_token_expiry_at: new_expiry_at }`
10. Return raw (unencrypted) `access_token`

**AC6 caller contract (NOT in this utility — documented for E06S11 implementor):**
```typescript
// In E06S11 and any future Google Calendar API caller:
let result = await callGoogleCalendarApi(accessToken);
if (result.status === 401 || result.status === 403) {
  try {
    accessToken = await refreshGcalToken(expertId, env);
    result = await callGoogleCalendarApi(accessToken);
    if (result.status === 401 || result.status === 403) {
      await supabase.from('experts').update({ gcal_connected: false }).eq('id', expertId);
      return { error: 'gcal_disconnected' };
    }
  } catch {
    await supabase.from('experts').update({ gcal_connected: false }).eq('id', expertId);
    return { error: 'gcal_disconnected' };
  }
}
```

### Step 6 — GCal Route Handlers (AC1, AC2, AC3, AC4, AC7)

**File:** `src/handlers/experts/gcal.ts` (NEW)

#### handleGcalAuthUrl(request, env, user, expertId) — AC1

1. Ownership guard: `if (user.id !== expertId)` → 403
2. Verify expert exists in DB → 404 if not found
3. Build Google OAuth2 authorization URL:
   - Base: `https://accounts.google.com/o/oauth2/v2/auth`
   - Params: `client_id`, `redirect_uri: ${env.WORKER_BASE_URL}/api/gcal/callback`, `response_type: code`, `scope: https://www.googleapis.com/auth/calendar.events https://www.googleapis.com/auth/calendar.readonly`, `access_type: offline`, `prompt: consent`, `state: expertId`
4. Return `200 { auth_url: string }`

#### handleGcalCallback(request, env) — AC2, AC3

**Unauthenticated.** Placed OUTSIDE the `/api/experts/` authenticated block.

1. Parse `code` and `state` from URL query params
2. Validate `state` is non-empty string → redirect to `/onboarding/gcal-error?reason=invalid_state` if missing
3. Validate `code` is present → redirect to `/onboarding/gcal-error?reason=no_code` if missing
4. Verify expert exists in DB using `state` as `expertId` → redirect to `gcal-error?reason=expert_not_found` if missing
5. Exchange `code` for tokens: POST `https://oauth2.googleapis.com/token` with `{ code, client_id, client_secret, redirect_uri, grant_type: 'authorization_code' }`
6. On non-200 → redirect to `gcal-error?reason=token_exchange_failed`
7. Extract `{ access_token, refresh_token, expiry_date }` from response
8. Encrypt `access_token` via `encryptToken`
9. **AC3:** Only encrypt+store `refresh_token` if present AND non-null in response:
   - If present → `encryptedRefreshToken = await encryptToken(refresh_token, env.GCAL_TOKEN_ENCRYPTION_KEY)`
   - If absent or null → `encryptedRefreshToken = undefined` (OMIT from update object)
10. Fetch `gcal_email` via `GET https://www.googleapis.com/oauth2/v3/userinfo` with `Authorization: Bearer {access_token}` → non-blocking (proceed on failure with `gcal_email: null`)
11. Build update object:
    - Always: `gcal_access_token`, `gcal_token_expiry_at: expiry_date ? new Date(expiry_date).toISOString() : null`, `gcal_connected: true`, `gcal_connected_at: new Date().toISOString()`
    - Conditional: `gcal_refresh_token` (only if `encryptedRefreshToken` is defined), `gcal_email` (only if email obtained)
12. Update DB → on error redirect to `gcal-error?reason=db_error`
13. On success → redirect to `/onboarding/gcal-connected`

#### handleGcalStatus(request, env, user, expertId) — AC4

1. Ownership guard → 403
2. Query DB: `select('gcal_connected, gcal_email, gcal_connected_at')` → `PGRST116` → 404
3. Return `{ connected: data.gcal_connected ?? false, google_email: data.gcal_email ?? null, connected_at: data.gcal_connected_at ?? null }`

#### handleGcalDisconnect(request, env, user, expertId) — AC7

1. Ownership guard → 403
2. Fetch expert: `select('gcal_refresh_token')` → 404 if not found
3. If `gcal_refresh_token` is non-null:
   a. Decrypt via `decryptToken` (swallow `GcalDecryptionError` — proceed to clear)
   b. POST `https://oauth2.googleapis.com/revoke?token={decryptedToken}` — non-blocking (proceed on 400)
4. Update DB: `{ gcal_connected: false, gcal_access_token: null, gcal_refresh_token: null, gcal_token_expiry_at: null, gcal_email: null, gcal_connected_at: null }`
5. Return `200 { disconnected: true }`
