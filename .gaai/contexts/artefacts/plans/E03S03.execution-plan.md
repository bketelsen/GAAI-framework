## Technical Decisions

- **TD-1:** Function signature `renderResultsPage(config, posthogApiKey, coreApiUrl)` — no Turnstile needed
- **TD-2:** SessionStorage guard on page load (prospect_id + token) → redirect to `/match` if missing
- **TD-3:** 202 polling: exponential backoff `delay = (estimated_seconds * 1000 || 3000) * 2^retryCount`, max 3 retries
- **TD-4:** Two-phase DOM rendering — Phase 1: anonymized cards + email gate; Phase 2: in-place card mutation after identify
- **TD-5:** Skeleton shimmer — 3 static skeleton cards replaced by real cards on API success
- **TD-6:** Email gate section pre-rendered in HTML (hidden), revealed after cards render
- **TD-7:** Inline email validation: `type="email"` + regex before submit
- **TD-8:** E03S06 forward-compat booking stub — CTA fires `satellite.booking_cta_clicked` + dispatches `CustomEvent('booking-open', { detail: { expertId } })` on `window`
- **TD-9:** Tier badge/avatar color system: top=#F59E0B, confirmed=#64748B, promising=#7C3AED
- **TD-10:** Avatars: SVG silhouette (anonymized), initial-based colored circle (post-identify)
- **TD-11:** Score bars via `<div class="score-bar-fill" style="width:{score}%">`
- **TD-12:** No-matches state on empty array; computing fallback after 3 retries
