# Execution Plan — E06S05: Matching Engine Scoring Function + Match API

## Implementation Sequence

### Step 1: Add vitest test framework
- **Files:** `package.json`
- **Action:** Add `vitest` to devDependencies, add `"test": "vitest run"` script
- **Why:** AC11 requires unit tests. scoreMatch is a pure function — vitest covers it with zero CF Worker dependency.

### Step 2: Remove MATCHING_JOBS from env.ts (DEC-33 cleanup)
- **Files:** `src/types/env.ts`
- **Action:** Remove `MATCHING_JOBS: Queue` binding — matching-jobs queue was deleted (DEC-33). This was a leftover from E06S02.
- **Why:** MATCHING_JOBS no longer exists in wrangler.toml. Keeping it in the type causes TS errors or silent mismatch.

### Step 3: Create matching domain types
- **Files:** `src/types/matching.ts` (new)
- **Types to define:**
  - `ExpertProfile` — matches `experts.profile` JSONB shape (industries, skills, project_types, languages)
  - `ExpertPreferences` — matches `experts.preferences` JSONB shape (excluded_industries, languages, etc.)
  - `ProspectRequirements` — matches `prospects.requirements` JSONB shape (skills_needed, industry, budget_range, timeline, languages)
  - `MatchingWeights` — `{ skills_overlap, industry_match, budget_compatibility, timeline_match, language_match }` all number
  - `ScoreBreakdown` — per-component scores + optional deal_breaker flag
  - `MatchScore` — `{ score: number, breakdown: ScoreBreakdown }`
  - `QualityTier` — `'new' | 'rising' | 'established' | 'top'`
  - `MatchResult` — anonymized: skills, industries, project_types, score, score_breakdown, quality_tier (no display_name, avatar_url, cal_username)
  - `DEFAULT_WEIGHTS` — constant with default max-point values per AC2

### Step 4: Implement scoreMatch pure function
- **Files:** `src/matching/score.ts` (new)
- **Function signature (AC1):**
  ```
  scoreMatch(expertProfile, expertPreferences, expertRates, prospectRequirements, weights): MatchScore
  ```
  Note: rate_min/rate_max live on the expert row, not in profile JSONB, so pass them explicitly.
- **AC2:** Components: skills_overlap (max 40), industry_match (max 20), budget_compatibility (max 20), timeline_match (max 10), language_match (max 10)
- **AC3:** skills_overlap = (matching skills count / prospect.skills_needed.length) * weight; 0 if prospect has no skills_needed
- **AC4:** industry_match = full if expert.profile.industries includes prospect.requirements.industry; 0 otherwise
- **AC5:** budget_compatibility:
  - expert_monthly = rate_min * 20
  - 0 if no budget_range or no rate_min
  - full if budget_range.min <= expert_monthly <= budget_range.max
  - partial (weight * 0.5) if expert cost range [rate_min*20, rate_max*20] partially overlaps budget_range
  - 0 if no overlap
- **AC6:** deal-breaker: if expert.preferences.excluded_industries includes prospect.requirements.industry → score = 0, breakdown.deal_breaker = true (short-circuit before scoring)
- **timeline_match:** if prospect has no timeline requirement → full points; if expert.preferences has accepted_timelines → check inclusion; no expert preference → full points
- **language_match:** if prospect has no language requirement → full points; ratio of matching languages * weight; 0 if expert has no languages listed

### Step 5: Write unit tests
- **Files:** `src/matching/score.test.ts` (new)
- **Test cases (AC11):**
  1. Full match — all criteria perfectly aligned → score = 100
  2. Zero match — no skill/industry/budget overlap, no deal-breaker → score = 0
  3. Deal-breaker — expert.preferences.excluded_industries contains prospect.industry → score = 0, deal_breaker = true
  4. Partial skills overlap — 2 of 4 required skills match → skills_overlap = 20
  5. Missing optional fields — prospect has no languages/timeline, expert has minimal profile → graceful 0s, no runtime errors

### Step 6: Create matches route handlers
- **Files:** `src/routes/matches.ts` (new)
- **POST /api/matches/compute (AC7):**
  1. Parse + validate body: `{ prospect_id: string, satellite_id?: string }`
  2. Load prospect from DB → extract requirements JSONB
  3. Load satellite_configs.matching_weights if satellite_id provided; else DEFAULT_WEIGHTS
  4. Load all experts WHERE availability != 'unavailable'
  5. For each expert: cast profile/preferences JSONB → types, call scoreMatch
  6. Sort by score DESC, take top 20
  7. Upsert top-20 into matches table (delete existing + insert, or upsert)
  8. Return `{ computed: number, top_matches: MatchResult[] }`
- **GET /api/matches/:prospect_id (AC8, AC9, AC12):**
  1. Parse prospect_id from path
  2. Parse `page` query param (default 1), page_size = 5
  3. Query matches WHERE prospect_id = ? AND status != 'expired' ORDER BY score DESC
  4. For each match: join experts to get composite_score + profile fields
  5. Derive quality_tier from composite_score (AC12)
  6. Anonymize: strip display_name, avatar_url, cal_username (AC9)
  7. Return paginated result

### Step 7: Wire routes into src/index.ts
- **Files:** `src/index.ts`
- Import `handleMatchCompute`, `handleMatchGet` from routes/matches.ts
- Add route handling:
  - `POST /api/matches/compute` → handleMatchCompute
  - `GET /api/matches/:prospect_id` → handleMatchGet (pattern match on pathname)

### Step 8: Install dependencies and type-check
- `npm install` (to install vitest)
- `npm run type-check` (tsc --noEmit)
- `npm run test` (vitest run — unit tests only, no CF Worker bindings needed)

## Edge Cases per Acceptance Criterion

| AC | Edge Case | Handling |
|----|-----------|----------|
| AC1 | expert has null profile/preferences | Cast to empty object `{}` before scoring |
| AC3 | prospect.skills_needed is empty array | Return 0 for skills_overlap (per spec) |
| AC3 | expert has no skills | 0 matching → 0 score component |
| AC4 | expert.profile.industries is null | 0 for industry_match |
| AC5 | expert has rate_min but no rate_max | Use rate_min only for range |
| AC5 | budget_range has only max (no min) | Treat min as 0 |
| AC6 | expert.preferences is null | No deal-breaker check → continue scoring |
| AC7 | prospect_id not found | Return 404 |
| AC7 | satellite_id provided but not found in DB | Fall back to DEFAULT_WEIGHTS |
| AC7 | No experts available | Return { computed: 0, top_matches: [] } |
| AC8 | prospect_id has no matches | Return empty paginated result |
| AC8 | page beyond last page | Return empty results with pagination meta |
| AC10 | matching_weights in satellite_configs is null | Fall back to DEFAULT_WEIGHTS |
