## 2. Technical Decisions

### TD-1: confirm.ts signature
`renderConfirmPage(config: SatelliteConfig, posthogApiKey: string, coreApiUrl: string, turnstileSiteKey: string): string`

Follows the exact match.ts signature pattern. `turnstileSiteKey` is injected server-side into `window.__SAT__` alongside `apiUrl` and `satelliteId`.

### TD-2: quiz_answers field format
The `POST /api/prospects/submit` endpoint accepts `quiz_answers` and normalizes via `normalizeRequirements()`, which picks known field names: `challenge`, `skills_needed`, `industry`, `budget_range`, `timeline`, `company_size`, `languages`, `desired_outcomes`. The satellite sends extracted requirements directly as `quiz_answers` (the merged/confirmed requirements object). No format translation needed. For freetext satellites, `quiz_schema` is null → no required fields validation.

### TD-3: Turnstile widget approach
Use `appearance: 'interaction-only'`. The Turnstile container div is hidden (`display:none`) until the prospect clicks "Confirmer". On click, container becomes visible, `turnstile.render()` fires. The SDK (`https://challenges.cloudflare.com/turnstile/v0/api.js`) is loaded async in `<head>`. Widget is rendered on-click, not on page load, to avoid pre-solving a challenge that expires before submission. If SDK not yet loaded at click time, poll with 250ms interval (max 10s timeout).

### TD-4: Turnstile token lifecycle
1. Prospect clicks "Confirmer"
2. `turnstile.render('#cf-turnstile-container', { sitekey, callback, 'error-callback', appearance: 'interaction-only' })`
3. `callback(token)` fires on success → immediately call `POST /api/prospects/submit`
4. On API error: `turnstile.reset(widgetId)` for retry
5. On Turnstile error-callback: show error, reset for retry

### TD-5: Edit-in-place (AC4)
Each field row has display state (value + "Modifier" button) and edit state (input visible, display hidden). State tracked in JS `mergedReqs` object. Clicking "Modifier" switches that row to edit mode. Fields in `needs_confirmation` start in edit mode (question shown). Fields in the summary start in display mode.

### TD-6: UTM parameter handling
E03S01's match.ts reads UTMs for PostHog but does NOT persist to sessionStorage. E03S02 adds a minimal additive change to match.ts: persist UTMs to `sessionStorage['match:utm']` on page load. The confirm page reads `match:utm` from sessionStorage to include in the submit payload.

### TD-7: Max 3 questions constraint (AC3)
Only first 3 entries from `needs_confirmation` are rendered as questions. Fields beyond index 2 still appear in the summary with "Modifier" affordance.

### TD-8: Navigation target (AC7)
`window.location.href = '/results'` — matching E03S03's expected route. `prospect_id` and `token` stored in sessionStorage as `match:prospect_id` and `match:token`.

### TD-9: TURNSTILE_SITE_KEY as `[vars]` not a secret
Site key is public (embedded in client-side HTML). Added to `wrangler.toml` as `[vars]` entries. The Turnstile secret key lives on the Core API Worker. Placeholder values in wrangler.toml must be replaced with real keys before deployment.

### TD-10: CSP update for Turnstile
Add to `HTML_CSP`:
- `script-src`: add `https://challenges.cloudflare.com`
- `frame-src`: add `https://challenges.cloudflare.com` (Turnstile renders in an iframe)

**Client-side JS IIFE key logic:**
- On page load: read `match:extraction` from sessionStorage; redirect to `/match` if absent
- Build `mergedReqs` from `extraction.requirements`
- Render summary fields: FIELD_ORDER = ['challenge', 'skills_needed', 'industry', 'budget_range', 'timeline', 'company_size', 'languages']
- Each field row: confidence indicator (✅ ≥0.7, 🟠 0.4–0.7, ❓ <0.4) + label + value + "Modifier" button + hidden edit input
- Render confirmation questions (max 3): text input, radio/select, or budget dual inputs based on `options` field
- On "Modifier" click: activateEditMode(field), fire `satellite.confirmation_field_edited`
- PostHog `satellite.confirmation_viewed` on page load
- On "Confirmer" click: collectMergedReqs(), set isSubmitting, disable button, show Turnstile, await token
- On Turnstile callback: fire `satellite.confirmation_submitted`, call `POST /api/prospects/submit` with `quiz_answers: mergedReqs` + UTMs from `sessionStorage['match:utm']`
- On success: store `match:prospect_id`, `match:token`, fire `satellite.prospect_created`, navigate to `/results`
- On error: restore form, show error message per error type (AC8)
- escapeHtmlJs() function in IIFE for client-side HTML escaping

| CSP blocks Turnstile iframe | CERTAIN if not updated | HIGH | Step 3: add frame-src directive |
