# Execution Plan — E06S07: Satellite Funnel API

## Tier Assessment

**Tier 2** — complexity 7
- 10 ACs
- 4 new routes across 2 new route files
- JWT signing/verification (Web Crypto API — no external library)
- CORS dynamic origin validation (DB-backed satellite domain whitelist)
- New KV binding (EXPERT_POOL) + expert pool cache/fallback pattern
- Synchronous scoreMatch() orchestration + DB inserts in single request

**Risk flags:**
- JWT implementation uses raw Web Crypto API — test surface required
- CORS DB query on every request (acceptable at MVP scale; no extra KV cache added — YAGNI)
- EXPERT_POOL KV IDs are placeholder (human must provision via SETUP.md)

### Phase 1 — JWT utility
**File:** `src/lib/jwt.ts` (NEW)

Functions:
- `signProspectToken(prospectId, secret): Promise<{ token: string; expiresAt: string }>`
  - Builds JWT: `{ alg: 'HS256', typ: 'JWT' }` header + `{ prospect_id, exp: now + 86400 }` payload
  - Signs with HMAC-SHA256 via Web Crypto API (`crypto.subtle`)
  - Returns base64url-encoded token + expiresAt ISO string
- `verifyProspectToken(token, prospectId, secret): Promise<boolean>`
  - Splits token into 3 parts; returns false if malformed
  - Verifies HMAC-SHA256 signature
  - Checks `exp` not in past
  - Checks `claims.prospect_id === prospectId`
- Helper: `toBase64Url(data: Uint8Array | ArrayBuffer): string`
- Helper: `fromBase64Url(str: string): Uint8Array`

Edge cases:
- fromBase64Url on malformed input → wrapped in try/catch → returns false
- Token with wrong prospect_id → returns false (not 401 ambiguity)
- Expired token → returns false

Test checkpoint: `src/lib/jwt.test.ts`

### Phase 2 — CORS utility
**File:** `src/lib/cors.ts` (NEW)

Functions:
- `validateOrigin(origin: string, env: Env): Promise<boolean>`
  - `*.callibrate.io` → `origin.endsWith('.callibrate.io')` OR `origin === 'https://callibrate.io'`
  - DB query: `satellite_configs` where `domain = originHost` (strip protocol)
  - Returns true if allowed, false otherwise
- `corsHeaders(origin: string): Record<string, string>`
  - Returns `{ 'Access-Control-Allow-Origin': origin, 'Access-Control-Allow-Methods': 'GET, POST, OPTIONS', 'Access-Control-Allow-Headers': 'Content-Type, Authorization', 'Access-Control-Max-Age': '86400' }`
- `handleCors(request, env): Promise<{ allowed: boolean; origin: string | null; preflight: Response | null }>`
  - No Origin header → `{ allowed: true, origin: null, preflight: null }` (server-to-server)
  - OPTIONS preflight → validate origin; if valid return 204 preflight response; if not → 403
  - Other methods → validate origin; return allowed/forbidden result

Edge cases:
- Requests with no Origin header are allowed (direct server calls, health checks)
- Origin comparison is case-insensitive on the host portion
- DB query failure → default to deny for security

### Phase 3 — Expert pool loader
**File:** `src/lib/expertPool.ts` (NEW)

Type:
```typescript
export interface ExpertPoolEntry {
  id: string;
  profile: Json | null;
  preferences: Json | null;
  rate_min: number | null;
  rate_max: number | null;
  composite_score: number | null;
}
```

Functions:
- `loadExpertPool(env: Env): Promise<ExpertPoolEntry[]>`
  - Try: `await env.EXPERT_POOL.get('expert_pool', { type: 'json' })`
  - Cache hit: return parsed array
  - Cache miss: query Supabase `experts` where `availability != 'unavailable'`, select `id, profile, preferences, rate_min, rate_max, composite_score`
  - Write result to KV with TTL 300s: `await env.EXPERT_POOL.put('expert_pool', JSON.stringify(experts), { expirationTtl: 300 })`
  - Return experts array

Edge cases:
- Empty expert pool → return [] (caller handles gracefully)
- KV write failure → non-blocking (log would be ideal but no logger at MVP) — proceed with DB data
- DB error → propagate as empty array (defensive) → caller returns empty matches, AC6 guard kicks in

### Phase 4 — Satellite config route
**File:** `src/routes/satellites.ts` (NEW)

Functions:
- `handleSatelliteConfig(request: Request, env: Env, satelliteId: string): Promise<Response>`
  - Query `satellite_configs` by `id`

### Phase 5 — Prospect funnel routes
**File:** `src/routes/prospects.ts` (NEW)

#### 5a. `handleProspectSubmit(request, env): Promise<Response>`
Steps:
1. Parse body: `{ satellite_id, quiz_answers, utm_source?, utm_campaign? }`
2. Validate `satellite_id`: non-empty string → 422 if missing
3. Validate `quiz_answers`: object → 422 if missing
4. Load satellite config → 404 if not found
5. Extract required keys from `quiz_schema`:
   - quiz_schema is `Record<string, { required: boolean, ... }>` — filter keys where `required === true`
   - Check quiz_answers has all required keys → 422 with `{ error: "Validation failed", details: { missing_fields: [...] } }` if not
6. Normalize quiz_answers → requirements: pick known ProspectRequirements fields from quiz_answers directly (type cast)
7. Create prospect: INSERT into `prospects` with `{ satellite_id, quiz_answers, requirements, status: 'anonymous', utm_source, utm_campaign }`
8. Load expert pool via `loadExpertPool(env)`
9. Score each expert with `scoreMatch()` using satellite's `matching_weights` (or DEFAULT_WEIGHTS)
10. INSERT all scored results into `matches` table
11. Sign JWT token via `signProspectToken(prospect.id, env.PROSPECT_TOKEN_SECRET)`
12. Return `{ prospect_id: prospect.id, token, token_expires_at }`

Edge cases:
- Empty expert pool → matches table will be empty → AC6 guard in GET /matches will return 202
- INSERT matches failure → treat as partial success (AC6 covers this)
- satellite_id not found → 404 before creating prospect

#### 5b. `handleProspectMatches(request, env, prospectId): Promise<Response>`
Steps:
1. Extract `token` from query string
2. If no token → 403
3. Verify token via `verifyProspectToken(token, prospectId, env.PROSPECT_TOKEN_SECRET)` → 403 if invalid
4. Load matches from `matches` table: `prospect_id = prospectId`, status != 'expired', ORDER BY score DESC
5. If matches.length === 0 → 202 `{ status: "computing", estimated_seconds: 5 }` (AC6 guard)
6. Load expert profiles for anonymized fields: `experts.select('id, composite_score, profile, rate_min, rate_max').in('id', expertIds)`
7. Build anonymized cards: `{ rank, score, score_breakdown, skills, industries, project_types, languages, rate_min, rate_max }`
8. Return `{ matches: [...] }`

Edge cases:
- Missing token → 403 (not 422 — matches the "token required" security gate)
- Expired token → 403
- No matches (AC6) → 202 (not 404 — defensive guard for empty pool/error cases)

#### 5c. `handleProspectIdentify(request, env, prospectId): Promise<Response>`
Steps:
1. Parse body: `{ email, token }`
2. Verify token → 403 if invalid
3. Validate email format (regex or basic check) → 422 if invalid
4. Load prospect: check `prospects.email` — if not null → 409 `{ error: "Prospect already identified" }` (AC8)
5. Update prospect: `email = email, status = 'identified'`
6. Load matches for prospectId to get expert_ids (active, score DESC)
7. Load full expert profiles: `experts.select('id, display_name, headline, bio, profile, preferences, rate_min, rate_max, composite_score, cal_username').in('id', expertIds)`
8. Join with match scores
9. Build full profiles: `{ display_name, headline, bio, skills, industries, rate_min, rate_max, score, booking_url }`
   - `booking_url = https://cal.com/${expert.cal_username}/intro-call` (AC7)
10. Return `{ experts: [...] }`

Edge cases:
- cal_username null → booking_url null or omitted (expert not yet onboarded to Cal)
- Email already set → 409 before DB update

## Risk Register

| Risk | Mitigation |
|---|---|
| JWT Web Crypto in CF Workers | Use `crypto.subtle` (available in CF Workers runtime) — no import needed |
| EXPERT_POOL KV not provisioned | Placeholder IDs in wrangler.toml + SETUP.md instructions + graceful DB fallback |
| PROSPECT_TOKEN_SECRET not set | Will throw at runtime — documented in SETUP.md under secrets |
| CORS DB query latency | Acceptable at MVP scale; satellite domain count is O(10) |
| quiz_schema format assumptions | Implemented with defensive parsing; unknown formats return no required fields (liberal validation) |
| Empty cal_username | booking_url handled gracefully (null or omitted) |
