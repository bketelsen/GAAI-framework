## Technical Decisions

- **TD-1:** Widget as isolated module `booking-widget.ts` exporting `getBookingWidgetStyles(): string` and `getBookingWidgetScript(): string`.
- **TD-2:** Widget JS is a self-invoking IIFE stored as template-literal string. Listens for `booking-open` CustomEvent. Identical pattern to results.ts IIFE.
- **TD-3:** State machine: `IDLE → DATE_STEP → SLOT_STEP → HOLD_STEP → SUCCESS` / `HOLD_STEP → EXPIRED → DATE_STEP`.
- **TD-4:** Widget DOM injected dynamically into the clicked `.match-card` below `.booking-btn`. Only one widget instance in DOM at a time.
- **TD-5:** `booking-open` event `detail: { expertId, expertName }` — results.ts updated to include `expertName`.
- **TD-6:** Email pre-fill: results.ts saves `sessionStorage.setItem('match:identified_email', email)` on email gate success. Widget reads it.
- **TD-7:** Availability API: `GET {apiUrl}/api/experts/{expertId}/availability` → `{ slots: string[] }`. Slots are UTC ISO strings.
- **TD-8:** Calendar: 14-day range. Desktop: 2-row 7-column grid. Mobile (<480px): horizontally scrollable strip.
- **TD-9:** Slot chips: `<button class="bw-slot-chip">HH:MM</button>` per available slot. Selected chip gets `bw-slot-chip--selected`.
- **TD-10:** Hold API: `POST {apiUrl}/api/bookings/hold` with `{ expert_id, start_at, end_at, prospect_id, token }`. 409 slot_taken → re-fetch slots + toast. 409 max_holds → inline message.
- **TD-11:** Countdown: `held_until` UTC ISO → `setInterval(1000)` → `mm:ss`. When 0: `EXPIRED` state.
- **TD-12:** Confirm API: `POST {apiUrl}/api/bookings/{bookingId}/confirm`. 409 → reset to DATE_STEP. 422 details → field errors. 422 Hold expired → EXPIRED. 502 → retry message.
- **TD-13:** Add `prep_token: booking.prep_token` to confirm.ts response (already in SELECT).
- **TD-14:** Prep URL: `https://${window.location.hostname}/prep/${data.prep_token}`.
- **TD-15:** Accessibility: focus to widget on open, back to CTA on close. `aria-label` on date cells and slot chips. `aria-describedby` on errors.
- **TD-16:** Animation: CSS `max-height` transition 0→2000px via `.bw-container--open` class.
- **TD-17:** PostHog: `firePostHog()` helper same pattern as results.ts. 7 events per AC10.
- **TD-18:** `escHtml()` inline in IIFE.
- **TD-19:** Tests follow exact pattern of `results.test.ts` — static HTML inspection only.

### Step 3 — `booking-widget.ts`: create module

#### `getBookingWidgetStyles()` — CSS string (no `<style>` tags)

All classes prefixed `bw-`. Key classes:
- `.bw-container` — `max-height:0; overflow:hidden; transition:max-height 0.3s ease; border-top:1px solid #e5e7eb; margin-top:0.5rem`
- `.bw-container--open` — `max-height:2000px`
- `.bw-inner` — `padding:1.5rem 0`
- `.bw-close-btn` — top-right close button
- `.bw-skeleton-row` / `.bw-skeleton-day` — loading state
- `.bw-cal-day--available` — clickable, highlighted days
- `.bw-cal-day--unavailable` — grayed out, `pointer-events:none`
- `.bw-cal-day--selected` — primary color background
- `.bw-slot-chip` — time slot button, min 44px touch target
- `.bw-slot-chip--selected` — selected slot style
- `.bw-countdown` — hold timer display with `aria-live="polite"`
- `.bw-input`, `.bw-textarea`, `.bw-label`, `.bw-field-error` — form fields
- `.bw-confirm-btn` — submit button
- `.bw-step-success`, `.bw-step-expired` — end states
- Mobile `@media (max-width:480px)` — scrollable calendar strip, full-width form
- `@keyframes bw-shimmer`, `@keyframes bw-spin`

#### `getBookingWidgetScript()` — JS IIFE string

State variables: `STATE, currentExpertId, currentExpertName, currentSlots, slotsByDate, selectedDate, selectedSlot, bookingId, heldUntil, countdownInterval, containerEl, triggerBtn`

Key functions:
- `openWidget(expertId, expertName)` — creates container, inserts in card, fetches availability
- `closeWidget()` — removes container, returns focus to trigger
- `fetchAvailability()` — GET /availability, handles 422 gcal_not_connected
- `buildSlotsByDate(slots)` — groups UTC ISO strings by local date
- `renderDateStep()` — 14-day calendar grid with day headers
- `handleDateSelect(dateKey)` — fires PostHog, calls `renderSlotStep()`
- `renderSlotStep()` — slot chips for selected date
- `holdSlot()` — POST /hold, handles 409 slot_taken + max_holds
- `renderHoldStep()` — countdown + confirm form, pre-fills email from sessionStorage
- `startCountdown()` / `clearCountdown()` — `setInterval` countdown from `heldUntil`
- `handleHoldExpired()` — EXPIRED state with reselect button
- `handleConfirm()` — validates form, POST /confirm, handles all error codes
- `renderSuccess(data, email)` — success state with Meet link + prep link
- `handleGcalNotConnected()` — hides widget, shows fallback message
- `renderInlineError(msg, retryFn)` — generic error with optional retry
- `showFieldError(inputId, msg)` — field-level validation error
- `showConfirmError(msg, showRetry)` — confirm-level error
- `buildHeaderHTML(expertName)` — common header with close button
- `buildSkeletonHTML(expertName)` — loading skeleton
- `formatLocalDate(d, tz)` — returns 'YYYY-MM-DD' in local TZ
- `formatTimeLabel(d, tz)` — returns 'HH:MM' in local TZ
- `formatDayAriaLabel(d)` — returns 'Lundi 2 mars' format
- `formatHumanDate(d)` — returns 'lundi 2 mars' for slot header
- `formatSlotLabel(d, tz)` — returns 'lundi 2 mars à 10:00' for success state
- `firePostHog(event, props)` — PostHog capture helper
- `escHtml(str)` — HTML escape helper

## Risk Register

| Risk | Mitigation |
|------|-----------|
| TypeScript type errors on `booking.prep_token` | Verify `BookingRow` type has `prep_token: string \| null` |
| Widget JS string contains backtick/dollar signs that break template literal | Use `\`` escaping in the returned string |
| `window.__SAT__` not available when widget script runs | Guard with `if(!window.__SAT__)return;` at IIFE entry |
| sessionStorage not available (privacy mode) | Wrap all sessionStorage calls in try/catch |
| Mobile: calendar grid overflow | Verified: `overflow-x:auto` + `display:flex; flex-wrap:nowrap` |
| Countdown timer memory leak | `clearCountdown()` called in `closeWidget()` and state transitions |
