# Execution Plan — E06S24: Service Bindings — Worker Split

## Tier
Tier 3 — Core Team + Specialists (complexity 9, files 21 changed, criteria 8)

## Approach

Split the monolithic callibrate-core Worker into two independently-deployable Workers:

- **`callibrate-matching`** — compute-heavy: Vectorize semantic search, Workers AI embedding, D1 edge cache, scoring pipeline
- **`callibrate-core`** — gateway: auth, routing, CORS, security, delegation via Service Binding

Communication is zero-latency via Service Binding (`env.MATCHING_SERVICE.fetch()` — in-process RPC, no network hop).

## Route Contracts (Matching Worker)

### `POST /match`
Input: `{ prospect_id: string, satellite_id?: string }`
Pipeline: load prospect + weights → load expert pool (3-tier) → Vectorize pre-filter → blended scoring (0.7×exact + 0.3×vector) → write top-20 matches → return `{ computed: number, top_matches: MatchResult[] }`

### `POST /embed`
Input: `{ expert_id: string, profile?: {...}, rate_min?: number, rate_max?: number, availability?: string }`
Fire-and-forget via `ctx.waitUntil`. Returns `{ ok: true }` immediately.

### `POST /admin/reindex`
No auth (Core auth-gates before proxying). Batch reindex all available experts via Workers AI + Vectorize. Returns `{ queued: number }` with 202.

## AC6 Fallback Strategy

When `MATCHING_SERVICE` is undefined (dev environment, CI):
- `handleProspectSubmit` → local deterministic scoring via `scoreMatch()` (no vector component)
- `handleMatchCompute` → local deterministic scoring via `scoreMatch()`
- Embedding triggers → silently no-op (if-guard on `env.MATCHING_SERVICE`)
- Admin reindex → 503 (no fallback — requires Matching Worker)
