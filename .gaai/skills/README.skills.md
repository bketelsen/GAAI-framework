# GAAI Skills ‚Äî Index

Skills are **pure execution units**. They perform a single, well-defined operation and produce explicit outputs. They never reason about intent or strategy.

> Agents decide. Skills execute.

---

## Structure

Each skill lives in its own directory with a `SKILL.md` file:

```
skills/
‚îú‚îÄ‚îÄ discovery/   ‚Üê produce artefacts (PRD, Epics, Stories, validation)
‚îú‚îÄ‚îÄ delivery/    ‚Üê orchestrate and execute (planning, implementation, QA)
‚îú‚îÄ‚îÄ cross/       ‚Üê memory, context, governance, analysis ‚Äî usable by any agent
‚îî‚îÄ‚îÄ domains/     ‚Üê domain-specific skill packs (mirrors memory/domains/)
    ‚îî‚îÄ‚îÄ content-production/
```

**The source of truth for available skills is the directory itself.**
The `skills-index.yaml` file in this directory is a derived cache ‚Äî generated by `build-skills-index`, never edited manually.

## Discovery Protocol

Skills are loaded in three phases ‚Äî load the minimum needed, not everything:

### Phase 1 ‚Äî Filter by frontmatter (read SKILL.md frontmatter only)

Read `skills-index.yaml` for a fast aggregated view, or scan `SKILL.md` frontmatter directly if the index is absent or stale.

Frontmatter fields available for filtering:

| Field | Values | Use for |
|---|---|---|
| `metadata.category` | `discovery` \| `delivery` \| `cross` | Narrow by agent track |
| `metadata.track` | `discovery` \| `delivery` \| `cross-cutting` | Secondary filter |
| `metadata.tags` | list of keywords | Cross-cutting capability search |
| `description` | one sentence | **Primary relevance signal** ‚Äî what it does + when to activate |
| `metadata.id` | `SKILL-{CAT}-{NNN}` | Exact lookup by ID |

**Directive:**
1. Use `metadata.category` to scope to the right directory first
2. Read `description` to confirm relevance ‚Äî it is the single most important field
3. Use `metadata.tags` for cross-cutting capabilities (e.g. `memory`, `governance`, `qa`)
4. Do **not** load all skills ‚Äî load only SKILL.md files whose description matches your task

### Phase 2 ‚Äî Load full SKILL.md

Load the full `SKILL.md` body only for skills confirmed relevant in Phase 1.
Budget: <5,000 tokens per skill, <500 lines.

### Phase 3 ‚Äî Load references/ (execution only)

Load files in `references/` or `assets/` only during execution, never at discovery time.

## Invocation Rules

1. Skills are **never invoked implicitly** ‚Äî an agent always selects and invokes explicitly
2. Skills execute in **isolated context windows** ‚Äî no shared state between skills
3. Skills **never chain** other skills ‚Äî only agents orchestrate
4. Skills **never access memory** autonomously ‚Äî context is always provided by the agent
5. Skills **never make product or architectural decisions** ‚Äî they execute only

## Skill Authoring Guidance

### When to add a skill vs. when to add a rule

| You want to... | Use... |
|---|---|
| Add a new **execution capability** (something an agent will *do*) | `create-skill` |
| Add a new **constraint** (something an agent must *not* do, or a standard it must follow) | `rules-normalize` |

**Skills** are procedural: they perform a defined operation and produce outputs. They live in `.gaai/skills/`.

**Rules** are declarative: they state constraints, policies, and governance boundaries. They live in `.gaai/contexts/rules/`.

### Decision test

Ask: "Does this describe *how to do something*, or *whether something is allowed*?"

- "How to generate a Story from an Epic" ‚Üí skill (`generate-stories`)
- "Stories must have acceptance criteria before entering the backlog" ‚Üí rule (`orchestration.rules.md`)
- "How to compact memory when it exceeds a size threshold" ‚Üí skill (`memory-compact`)
- "Memory must never be auto-loaded by a skill" ‚Üí rule (`orchestration.rules.md`)

If your answer is "how to do something" ‚Üí skill.
If your answer is "a constraint the system must enforce" ‚Üí rule.

### How to create each

- New skill: invoke `create-skill` (`.gaai/skills/cross/create-skill/SKILL.md`)
  ‚Üí `create-skill` Step 6 invokes `build-skills-index` to regenerate `skills-index.yaml`
- New rule: invoke `rules-normalize` (`.gaai/skills/cross/rules-normalize/SKILL.md`)
